# --interactive--

在网页开发中，隐藏网页内容是一种常见做法，但关键是要以一种不会损害无障碍性的方式进行。不同的隐藏技术对辅助技术如何解释和向用户呈现内容有不同的影响。让我们探讨一些在保持无障碍性的同时隐藏内容的最佳实践。隐藏内容的一种常见方法是使用`display: none`。这是一个示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">隐藏的文本</p>
<p>可见的文本</p>
```

```css
.hidden {
  display: none;
}
```

:::

虽然这有效地在视觉上隐藏了内容，但它也从无障碍树中移除了内容。无障碍树是辅助技术（如屏幕阅读器）用来解释和与网页内容交互的结构。它以辅助技术能够理解和呈现给用户的方式表示内容及其语义含义。

使用`display: none`意味着屏幕阅读器和其他辅助技术将无法访问此内容，因为它不包含在无障碍树中。因此，只有当你想完全从视觉呈现和无障碍性中移除内容时，才应使用此方法。

另一种隐藏内容的方法是使用`visibility: hidden`：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">隐藏的文本</p>
<p>可见的文本</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden`在视觉上隐藏内容，但保持它在文档流中，这意味着它仍然占据页面上的空间。与`display: none`一样，`visibility: hidden`也从无障碍树中移除内容。这意味着辅助技术（如屏幕阅读器）将无法访问隐藏的内容。只有当你想对所有人（包括使用辅助技术的人）隐藏内容时，才使用`visibility: hidden`。

对于应该在视觉上隐藏但对屏幕阅读器保持可访问的内容，你可以使用通常被称为"视觉隐藏"或"仅屏幕阅读器"的技术。以下是一个使用`.sr-only` CSS类的示例，这是一种常见的技术，用于在视觉上隐藏内容，同时保持其对屏幕阅读器的可访问性：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">隐藏的文本</p>
<p>可见的文本</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

在这个示例中，我们使用了`position`、`clip`和`white-space`等属性，你将在未来的课程中学习这些属性。现在，只需知道这个CSS规则有效地在视觉上隐藏了内容，同时保持其对屏幕阅读器的可访问性。这对于向屏幕阅读器用户提供额外的上下文而不会影响视觉布局很有用。

对于切换内容可见性，考虑使用`hidden`属性：

:::interactive_editor

```html
<p hidden>此内容被隐藏</p>
<p>此内容可见</p>
```

:::

`hidden`属性被大多数现代浏览器支持，并且在视觉上和从无障碍树中都隐藏内容。它可以通过JavaScript轻松切换。最后，要谨慎隐藏重要内容。如果信息对于理解或使用网站至关重要，它应该对所有用户都可见和可访问。只有在真正增强用户体验时才隐藏内容。通过遵循这些最佳实践，你可以确保你的内容对所有用户都保持可访问，无论他们如何与你的网站交互。
