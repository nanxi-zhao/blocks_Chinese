# --interactive--

让我们学习什么时候使用`appearance: none`来解决搜索输入框、复选框和单选按钮的样式问题。

浏览器对许多元素应用默认样式。对于输入元素，你用CSS样式化它们的能力可能会感到相当受限。因此，你可能想要使用`appearance: none`来隐藏默认元素的某些方面，并构建你自己的样式。例如，这将隐藏复选框输入的默认复选框，允许你使用自定义指示器，如绿色勾号和红色X来显示状态。对于搜索输入框，基于WebKit的浏览器会显示默认的搜索图标和取消按钮。隐藏这些元素允许你创建在所有浏览器上都会出现的自己的指示器。

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}
```

:::

WebKit是一个帮助网页浏览器显示网站的软件引擎。像Safari这样的浏览器使用WebKit来确保网页看起来和工作正常。这个`appearance: none` CSS属性让你完全控制样式，但它确实有一些需要注意的地方。输入元素的默认交互组件包括焦点和错误指示器等功能，你需要确保这些不会丢失。

创建一致的跨平台样式是使用此属性的一个很好的理由。你也可以使用它来确保移动设备上的点击目标足够大，或者复选框的颜色有足够的对比度。
