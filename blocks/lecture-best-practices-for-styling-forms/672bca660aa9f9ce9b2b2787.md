# --interactive--

让我们学习在尝试为特殊输入元素（如`datetime-local`和`color`输入）设置样式时的一些常见问题。

这些特殊类型的输入依赖于复杂的伪元素来创建日期和颜色选择器等组件。这给为这些输入设置样式带来了重大挑战。一个挑战是默认样式完全依赖于浏览器，所以你编写的让选择器看起来符合预期的CSS在另一个浏览器上可能完全不同。

以下是一个颜色输入的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">选择你喜欢的颜色：</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

另一个可能是伪元素的复杂性。考虑日期选择器；这里有很多移动部件，伪元素的复杂结构可能会在将样式应用到正确区域时带来重大挑战。

以下是一个日期输入的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">选择你的出生日期：</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

当然，对于这些复杂元素，当你手动为它们设置样式时，你也面临着意外丢失重要功能的风险。你不仅可能丢失重要的指示器（如焦点状态或选中项），还可能完全破坏选择器。

由于这些原因，许多开发人员依赖JavaScript库或完全自定义组件，而不是使用浏览器的内置组件。
