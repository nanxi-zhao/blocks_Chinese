# --description--

在设计新功能或应用程序时，一个好的第一步是创建设计简报。

设计简报是一份概述项目目标、目的和要求的文档。它是指导设计过程的路线图，确保最终产品能满足客户的需求。

通常由客户编写设计简报，它将作为工作草案。有时，设计师可能会编写一份并与客户协商，以确保它满足他们的需求。

设计简报中应包含几个关键要素。

第一个要素是项目和业务的概述。这个概述应包括公司的详细信息、使命、价值观、独特卖点以及产品或服务。

下一个关键要素应该是记录项目的目标和目的。这应包括项目的目的和预期结果。

目标的例子包括增加网站流量或按 X 百分比增加每月页面访问量。

另一个关键要素是目标受众。设计简报应包括有关目标人口统计、兴趣和受众需求的信息。

你还应该包括有关竞争对手的信息，以及项目将如何与竞争对手区分开来。

另一个关键要素是项目范围。这应包括交付物、时间表和预算。交付物应包括作为项目一部分将产生的所有项目的列表，例如模型和最终设计。

如果没有明确定义项目范围，事情可能会失控并超出预算。因此，最好尽可能详细地说明预期交付的内容和时间。

项目设计中具有挑战性的方面之一是时间安排和预算。重要的是要现实地看待在给定的时间框架和预算内可以实现什么。因此，制定一份概述这些约束条件的设计简报很重要。

一旦讨论并记录了所有这些细节，设计简报应在项目开始前由所有利益相关者审查和批准。在那一点上，设计师就可以开始他们的工作了。

那么，开发人员在这一切中的角色是什么？开发人员的角色是接受设计，理解项目要求，并将其转化为工作产品。

这包括编写代码、测试和调试应用程序，以确保它满足设计简报中概述的要求。

通常，开发人员将在团队中工作，工作会在多个开发人员之间分配。

通常还会有一个项目经理，负责协调工作并确保项目按计划进行。

因此，虽然作为开发人员，你可能不会参与设计和初始决策过程，但了解设计简报以及它将如何影响你的工作仍然很重要。
