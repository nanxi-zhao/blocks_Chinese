# --interactive--

继承（Inheritance）是 CSS 的重要概念之一，用来描述样式如何从父元素向子元素传递。

就像现实中孩子可能继承父母的某些特征一样，在 CSS 中某些属性也会由父元素传给子元素。

这可以让你更高效地在文档中保持样式一致性。

需要注意的是，并非所有属性默认都会被继承。例如 `color`、`font-family`、`line-height` 等通常会被继承；也就是说如果你在父元素上设置了文本颜色，子元素会继承该颜色，除非你显式覆盖它。

比如下面这个例子，父元素 `div` 通过内联样式设置了颜色，子元素 `p` 会继承该颜色：

:::interactive_editor

```html
<div style="color: blue;">
  This is the parent element.
  <p>This is the child element inheriting the color.</p>
</div>
```

:::


在这种情况下，父 `div` 和子 `p` 都会显示为蓝色，因为颜色是被继承的。

另一方面，像 `margin`、`padding`、`border` 和 `background` 这些属性默认并不会被继承。如果你希望子元素继承这些属性，需要显式设置，或者使用 `inherit` 关键字。

`inherit` 可以强制让子元素继承父元素的某个属性，即便该属性通常不被继承。

例如，如果你希望某个子元素的 `padding` 与父元素相同，可以在子元素上写 `padding: inherit`：

:::interactive_editor

```html
<div style="padding: 20px;">
  This is the parent element with padding.
  <p style="padding: inherit;">This is the child element inheriting the padding.</p>
</div>
```

:::


在这个例子中，子 `p` 会从父 `div` 继承到 `20px` 的 padding。

继承在保持样式一致性和减少重复代码方面尤其有用。

你可以把通用样式写在父元素上，子元素自然继承，这样可以让 CSS 更简洁、更易维护。

但要记住，继承是单向的——从父到子。如果你在子元素上覆盖了某个样式，这并不会影响父元素。
