# --description--

开发者工具、检查工具与 CSS 校验器是调试 CSS 的好帮手，能帮助你发现语法与布局问题，让样式表更可靠、更高效。

这些工具能直观展示 CSS 是如何被应用到页面元素上的，便于你定位和修复问题。

首先介绍浏览器自带的开发者工具（通常称为 DevTools）。主流浏览器（如 Chrome、Firefox、Safari）都内置了 DevTools。你可以在页面元素上右键选择“检查（Inspect）”，或使用快捷键 `F12`（macOS 上为 `Cmd+Option+I`）打开它们。

DevTools 可以实时查看并修改 CSS。样式（Styles）面板会列出所选元素的所有 CSS 规则（包括继承的样式）。

你可以开关单个属性、修改属性值，甚至直接在浏览器中新增规则。即时反馈非常适合用于尝试不同的样式而不改动源代码。

检查工具还能在鼠标悬停时显示元素的盒模型信息——包括外边距（margin）、边框（border）、内边距（padding）与内容区域（content），这对排查布局问题尤其有帮助。

另一个常用工具是 CSS 校验器，例如 W3C CSS Validator。校验器会根据官方规范检查你的 CSS，并报告错误或警告。你可以上传 CSS 文件、直接粘贴代码，或通过 URL 校验。

例如，下面的 CSS 中缺少了一个分号：

```css
.container {
  width: 100%;
  height: 200px
  background-color: #F0F0F0;
}
```

校验器会提示 `height` 属性后缺少分号。虽然这类错误很容易忽略，但会导致样式表出现显著问题。

在调试响应式布局时，DevTools 的设备模拟功能非常实用。它可以模拟不同屏幕尺寸和设备，帮助你发现断点（breakpoint）问题或样式在不同视口下无法正常缩放的情况。

通常把这些工具组合使用会事半功倍：先用校验器捕捉语法错误，再用 DevTools 定位具体元素并尝试修改，最后用设备模拟确认不同屏幕下的效果。

熟练使用这些调试技巧能显著加快修复速度，让你的样式更健壮。继续练习，你会越来越得心应手！
