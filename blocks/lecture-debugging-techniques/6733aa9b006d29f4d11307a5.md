# --description--

在用 JavaScript 编程时，你肯定遇到过各种错误信息。理解常见错误有助于你更有效地调试，逐步成为更可靠的开发者。常见的四类错误包括 `SyntaxError`、`ReferenceError`、`TypeError` 和 `RangeError`。

`SyntaxError`（语法错误）发生在代码写错语法时，例如缺少括号或分号。把它想成句子里的语法错误。下面是创建数组时常见的错误示例：

```js
const arr = ["Beau", "Quincy" "Tom"]
```

数组元素之间必须用逗号分隔，否则会报错。

第二类常见错误是 `ReferenceError`（引用错误）。`ReferenceError` 有多种触发方式，其中一种是引用未定义的变量：

```js
console.log(price);
```

在这个例子中，我们试图打印 `price`，但它并没有定义，因此会抛出 `ReferenceError`。

另一种常见场景是，在用 `let` 或 `const` 声明的变量被定义之前就访问它：

```js
console.log(b);
const b = 50;
```

上面的代码会报 `Cannot access 'b' before initialization`。

第三类是 `TypeError`（类型错误），当你对错误的数据类型执行不支持的操作时会发生。例如对对象使用数组方法 `map`：

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

这会报 `developerObj.map is not a function`，因为 `map` 是数组方法，不适用于对象。

最后一种是 `RangeError`（范围错误）。当代码尝试使用超出 JavaScript 可接受范围的数值时会触发，例如给数组长度赋一个无效索引：

```js
const arr = [];
arr.length = -1; 
```

因为 `-1` 不是合法的数组长度，这会导致 `RangeError`。

随着编程实践的积累，留意这些错误类型以及它们出现的原因，会让你更快定位并修复问题，加油！
