# --description--

`debugger` 语句是一个强大的 JavaScript 调试工具，可以在指定行暂停代码执行，帮助你检查程序状态。正确使用 `debugger` 能为你节省大量排查问题的时间。

JavaScript 按从上到下的顺序执行代码。当执行到 `debugger` 语句时，如果开发者工具（DevTools）处于打开状态，执行会在该行暂停，允许你检查变量、函数以及代码的执行流程。

通过这样的暂停，你可以准确看到程序哪里正确、哪里出问题。但如果浏览器的开发者工具未打开，`debugger` 语句会被忽略，代码照常运行。

现代浏览器在遇到 `debugger` 时会暂停执行，并允许你通过点击播放按钮继续执行。页面不会自动刷新，除非你手动触发刷新。

把 `debugger` 放在想要暂停的那一行。例如：

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // 代码会在此处暂停
let sum = firstNumber + secondNumber;
console.log(sum);
```

在这个例子中，如果你在代码运行前没有打开控制台、后来打开并刷新页面，你会看到执行在 `debugger` 处暂停，然后可以观察到 `15`。

更复杂的示例：

```js
function calculateTotalPrice(price, discountPercentage) {
	debugger
	let discountAmount = (price * discountPercentage) / 100
	let totalPrice = price - discountAmount

	console.log(`Original Price: ${price}`)
	console.log(`Discount Amount: ${discountAmount}`)
	console.log(`Total Price after Discount: ${totalPrice}`)

	return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

在此示例中，`debugger` 放在 `calculateTotalPrice` 函数内部，函数被调用时会暂停执行。

当你打开控制台并刷新页面时，会发现页面不断在 `debugger` 处暂停，说明执行被暂停而不是停止；点击播放按钮可以继续执行。

在以后的开发中，试试看将 `debugger` 用到你的代码里，它能帮助你更好地理解执行流程、实时检查变量，并快速定位问题所在。
