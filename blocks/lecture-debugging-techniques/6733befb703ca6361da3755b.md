# --description--

调试 JavaScript 程序不仅仅是插入 `console.log()`。还有一些更高级的调试技巧，一旦习惯了它们，调试会变得轻而易举。下面我们介绍这些技巧，帮助你在遇到错误时更自信地解决问题。

首先是断点（breakpoints）。断点允许你在指定行暂停代码执行，暂停后可以检查变量、求值表达式并查看调用栈。

在 Chrome 中为某行设置断点：打开开发者工具（DevTools），切换到 Sources 面板，打开要调试的 JavaScript 文件，点击行号即可设置断点。

当执行到断点处暂停后，你可以使用界面上的控件逐步执行（step over / step into / step out）。你也可以右键行号添加条件断点（Add conditional breakpoint…）。

接下来是监视表达式（watch expressions），它可以在代码运行时持续监控变量或表达式的值，即使这些变量不在当前作用域中。

在 Sources 面板中找到 watch 面板，点击加号 (`+`) 添加要监视的表达式。

再来是性能分析（profiling）。性能分析可以帮助你定位性能瓶颈，记录 CPU 使用、函数调用和执行时间，甚至捕获执行快照。

在 Performance 面板中点击录制（record），执行你想分析的操作，停止录制后即可查看分析结果，找出耗时最多的函数或代码片段。

如果应用在某些操作时出现卡顿，性能分析器能帮助你找出是哪部分代码占用了最多资源。

接着是网络请求的检查（network inspection）。它能帮助你调试 API 请求相关的问题，例如参数错误、地址错误或服务器返回错误。

在 Network 面板中点击具体请求，可以查看请求头（headers）、响应（response）和请求体（payload）等详细信息。

最后介绍两个有用的控制台方法：`console.table` 与 `console.dir`。

`console.table()` 可将表格式数据以表格形式显示在控制台，接受一个必须是数组或对象的参数，以及一个可选参数指定要显示的字段。

`console.dir()` 则以可展开的层级结构展示指定对象的属性，方便查看嵌套属性。

现在你已经掌握了更多调试工具，把它们应用到实际项目中，会让你更快定位并解决问题，写出更整洁、更可靠的代码，继续保持好奇心和练习！
