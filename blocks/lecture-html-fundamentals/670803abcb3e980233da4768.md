# --description--

`div` 元素用作容器，用来把其它元素组合在一起。

下面是一个 `div` 的示例。请在 `div` 内再添加一个段落元素，然后在预览窗口中查看变化。

:::interactive_editor

```html
<div>
  <p>示例段落。</p>

</div>
```

:::

当你希望把一组 HTML 元素聚合在一起并共享同一组 CSS 样式时，通常会使用 `div` 元素。后续课程会详细讲解 CSS。

尽管 `div` 在真实项目中使用广泛，但也不要滥用。在有语义化元素更合适时，应优先使用语义化元素。

例如，如果你想把内容分成若干章节，`section` 元素比 `div` 更语义化、更合适。

请在第一个 `section` 下面添加另一个 `section`，在其中放入 `h2` 和 `p` 元素，输入任意文本并在预览中查看效果。

:::interactive_editor

```html
<section>
  <h2>哺乳动物</h2>
  <p>哺乳动物是有毛发或体毛的温血动物，大多数会产下活体幼崽。</p>
  <ul>
    <li>狮子</li>
    <li>大象</li>
    <li>海豚</li>
  </ul>
</section>
```

:::

`section` 相对于 `div` 是有语义含义的。语义（semantics）类似于语言中单词或短语的含义。HTML 元素也有语义含义，因此使用 `section` 可让浏览器和辅助工具理解该区域是独立的章节或部分，无论是在桌面、移动还是其他设备上。
  </ul>
</section>
```

:::

`section` 元素比 `div` 元素具有语义含义，而 `div` 元素没有语义含义。语义是语言中单词或短语的含义。HTML 是一种语言，元素也有自己的语义含义。因此，这意味着如果你使用了 `section` 元素，浏览器将获取其语义含义并理解应将其视为一个独立部分——无论是在台式机上、移动设备上，还是其他任何设备上。
