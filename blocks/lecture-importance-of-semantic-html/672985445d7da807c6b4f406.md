# --interactive--

构建良好的结构层级的关键是：正确使用标题元素（例如 `h1`、`h2`、`h3` 等），这些标签的数字表示它们在文档中的层级。

就像写文章一样，我们不应该打乱标题顺序：`h1` 是最高级别的标题，一页中通常只会有一个 `h1`，并且它通常位于内容的开头；`h2` 是 `h1` 的子标题，你可以使用多个 `h2` 来划分不同的章节。

按照层级规则，`h3` 应该跟在 `h2` 之后 —— 也就是说，不应该直接从 `h1` 跳到 `h3`。下面是一个语义正确的示例：

```html
<div>
  <section>
    <h1>MasterPuti</h1>
    <h2>Learn Front-End Development</h2>
    <h2>Learn Back-End Development</h2>
  </section>
</div>
```

下面的代码则不正确，因为 `h3` 出现在 `h2` 之前：

:::interactive_editor

```html
<div>
  <section>
    <h1>MasterPuti</h1>
    <h3>Introduction to HTML</h3>
    <h2>Learn Frontend Development</h2>
  </section>
</div>
```

:::

有时人们会因为想要更大的文字而错误地使用某个标题级别（例如把 `h1` 作为普通的“大字”），但应避免这样做。正确做法是选择语义合适的元素，再用 CSS 控制样式：

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

视觉样式应由 CSS 负责，而不是改变元素的语义。

结构层级的重要性：

- 无障碍：屏幕阅读器等辅助工具依赖清晰的标题结构来正确朗读页面；错误的层级可能会让用户错过关键信息。
- SEO：搜索引擎会根据结构判断页面的主题与重要性，结构混乱可能影响搜索结果排名。
- 程序兼容性与规范：严重的结构错误可能导致 HTML 无效，浏览器会尝试修复，而它的修复结果可能不是你想要的。

总之，保持良好的结构层级会让页面对用户、开发者和自动化工具都更友好，记得在构建页面时先考虑语义结构，再调整样式。
