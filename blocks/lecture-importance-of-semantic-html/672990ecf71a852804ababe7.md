# --interactive--

表现性（presentational）HTML 更侧重页面的外观与样式。在早期的 HTML 中，开发者常用像 `center`、`big`、`font` 这样的标签来直接改变外观。但在现代前端实践中，应避免使用这些标签，因为它们会降低可访问性并增加维护成本。

这些表现性标签中的许多已被弃用（deprecated），也就是说有更好的（通常更语义化且可维护）的替代方式——例如使用 CSS 来控制视觉效果。尽管如此，了解它们有助于理解历史与语义化的价值，我们来看几个示例。

`font` 曾用于设置文本的大小和颜色。示例（把 `size` 从 5 改为 7 可以看到字体变大）：

:::interactive_editor

```html
<font size="5" color="blue">This text is blue and large.</font>
```

:::

上例里 `size` 的范围是 1 到 7（默认 3）。虽然这个标签仍能工作，但推荐的做法是使用 CSS 来设置字号和颜色，而不是在 HTML 里硬编码这些表现性属性。

`center` 标签曾用于居中内容。下面的示例展示了 `center` 包裹文本与段落的用法；你可以把 `<p>Another example text.</p>` 加上 `center` 标签来观察变化：

:::interactive_editor

```html
<center>
  This text is centered.
  <p>HTML is awesome.</p>
</center>

<p>Another example text.</p>
```

:::

`big` 标签会把包含的文本放大一级，下面是示例（尝试在 `Some other text` 周围添加 `big`）：

:::interactive_editor

```html
<p>
  This text has a normal font size.
  <big>This text is larger.</big>
  Some other text.
</p>
```

:::

总之，表现性标签会影响外观，但现代开发应把样式交给 CSS 来处理。

接下来，语义化 HTML（semantic HTML）描述的是元素的含义与用途，因而更易读、易维护，也更利于无障碍与搜索引擎理解页面内容。

## --text--

以下哪一项最能说明“表现性 HTML”与“语义化 HTML”之间的区别？

## --answers--

- 表现性 HTML 更关注页面的外观与样式，而语义化 HTML 更关注内容的含义与结构。

### --feedback--

很好！记得区分“外观（presentation）”和“含义（semantics）”。语义化元素描述内容本身。

---

- 语义化 HTML 更注重内容结构，而表现性 HTML 侧重外观。

---

- 表现性 HTML 与语义化 HTML 没有区别。

### --feedback--

不是的：两者的关注点不同。

---

- 两者都以样式为主，但表现性 HTML 更现代。

### --feedback--

不完全正确：表现性标签通常是旧做法，现代推荐用 CSS 控制样式并用语义元素描述内容。
