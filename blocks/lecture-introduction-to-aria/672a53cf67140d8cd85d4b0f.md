# --interactive--

让静态内容对所有人都可访问相对容易，但动态内容和自定义组件的无障碍性更具挑战——这正是 WAI-ARIA 发挥作用的地方。

下面我们来了解什么是 WAI-ARIA、它的用途、工作原理，以及一些示例。

WAI-ARIA 是“Web Accessibility Initiative - Accessible Rich Internet Applications”的缩写，用于增强动态内容与用户界面组件的无障碍性。

注意：WCAG 与 WAI-ARIA 并非同一概念——WCAG 提供总体的无障碍指南，而 WAI-ARIA 提供在动态和交互式内容中向辅助技术传达语义信息的具体方法。

WAI-ARIA 的主要目标是为那些没有直接 HTML 等价项的动态内容与 UI 组件提供可访问性支持。

它通过一组以 `aria-` 前缀命名的属性来工作，这些属性一般分为角色（role）、状态（state）与属性（property），用于向辅助技术传达额外的语义信息。

ARIA 角色用来表明元素在页面中的用途。例如，你可以把一个 `div` 标记为按钮的角色：

```html
<div role="button">Click Me</div>
```

这会告诉辅助技术该元素应被视为“按钮”，但请注意：为元素设置角色并不会自动赋予其行为（例如响应鼠标或键盘事件）。要让自定义元素像按钮那样工作，仍需要配合相应的 CSS 与 JavaScript（因此在可能的情况下，优先使用原生的 `<button>` 或带有 `type="button"` 的 `<input>`）。

下面给出一个使用 HTML、CSS 和 JavaScript 创建自定义按钮的示例（不用担心代码细节，你会在后续课程学到这些语言）：

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

ARIA 属性还能为元素提供更多细节信息，例如 `aria-labelledby` 可以把元素与页面上已存在的标签文本关联起来：

```html
<h2 id="header-id">About MasterPuti</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

这有助于辅助技术用户理解并导航这些元素。

总之：优先使用语义化的原生 HTML。如果原生元素无法满足需求，再谨慎使用 WAI-ARIA；并始终使用屏幕阅读器或请有无障碍需求的用户来测试你的实现。切记不要滥用 ARIA，以免造成混淆。
