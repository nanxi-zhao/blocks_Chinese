# --description--

确保网站对所有用户都友好、可访问是非常重要的。

对于使用屏幕阅读器的用户，`aria-label` 和 `aria-labelledby` 可以为那些没有明显可见标签的控件提供朗读文本，从而让辅助技术理解元素的用途。

两者都是 ARIA（Accessible Rich Internet Applications）属性：

- `aria-label`：直接为元素提供一个“不可见”的标签文本，适合用于只显示图标或没有可见文本的控件。例如：

```html
<button aria-label="Search">
  <i class="fas fa-search"></i>
</button>
```

即便按钮仅包含图标，屏幕阅读器仍会朗读为 “Search, button”，因为 `aria-label` 指定了应读出的文本。

- `aria-labelledby`：不是直接写入文本，而是通过引用页面上已有元素的 `id` 来作为标签。例如：

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

在这个例子中，输入框的标签内容来自按钮的可见文本；如果按钮文字从 “Search” 改为 “Find”，屏幕阅读器读出的标签也会自动随之更新。

`aria-labelledby` 还能引用多个 `id`，将多个文本节点组合成一个复合标签：

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

对于这个滑块，屏幕阅读器会把两段文本合并并朗读，例如 “Volume Adjust the volume level”。

如何选择使用哪一个？两者功能相近，但在下列情况下 `aria-labelledby` 更常用：

- 页面会被本地化/翻译时，引用可见文本的 `aria-labelledby` 更容易保持同步；
- 想避免可见标签与隐藏标签不一致的问题时；
- 需要将多个文本节点组合成复杂标签时。

不要在同一元素上同时使用 `aria-label` 和 `aria-labelledby`，因为 `aria-labelledby` 的优先级更高，会覆盖 `aria-label`。

总结：在不能或不方便使用可见标签时，使用 `aria-label`；如果已有可见文本可作为标签来源，优先使用 `aria-labelledby`，这样可以提高可维护性与本地化友好性。
