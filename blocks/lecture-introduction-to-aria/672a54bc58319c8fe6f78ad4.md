# --interactive--

如果你希望在视觉上仍显示某些内容，但对屏幕阅读器等辅助技术的用户隐藏它，可以使用 `aria-hidden` 属性。

只需在要隐藏的元素上添加 `aria-hidden="true"` 即可。该属性会把该元素及其所有子元素从可访问性树（accessibility tree）中移除，但元素仍会在页面上可见。常见用例包括：

- 纯装饰性的图标或图片；
- 与页面其他内容重复的信息。

注意：`aria-hidden` 只会影响辅助技术的可见性，而不会把元素从 DOM 中删除。如果你希望元素对所有用户都不可见（包括键盘用户），应使用 CSS（例如 `display: none`）或从 DOM 中移除该元素，而不是简单设置 `aria-hidden`。

切忌对可键盘聚焦（focusable）的元素使用 `aria-hidden="true"`；这种做法会导致键盘用户能聚焦到某个元素，但屏幕阅读器却不朗读它，从而造成困惑。

下面示例展示了如何对仅作装饰用途的图标使用 `aria-hidden="true"`，并保留用于屏幕阅读器的可访问标签文本：

**注意**：此互动示例引入了 Font Awesome CDN，以便在预览中显示齿轮图标。

:::interactive_editor

```html
<head>
  <!-- Font Awesome CDN -->
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.5.0/css/all.min.css"
  />
</head>

<button>
  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>
</button>
```

:::

在以下情况无需使用 `aria-hidden`：

- 元素已经含有原生的 `hidden` 属性；
- 元素或其祖先已使用 `display: none` 隐藏；
- 元素或其祖先已使用 `visibility: hidden` 隐藏。

在这些情况下，元素已经不在可访问性树中，因此不需要额外设置 `aria-hidden`。

和其它 ARIA 属性一样，请始终使用辅助技术测试你的实现，最好邀请有无障碍需求的用户进行实际测试，以确保体验良好。

还应注意：如果父元素设置了 `aria-hidden="true"`，即便子元素将 `aria-hidden` 设为 `false`，子元素仍不会对辅助技术可见。

总结：`aria-hidden` 可用于隐藏纯装饰或重复内容，从而简化屏幕阅读器的朗读内容，但应该谨慎使用，不要用来隐藏你认为“屏幕阅读器用户不想看”的信息——他们有权访问页面上的所有信息。
