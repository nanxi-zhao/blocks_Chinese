# --description--

如果你曾经处理过像论文这样的重要文档，并在做了重大修改后保存多个版本，取名类似 `Thesis-Draft`、`Thesis-Final`、`Thesis-Final-For-Real`、`Thesis-Absolute-Final-DO-NOT-CHANGE`，那你其实就在用一种非常原始的"版本控制"方法。现在想象一下，如果有更好、更可靠的方法可以管理这些版本，会怎么样？

你可能听说过像 Git、SVN 或 Mercurial 这样的工具。它们就是版本控制系统（Version Control Systems），是每个开发者工作流中的关键组成部分。版本控制可以让你追踪与管理项目中的更改。对于本系列课程，我们将主要聚焦于 Git。

Git 提供了强大的功能来管理项目的编辑历史。你会在后续的课程中学到如何使用 Git 以及它提供的各种功能。现在先记住几个核心概念：

- commit：将当前代码状态拍成"快照"。你可以比较不同的 commit 来查看修改内容，或者回滚到某个不正确的 commit；
- branch（分支）：把代码开发分成多条独立的路径。把分支想象成你走的那条路，而 commit 则是路上的里程碑。使用分支可以让你在不影响主线代码的情况下开发新功能，如果实验失败，只需删除该分支即可。

这些功能使得版本控制在团队协作中变得不可或缺。当多人在同一项目中工作时，分支能让每个人独立开发，避免互相覆盖对方的修改。

你通常会把 Git 这类版本控制系统与像 GitHub 或 GitLab 这样的托管平台配合使用，这类平台提供额外的协作工具并能将项目备份到云端。

即使不使用云端备份，版本控制系统也能显著改善你的工作流程。再也不用写 `THIS-IS-THE-FINAL-COPY-FOREVER` 这种文件名了。

关于如何使用这些系统有很多有趣且实用的内容，接下来的几节课我们将深入讲解！
