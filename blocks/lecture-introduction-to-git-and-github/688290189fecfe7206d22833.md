# --description--

在前一节课中我们简单提到了 GitHub。GitHub、GitLab 这类服务是版本控制托管平台（version control providers）。它们提供基于云的仓库（repositories）用于存储受版本控制的项目，并为团队协作提供许多便利功能。

先说项目类型：许多项目是"开源"（open-source）的，这意味着任何人都可以查看你发布的代码、提出修改建议、报告问题，甚至运行你修改后的版本。

但有时你只希望团队成员能够查看和修改代码。在这种情况下可以将项目设为"闭源"（closed-source），只有你明确授权的人才能访问和参与该项目。

无论哪种方式，像 GitHub 和 GitLab 这样的平台都能显著提升团队的协作效率。

举例来说：当某个同事在开发某个功能时遇到困难，他无需把代码粘贴到聊天工具里，也不用立即发起屏幕共享。只需把更改上传到 GitHub，你就可以在浏览器中查看变更，甚至把代码下载到本地运行和调试。

当同事完成了一个功能后，他可以通过这些平台发起变更请求（pull/merge request）。你可以在线查看他们的提交、在具体文件或代码行上发表评论，甚至直接在浏览器中提出建议性修改，供对方接受并应用。

如果你确认这些更改没问题，就可以把它们合并到主分支（primary/main branch）。团队的其他成员随后可以拉取最新代码，以便在开始新任务时使用最新的代码基线。

这些平台还提供强大的问题跟踪（issue）系统，你可以用它来分配任务、接收公众的 bug 报告，甚至通过项目看板（project boards）组织整个冲刺（sprint）。

还有很多其他有用的功能，接下来的课程中我们会逐步介绍，让你能快速上手并开始协作！
