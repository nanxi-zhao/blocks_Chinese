# --description--

在真正开始使用版本控制或像 GitHub 这样的托管服务之前，你需要先在本机安装并配置好 Git。

要检查机器上是否已安装 Git，可以在终端运行：

```sh
git --version
```

如果看到版本号，说明已安装；如果没有，则需要先安装 Git。

在大多数 Linux 发行版上，Git 通常会预装。如果未安装，可以使用包管理器安装，例如 `sudo apt-get install git` 或 `sudo pacman -S git`。

在 macOS 上，可以通过 Homebrew 安装：`brew install git`，或者从 Git 官方网站下载安装程序。

在 Windows 上，可以从 Git 官网下载可执行安装程序；如果你已配置 Chocolatey，也可以在 PowerShell 中运行 `choco install git.install`。在 Windows 上，通常还会安装 Git Bash，以便使用类 Unix 的命令行环境。

安装完成后，再次运行 `git --version` 以确认安装成功。

安装好 Git 后，你需要做一些配置项的设置（通常只需在这台电脑上设置一次）。

`git config` 用于设置影响 Git 在你机器上如何运行的配置变量。要查看当前的配置项及其存储位置，可以运行：

```sh
git config --list --show-origin
```

如果是首次安装，你通常只会看到系统级配置；如果以前已安装过 Git，可能还会看到用户级的 `user.name` 和 `user.email` 配置。

设置用户名的示例命令：

```sh
git config --global user.name "Jane Doe"
```

这里使用 `--global` 标志表示将该用户名设置为系统上所有 Git 项目的默认用户名。如果只想为某个特定项目覆盖该设置，可在该项目目录中运行相同命令但不加 `--global`。

设置用户邮箱的示例命令：

```sh
git config --global user.email janedoe@example.com
```

配置用户名与邮箱很重要，因为 Git 会将这些信息记录在每次提交（commit）中。

你还可以配置 Git 使用的首选编辑器，例如将编辑器设置为 Emacs：

```sh
git config --global core.editor emacs
```

其他常见选项包括 Vim、Nano、VS Code 等。若使用 VS Code，可设置为：

```sh
git config --global core.editor "code --wait"
```

若不设置首选编辑器，Git 将使用系统的默认编辑器。

在 Windows 上更改编辑器时，通常需要给出完整可执行文件路径，例如：

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

查看配置列表的命令为：

```sh
git config --list
```

至此，Git 已安装并配置完毕，你可以开始为项目添加版本控制了。
