# --description--

我们已经介绍了一些关于 Git 与 GitHub 的概念，现在可以把它们结合起来使用。要开始，你需要一个仓库（repository）。仓库可以被看作是存放一个项目的容器——如果你正在开发一个应用，就把该应用的所有文件放在对应的仓库里。

仓库可以存在于你的本地电脑上，也可以托管在像 GitHub 这样的远程服务上。创建仓库可以从本地或远程开始——两者的最终结果是相同的，按你喜欢的工作流选择即可。如果你打算从远程仓库开始，可以直接在 GitHub 网站上创建，步骤如下：

打开 <a href="https://github.com" target="_blank">GitHub 网站</a>，确认已登录，点击右上角导航栏的加号（+），在出现的选项中选择 “New Repository” 来打开创建仓库的界面。

第一个选项是选择模板（template）。模板是特殊的仓库类型，可以作为新仓库的起点，方便在每个项目中包含统一的文档或 issue 模板。

接下来选择仓库的所有者（owner），可以是你的个人账户或你有权限的组织。然后为仓库命名——最好使用简洁描述项目用途的名称，建议避免在仓库名中使用空格，GitHub 会自动把空格替换为连字符（-）。

你可以选择性地添加一个描述（description），它会显示在仓库首页上。

还可以选择将仓库设为公开（public）或私有（private）。公开仓库任何人都能查看和下载，而私有仓库只有你和你授权的人可以访问。

如果不选择模板，GitHub 会提供自动生成一些常见文件的选项：例如 README 文件会出现在仓库首页，便于为项目提供说明；`.gitignore` 文件可以列出 Git 应忽略的文件（如依赖、环境文件等）；授权许可证（license）是法律文件，用来说明谁可以使用你的软件以及可以做什么。

当所有设置完成后，点击 “Create Repository” 按钮就能创建仓库，并自动跳转到新仓库的首页。

在仓库页面点击 “Code” 按钮，可以看到将仓库克隆（clone）到本地的选项。

本课不需要关注 “Codespaces” 与 “Copilot” 标签，我们关注 “Local” 选项卡中的三种方式。前两种（HTTPS 和 SSH）的使用方法一致：在终端中使用 `git clone` 命令并填入提供的 URL 即可克隆远程仓库。区别在于连接与认证方式不同。

对于克隆公开仓库，你通常不需要认证；但克隆私有仓库或向仓库推送（push）时都需要认证。HTTPS 方式通过 GitHub 用户名和访问令牌（token）进行认证（以前使用密码，但出于安全考虑现在改用令牌）；相较之下，SSH 使用公私钥对来认证，既更安全，又能免去每次输入令牌的麻烦。关于 SSH 的更深入内容会在单独课程中讲解，你也可以参考 GitHub 的官方文档了解如何配置 SSH。

克隆仓库的命令示例：

```sh
git clone git@github.com:your-username/your-repository-name
```

该命令会在本地创建一个以仓库名命名的文件夹，包含仓库中的所有文件。随后可以通过：

```sh
cd name-of-cloned-repo
```

进入克隆下来的仓库目录。

本地副本会包含一个指向远程仓库的 `remote`，你可以使用以下命令查看：

```sh
git remote -v
```

另一种克隆方式是使用 GitHub CLI（命令行工具），它可以让你在终端中完成更多 GitHub 专属操作。如果还未安装，可参考 GitHub 文档通过系统包管理器安装。

使用 CLI 克隆仓库的命令示例：

```sh
gh repo clone your-username/your-repository-name
```

以上命令的最终效果一样：在本地获得仓库的副本。

你还可以使用 GitHub CLI 创建仓库：

```sh
gh repo create
```

如果不传入参数，CLI 会进入交互式向导，允许你选择与网页 UI 相同的选项，并可选择在创建后立即克隆该仓库。

现在你已对仓库的作用与创建方式有了基本了解，后续课程会讲解如何把本地已有仓库推送到 GitHub 以及如何向远程仓库推送更改。
