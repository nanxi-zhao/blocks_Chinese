# --description--

在上一节课你学会了如何在 GitHub 上创建新仓库。那么如果你本地已经有一个项目，想把它添加到 GitHub 呢？本节将介绍如何把本地仓库推送到 GitHub，并从远程仓库拉取（pull）更新。

首先在 GitHub 上创建一个不含任何文件的新仓库。创建完成后，你会被重定向到一个页面，页面上会显示用于克隆（clone）仓库的 URL。

然后在终端进入你想要上传到 GitHub 的本地项目目录。本示例中，我们使用名为 `super-awesome-game` 的本地项目，其目录下包含：

```sh
jessicawilkins super-awesome-game >> ls
README.md  index.html script.js  styles.css
```

要为该项目启用版本控制，需要在项目目录运行 `git init`。该命令会初始化一个空的 Git 仓库，使 Git 开始跟踪该项目的更改。成功执行后，终端会显示类似如下的信息：

```sh
jessicawilkins super-awesome-game >> git init
Initialized empty Git repository in /Users/jessicawilkins/workspace/MasterPuti/super-awesome-game/.git/
```

初始化仓库后，会在项目中新增一个名为 `.git` 的隐藏目录。你可以用 `ls -a` 查看隐藏文件和目录：

```sh
jessicawilkins super-awesome-game >> ls -a
.          ..         .git       README.md  index.html script.js  styles.css
```

切勿删除 `.git` 目录，否则会丢失该项目的所有 Git 历史记录。

初始化仓库后，可以使用 `git status` 检查工作目录当前状态——这是日常工作流中经常要运行的命令。你会看到类似如下的输出：

```sh
jessicawilkins super-awesome-game >> git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
    README.md
    index.html
    script.js
    styles.css

nothing added to commit but untracked files present (use "git add" to track)
```

`On branch main` 表示当前处于 `main` 分支。分支是 Git 中的独立工作线，`main` 通常代表主分支或生产分支。开发团队常在分支上开发新功能或修复 bug，然后再将变更合并回 `main`。

`Untracked files` 表示这些文件尚未被 Git 跟踪。

文件通常会处于五种状态之一：

1. "Untracked"（未跟踪）：表示该文件对仓库是新的，Git 以前未见过；
2. "Modified"（已修改）：该文件在上一次提交中存在，但自那之后有未提交的修改；
3. "Ignored"（被忽略）：被 `.gitignore` 列出的文件通常会被排除在 Git 操作之外；
4. "Deleted"（已删除）：曾存在但已被移除的文件；
5. "Renamed"（已重命名）：内容未变但名称或路径发生更改的文件。

当有未提交的更改时，需要先把它们"暂存"（stage）到暂存区（staging area），暂存区是 `.git` 目录中的一个特殊结构，保存将被提交的信息。提交（commit）就是仓库在某一时刻的"快照"。

单个暂存文件的命令：

```sh
git add name-of-file
```

若要暂存所有更改，可以使用 `git add .`（`.` 代表当前目录）。有些开发者不建议随意使用 `git add .`，以免意外暂存不想提交的文件；如果发生此类情况，可用 `git reset name-of-file` 来取消暂存。

在暂存后，使用 `git commit` 提交更改。默认会打开你在 Git 配置中设置的编辑器，让你输入详细的提交说明；也可以用 `-m` 提供简短信息，例如：

```sh
git commit -m "short message goes here"
```

建议仍尽量写清晰的提交信息，例如：

```sh
git commit -m "feat(api): implement JWT-based authentication"
git commit -m "fix(auth): correct token expiration time"
git commit -m "refactor: simplify login flow logic"
```

`feat(api)` 和 `fix(auth)` 属于 Conventional Commits 规范，有助于描述提交类型。

提交完成后，再次运行 `git status` 应显示：

```sh
On branch main
nothing to commit, working tree clean
```

要查看历史提交可用 `git log`，该命令会列出每次提交的作者、时间、提交信息和 commit hash（用于唯一标识提交的长字符串）。

提交并准备好后，需要把更改推送到 GitHub。首先要设置远程仓库地址（remote）：

```sh
git remote add origin git@github.com:your-github-username/name-of-repo.git
```

然后执行推送：

```sh
git push -u origin main
```

`-u`（或 `--set-upstream`）会把本地 `main` 分支与远程 `origin/main` 关联，以后可以直接使用 `git push` 推送。

推送成功后，终端会显示类似信息：

```sh
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Delta compression using up to 8 threads
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 253 bytes | 253.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
To https://github.com/jdwilkin4/super-awesome-game.git
 * [new branch]      main -> main
branch 'main' set up to track 'origin/main'.
```

刷新 GitHub 仓库页面，你应该能看到刚刚推送的变更。

与他人协作时，可用 `git pull` 将远程变更拉取到本地。有关团队协作的更多内容会在后续课程介绍。
