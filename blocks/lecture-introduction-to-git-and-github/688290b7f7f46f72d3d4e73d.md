# --description--

你已经学会了如何创建提交并推送到自己控制的仓库。对别人维护的项目也可以做类似的贡献，但需要多一些步骤。

首先，你通常不会对别人的仓库拥有"写入权限"（write access），这意味着你无法直接向其推送（push）提交。相反，你需要先创建该仓库的一个副本来进行开发，这个过程称为"派生（fork）"。

要派生一个仓库，打开你想贡献的仓库页面，点击页面顶部的 "Fork" 按钮，会进入创建 fork 的界面。

你会看到与创建自己仓库相似的界面，但选项更少。可以选择所有者（个人账户或你有权限的组织）、为 fork 指定新名称以及添加描述。

你可以选择只复制 `main` 分支（或仓库配置的默认分支），通常这样做是个好主意，除非你需要对特定的功能分支进行贡献。关于分支的更多内容会在后续课程讲解，当前保持默认即可。

点击 "Create fork" 后，系统会为你创建 fork 并自动跳转到新的仓库页面。

接着像处理你自己的仓库一样，将其克隆到本地（使用 SSH 或 GitHub CLI）。

现在检查本地克隆配置的远程（remotes）：在仓库目录运行 `git remote -v`，你会看到类似：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
```

从 GitHub 克隆时，Git 会默认把你的 fork 配置为 `origin` 远程。但当你基于 fork 开发时，常常需要把原始仓库的更新合并进来。

为此，可以把原仓库添加为另一个远程。回到你派生的原仓库页面，点击 "Code" 获取 SSH URL，然后运行：

```sh
git remote add upstream git@github.com:original-repo-username/repo-name.git
```

`upstream` 是通常采用的命名约定（你也可以用其他名字）。再次查看远程列表，你会看到新添加的 upstream：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
upstream git@github.com:original-repo-username/your-repo-name.git (fetch)
upstream git@github.com:original-repo-username/your-repo-name.git (pull)
```

有了 `upstream`，你就可以运行 `git fetch upstream` 和 `git merge upstream/main`，把原始项目的最新更改拉入你的本地 fork。

fork 本质上仍是一个仓库，提交和推送的流程与其他仓库相同。但在向他人项目贡献前，应该遵循一定的策略（如分支管理、清晰的提交信息等），这些内容会在后续课程中详细讲解。

将源码公开并接受社区贡献的项目称为"开源"项目。开源的好处很多，例如社区能帮忙发现和修复 bug，学习别人对你项目的贡献，并构建有价值的开发者网络。

当你对他人项目做出贡献时，你就是一名"开源贡献者"。参与开源既能让你体验专业开发团队的协作流程，也能为你的职业发展积累经验。

如果不确定如何贡献，大多数开源项目都会在仓库中提供 CONTRIBUTING 指南；如果仍有疑问，也可以在仓库中打开 issue 提问，或加入项目在 Discord 等社区平台上的讨论渠道寻求帮助。
