# --interactive--

在 JavaScript 中，理解原始类型（primitive）与非原始类型（non-primitive）之间的区别，对于编写高效且少出错的代码非常重要。

这两类类型在内存存储和操作方式上表现不同。

原始类型是最简单的数据形式，包括数字（Number）、字符串（String）、布尔值（Boolean）、`null`、`undefined` 与 `Symbol`。它们被称为“原始”的原因是它们表示单一值，且不是对象。

使用原始类型时，你直接操作的是值本身。例如，将原始值赋给变量时，该值会被复制并存储在变量中。

原始值是不可变的（immutable），一旦创建其值不可就地修改，但你可以把变量重新赋值为另一个值。示例：

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

在这个示例中，我们把 `num1` 的原始值 `5` 赋给 `num2`，这会创建一个独立的副本。因此修改 `num1` 不会影响到 `num2`。

非原始类型则更复杂，主要指对象类型（包括普通对象、数组和函数等）。与原始类型不同，非原始类型可以包含多个值作为属性或元素。

当你把对象赋值给变量时，变量中存储的是指向对象的引用（reference），而不是对象本身。这带来重要的行为差异。示例如下：

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

在这个示例中，`originalPerson` 和 `copiedPerson` 引用的是同一个对象，因此通过一个引用修改对象会在所有引用中反映出来。

在 JavaScript 中，将对象赋值给另一个变量会复制对象的引用而不是对象本身，这被称为按引用浅拷贝（shallow copy by reference）。因此通过任何引用对对象所做的修改都会影响到所有引用该对象的变量。

在今后的项目中，你会遇到许多需要区分这两类类型的场景，理解它们的区别是掌握 JavaScript 的基础。继续练习，很快你就会熟练掌握这些概念！
