# --interactive--

在 JavaScript 中，函数和对象方法都是封装可复用代码的手段，但它们在定义方式、使用场景和运行上下文上有一些关键差别。理解这些差异有助于写出结构化且高效的代码。

正如你在之前的模块中学到的，函数是执行特定任务的可复用代码块：

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

而对象方法是与对象关联的函数，以对象的属性形式定义，并能访问和操作该对象的数据。下面是一个包含方法的对象示例：

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

在这个示例中，`sayHello` 是 `person` 对象的方法。方法内部可以使用 `this` 关键字访问对象的属性。你将在后续课程中更深入地学习 `this` 的含义与用法。

函数与方法的一个区别在于调用方式：函数直接按名调用，例如 `greet("Alice")`；而对象方法则通过对象调用，例如 `person.sayHello()`。

另一个重要区别是运行上下文。普通函数有自己的作用域，但并不自动绑定到某个对象；方法则与所属对象相关联，可以通过 `this` 访问对象的属性和其它方法。

方法有助于将代码按对象逻辑组织起来，而函数更适合实现通用且独立的功能。

理解函数与方法之间的差别会让你的代码更有条理，也更易于维护。继续加油，练习会让你越来越熟悉这些概念！
