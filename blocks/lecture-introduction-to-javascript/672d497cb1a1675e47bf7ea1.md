# --interactive--

在 JavaScript 中，变量就像存放数据的容器，你可以在程序中的任何地方访问和修改它们。

可以把变量想象成装有值的盒子。使用变量，你可以保存数字或文本等值，并在程序需要时引用这些值。

声明变量的一种方式是使用 `let` 关键字。我们会在后续课程中学习更多关于 `let` 以及其他声明变量的方法。

下面是使用 `let` 声明名为 `age` 的变量的示例：

```js
let age;
```

此时 `age` 变量还没有被赋值。如果尝试使用它，会得到 `undefined`，表示没有值。

示例说明：

**说明**：`console.log()` 是一个将信息输出到控制台（浏览器用于调试代码的部分）的函数。你将在后续课程中详细学习 `console.log()`。另外，`//` 符号用于在代码中添加注释，注释在运行时会被忽略，仅供开发者参考。

:::interactive_editor

```js
let age;
console.log(age); // undefined
```

:::

要给变量赋值，需要使用赋值运算符，例如：

```js
let age = 25;
```

现在使用 `age` 变量就会得到数值 `25`。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

赋值运算符看起来像等号（`=`），但它不是用于比较是否相等。你将在后续课程中学习用于比较相等性的运算符。

赋值操作就是把一个值放入变量，这个过程称为初始化。

使用 `let` 声明变量的一个好处是可以重新赋值（reassign）。重新赋值就是给已存在变量分配一个新值。

下面是重新赋值 `age` 的示例：

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

现在 `age` 变量的值是 `30`。注意不需要再次使用 `let`，因为变量已被声明。

在重新赋值时，只需使用变量名即可。重新赋值在程序运行过程中更新变量的值，例如在游戏中更新分数就是常见用例。

变量命名看似简单，但有一些规则和最佳实践可以提高代码的可读性和可维护性。

变量名应描述数据代表的含义。例如，不要用 `x` 这种难以理解的名称，使用 `age` 或 `points` 之类更具描述性的名字会让代码更易懂。

```js
// 不好的变量名
let x = 10;
let y = "John";

// 好的变量名
let age = 10;
```

在 JavaScript 中，变量名必须以字母、下划线（`_`）或美元符号（`$`）开头，不能以数字开头。

```js
// 有效的变量名
let age;
let _score;
let $total;

// 无效的变量名
let 1stPlace; // 以数字开头
```

变量名区分大小写，这意味着 `age` 与 `Age` 是两个不同的变量。

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

因此坚持使用一致的命名规范非常重要，例如驼峰式（camelCase）：第一个单词小写，其后每个单词首字母大写。

下面是使用 camelCase 的示例：

```js
let thisIsCamelCase;
let anotherExampleVariable;
let MasterPutiStudents;
```

某些 JavaScript 关键字不可用作变量名，例如 `let`、`const`、`function` 或 `return`，因为它们是语言的保留词。

还应避免在变量名中使用感叹号（`!`）、艾特符号（`@`）等特殊字符。最好使用字母、数字、下划线或美元符号来保持变量名的可读性。

遵循这些准则可以让你的代码在复杂性增加时依然保持清晰易维护。
