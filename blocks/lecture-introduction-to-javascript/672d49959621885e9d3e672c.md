# --interactive--

在使用 JavaScript 时，你经常需要声明变量来保存将贯穿整个程序使用的数据。

在现代 JavaScript 中，`let` 和 `const` 是推荐用于声明变量的方式，它们在赋值与重新赋值上的行为不同。

本节将介绍 `let` 和 `const` 在声明、赋值与重赋值时的区别。

`let` 关键字允许你声明可以在以后更新或重新赋值的变量。可以把 `let` 看作是一个灵活的容器——一旦存入一个值，你可以在程序运行过程中按需修改它。

下面是使用 `let` 声明并赋值的示例：

```js
let score = 10;
```

在这个例子中，变量 `score` 被赋值为 `10`，如果需要，你可以随后更新它的值：

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

现在 `score` 的值是 `20`。当你预期变量的值会随程序运行而改变时，`let` 非常有用。

另一方面，`const` 用于声明常量。一旦给 `const` 声明的变量赋值，就不能再次重新赋值。

这使得 `const` 非常适合用于那些你不希望在程序执行过程中被意外更改的值。

下面是使用 `const` 声明并赋值的示例：

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

一旦 `maxScore` 被赋值为 `100`，它就不能被更改：

```js
maxScore = 200; // 这会导致错误
```

尝试重新赋值 `const` 变量会在控制台抛出错误，因为 `const` 在赋值后不可重新赋值。

你可以声明一个 `let` 变量而不立即赋值，然后在后面赋值：

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

使用 `const` 声明的变量必须在声明时就被赋值；如果声明 `const` 时没有赋值，会出现错误：

```js
const age; // Error: Missing initializer in const declaration
```

当你需要声明会被重新赋值的变量（例如跟踪不断变化的分数）时，使用 `let`；当你希望值保持不变（例如配置项）时，使用 `const`。

历史上还有 `var` 关键字可以使用，但现在不太推荐，因为 `var` 的作用域更宽，容易在程序中引发意外问题。
