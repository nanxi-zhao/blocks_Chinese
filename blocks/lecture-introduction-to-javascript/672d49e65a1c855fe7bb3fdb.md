# --description--
注释是代码中的行或块，在代码执行时会被 JavaScript 引擎忽略。它们仅供阅读代码的人使用，不会影响程序运行。

下面哪种写法可以在 JavaScript 中正确创建单行注释？

### --feedback--

记住 JavaScript 单行注释使用哪个符号。

1. 其他开发者可以理解这段代码的用途。
2. 它有助于在做出修改或删除之前提供上下文，避免未经沟通造成的 Bug 或问题。

另一种注释是多行注释，语法如下：

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

当需要写较长的说明或注释时，多行注释非常有用。

下面是 MasterPuti 项目中多行注释的实际示例：

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

同样地，不必完全理解这些 JavaScript 代码，重点是阅读注释所提供的上下文信息。

团队中的开发者或新贡献者可以通过注释快速理解该代码段的目的，从而在修改前获得完整的背景信息。

尽管注释很有用，但也要避免过度注释。若代码本身已经清晰易懂，就没有必要对每一行都添加注释。

下面是用注释解释显而易见内容的示例：

```js
// 该代码使用 const 关键字声明一个名为 price 的变量。
// 我们将数字 10 赋值给 price 变量。
const price = 10;
```

在这种情况下没有必要添加注释，因为代码已经一目了然。注释的目的是增强可读性，而不是让代码变得冗杂。

在学习阶段，在个人项目中添加注释是可以的；但在与其他开发者协作的真实项目中，不应对显而易见的代码过度注释。

也不要用注释去掩盖混乱、过于复杂或糟糕的代码。遇到这种情况，最好对代码进行重构，从根本上让代码更清晰。

注释是记录代码和提高可读性的重要工具，应在需要时用于提供上下文或给自己和他人留下说明。
