# --description--

在 Python 中，变量就像贴了标签的盒子，用来存放并引用不同类型的数据。声明变量时，只需使用赋值运算符（`=`）把一个值赋给一个标识符；不像 JavaScript 需要 `let` 或 `const`，也不用像 C# 那样指定 `char`。

在 Python 中，你在左侧写变量名，然后是赋值号，再写要赋给变量的值。下面是声明 `name` 和 `age` 的示例：

```python
name = 'John Doe'
age = 25
```

上例中，变量 `name` 保存了字符串 `'John Doe'`。字符串用单引号或双引号表示，例如 `'Hello'` 或 `"Hello"`。后续课程会介绍更多关于字符串的操作。

在为变量命名时，有一些重要的规则需要遵守：

- 变量名只能以字母或下划线（`_`）开头，不能以数字开头。
- 变量名只能包含字母、数字和下划线（`a-z`、`A-Z`、`0-9`、`_`）。
- 变量名区分大小写，例如 `age`、`Age` 和 `AGE` 被视为不同的变量。
- 变量名不能是 Python 的保留关键字，例如 `if`、`class` 或 `def`。

如果违反上述规则，Python 程序会抛出 `SyntaxError`：

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

下面介绍一些常见的命名约定。

首先，变量名通常使用小写字母，多个单词之间用下划线分隔，这被称为 snake_case：

```python
my_variable_name = 'MasterPuti'
```

另外，建议使用有描述性的变量名。例如，要保存用户年龄时，`user_age` 比单纯的 `age` 或缩写 `ua` 更清晰：

```python
user_age = 30
```

在大型代码库中，清晰的变量名能更好地向团队成员（或将来的你）传达变量的用途。

尽量避免使用单字母变量名，因为单字母通常无法表达变量的意义：

```python
x = 56 # 这里的 x 表示什么？
```

在循环等场景中，像 `i`、`j`、`k` 这样的短变量名是常见且可以接受的。

顺便介绍一下注释。井号（`#`）后面的文本是注释，Python 会忽略该行中 `#` 后的内容：

```python
# This is a single-line comment
```

要写多行注释，可用连续的单行注释：

```python
# This is a
# multi-line
# comment
```

注释可以用来解释代码、给自己留备忘或说明某行代码的用意，尤其在学习或团队协作时非常有帮助。

不过，不要用注释来替代有意义的变量名——好名字本身就能说明用途，同时遵循前面提到的命名规则能避免语法错误。
