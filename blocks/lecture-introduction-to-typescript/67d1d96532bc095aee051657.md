# --description--

JavaScript 被视为一种动态类型语言。这意味着变量在运行时可以接收任意类型的值——把变量当作数字声明并不阻止你以后把字符串赋给它，函数参数也可以传入任何值。

动态类型语言的挑战在于，缺少类型安全性可能会在不经意间引入错误。TypeScript 在 JavaScript 的基础上加入了静态类型检查，这有助于在代码部署前捕捉这类错误。那么这是如何工作的呢？

来看一个 JavaScript 的例子：

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

上面的 `getRandomValue` 函数接受一个数组并从中返回一个随机值。但由于 JavaScript 不会在运行时校验类型，没有东西能阻止你传入一个数字：

```js
console.log(getRandomValue(10));
```

在当前示例中，控制台输出会是 `undefined`，因为函数期望的是数组而不是数字。

而在 TypeScript 中，你可以为数组参数定义类型：

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

这个类型定义告诉 TypeScript，`array` 参数必须是字符串数组。如果你调用 `getRandomValue` 时传入数字，TypeScript 会在编译阶段抛出一种不同于运行时错误的编译器错误（compiler error）。

大多数常见的 JavaScript 运行环境（例如浏览器或 Node.js）并不能直接运行 TypeScript。你需要先用 TypeScript 的编译器把 TypeScript 代码转换成普通的 JavaScript。编译器会在运行前检查类型不匹配的问题——例如把数字传给期望数组的函数——并报告错误。

后续课程会详细讲解类型如何工作。但正是这种额外的类型安全让 TypeScript 对许多开发者和组织来说更具吸引力。
