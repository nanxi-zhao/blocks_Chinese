# --description--

在软件中，依赖（dependency）指的是一个组件或模块需要另一个组件才能正常工作。依赖在软件项目中很常见，因为开发者可以复用别人写好的功能或工具，省时又省力。

如果你要构建一个 React 应用，就需要安装 React 的依赖包。没有这些依赖，你的代码可能无法正常运行，应用会报出一堆错误——不过别担心，我们可以一步步来安装它们。

在复杂项目中，你通常会依赖很多第三方包。如果这些依赖没有被良好管理或明确记录，就会出现所谓的“依赖地狱”（dependency hell）。

为了解决依赖管理问题，我们使用包管理器（package manager）。包管理器负责安装、更新和移除依赖。像 JavaScript、Python、Ruby、Java 等热门语言都有自己的包管理器。

前面的课程中你已经简单接触过一个常用的包管理器：`npm`。

使用 Vite 和 npm 创建一个新的 React 项目可以运行如下命令：

```sh
npm create vite@latest my-react-app -- --template react
```

如你所见，这个命令会生成启动 React 应用所需的模板代码。在启动之前，你还需要运行 `npm install`（或 `npm i`）来安装项目依赖。

项目根目录下的 `package.json` 文件会列出所有依赖项，你可以在其中查看当前使用的包和版本。

`package.json` 是项目的重要配置文件，包含项目元数据（比如名称、版本、依赖等），还定义了脚本、许可信息和其他有助于管理项目的设置。

安装依赖后，项目中会生成一个 `node_modules` 文件夹。

`node_modules` 用来存放项目所需的所有包和库，其中包含 `package.json` 中列出的直接依赖，以及这些依赖所依赖的其他包的代码。

对于 React 项目，通常至少需要 `react` 和 `react-dom` 这两个核心依赖：

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

`package.json` 会列出这些包当前使用的版本。如果你想在本地更新某个包，可以运行 `npm update`；若要全局更新包，可以运行 `npm update -g`。

除了 `package.json`，项目中还会有一个 `package-lock.json` 文件，用来锁定项目所用的各个包的精确版本。更新包时，锁文件也会随之更新。

`package.json` 的另一个重要部分是开发依赖（devDependencies）：

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

开发依赖是只在开发过程中使用、不会部署到生产环境的包。例如测试库 Jest 就通常作为开发依赖安装，因为它用于本地测试，而不是运行生产环境的应用。

本课主要聚焦于 npm，但还有其他包管理器，例如 `yarn` 和 `pnpm`。那应该选择哪个呢？简短的回答是：视情况而定。

如果你加入一个已有团队的项目，包管理器的选择通常由团队决定，你只需遵守现有配置；如果从零开始搭建项目，则需要调研各种包管理器的优缺点，选择最适合你需求的方案。
