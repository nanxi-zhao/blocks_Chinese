# --description--

到目前为止，你一直在用 React 构建用户界面。如果需要路由等附加功能，你需要引入第三方库（比如 React Router）来在不同视图间切换。

但如果要构建完整的全栈 Web 应用呢？你可以只用 JavaScript：前端用 React，后端用 Node 与 Express；也可以在后端使用 Go、Python、Java 等语言。

尽管这些都是可行方案，但有时使用 React 框架会更高效。React 框架通常内置路由、图片优化、数据获取、认证等功能，这意味着在某些场景下你不必单独搭建前后端应用。

我们来看看流行的 React 框架 Next.js。Next.js 的一个主要特性是基于文件系统的路由，它支持动态路由、并行路由、路由处理器、重定向、国际化等功能。

下面是创建自定义请求处理器的示例：

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

像 GET 或 POST 这样的路由处理器可以放在 `app/api` 目录下的 `route.js` 文件中进行定义。

另一个常见特性是 Next.js 提供自动的图片和字体优化。

以下是在 `page.js` 中使用 `Image` 组件的示例：

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

`Image` 组件扩展了原生的 HTML `img` 元素，能带来更快的页面加载和尺寸优化。也就是说图片只有在进入视口时才会加载，且 `Image` 会根据设备自动提供合适尺寸的图片。

上面只是 Next.js 的部分功能，实际上还有很多特性可以帮助你构建稳健的全栈应用。类似的 React 框架还有 Remix 等，它们也能协助你构建现代的全栈 Web 应用。

虽然 JavaScript 的库与框架生态不断演进，了解可用选项并权衡各框架的优缺点，对选择最合适的工具非常有帮助。
