# --interactive--

你之前学习了语义标记和 ARIA 属性如何使内容可访问。在现实世界的应用程序中，内容很少是静态的，页面经常通过 JavaScript 动态更新。当这种情况发生时，重要的是确保这些更改也反映在 HTML 中。这允许屏幕阅读器准确地向用户传达更新的状态。如果没有这个，依赖辅助工具的人可能永远不会知道某些东西发生了变化，或者可能被给予过时或误导性的信息。

让我们看一个常见场景，其中 JavaScript 改变元素的状态，以及你如何通过更新 ARIA 属性来保持 HTML 的可访问性。

假设你有一个打开和关闭下拉菜单的按钮：

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  菜单
</button>

<ul id="menuList" hidden>
  <li>列表项 1</li>
  <li>列表项 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

HTML 设置了一个控制下拉菜单的按钮。按钮使用 `aria-expanded` 来指示菜单是否打开，并使用 `aria-controls` 通过引用菜单的 `id` 将其链接到菜单。

JavaScript 监听菜单按钮上的点击。当点击时，它切换 `aria-expanded` 的值并更新菜单上的 `hidden` 属性。这保持了菜单的可见性和可访问性信息同步，因此视力用户和屏幕阅读器用户都知道菜单是打开还是关闭。

你将在接下来的课程中了解更多关于这些 ARIA 属性的信息。现在，只需记住当你的 JavaScript 改变元素的状态时，你也应该更新相关的 ARIA 属性，以便辅助技术准确地反映屏幕上实际的内容。
