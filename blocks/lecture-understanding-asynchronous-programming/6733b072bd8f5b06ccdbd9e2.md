# --description--

让我们来学习异步JavaScript。异步事件在不同时间发生，彼此独立。

在软件开发的背景下，"异步"指的是在后台运行的任务，独立于程序的主要流程。异步进程的主要优势是它们不会阻塞主程序的执行。

这对于可能需要很长时间运行的任务特别有帮助，比如从远程服务器获取数据、处理大文件、处理用户输入和执行复杂计算。这就是我们所知的异步编程。

这种方法与你迄今为止一直在使用的传统同步编程技术形成对比。当你编写多行代码时，通常可以预测会发生什么以及何时发生。第一行将被执行，然后是第二行，依此类推，按照你编写的顺序执行。

在这个例子中，定义变量的第一行将首先执行，然后是第二行，最后是第三行，字符串被记录到控制台：

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

一切都是顺序和可预测的。每一行都在下一行开始之前完成。这种类型的JavaScript程序被称为单线程。

线程的概念非常重要。线程是进程内的执行单元。它就像程序内的独立控制流。

在同步编程中，线程按顺序执行，一个接一个。如果一个线程被阻塞，比如等待用户输入，整个进程会被阻塞直到线程完成。

在异步编程中，线程可以并发执行，同时运行多个线程。这样，程序可以同时运行多个任务，即使其中一个线程被阻塞，也不会使主程序无响应。

异步编程通常涉及回调、承诺或async/await来处理非阻塞操作。

正如你在之前的课程中学到的，回调是你传递给另一个函数以便稍后调用的函数。事件处理程序是你之前使用过的一种特定类型的回调。它们曾经是JavaScript中实现异步函数的最常见方式。然而，如果回调函数还需要其他回调函数，代码和逻辑可能会很快变得相当复杂。

目前，JavaScript中实现异步编程最常用的技术是承诺。`Promise`是一个对象，代表异步进程的最终完成(或失败)及其值。

承诺的值在创建承诺时是未知的。只有当异步进程完成时才知道。例如，为你的Web应用程序从远程API获取数据的过程可能需要一些时间。同时，你希望你的用户有一个良好的用户体验，对吧？

为了实现这一点，你可以创建一个承诺，在异步进程运行时保持用户界面活跃和交互。当进程完成时，承诺将包含API发送回来的数据，所以应用程序可以在数据可用时适当地处理或渲染它。

异步编程是构建高效JavaScript应用程序的强大工具。通过理解同步和异步编程之间的差异，你可以为你的应用程序选择正确的方法并编写更高效的代码。
