# --description--

HTML `script`元素中的`async`和`defer`属性在JavaScript文件如何在网页中加载和执行方面起着至关重要的作用。理解它们可以提高你网站的性能和用户体验。当你在HTML文件中包含一个`script`时，它看起来像这样：

```js
<script src="example.js"></script>
```

当浏览器找到这个`script`标签时，它会停止解析HTML，下载脚本，执行它，然后继续解析HTML。这可能会减慢你网页的加载速度，特别是如果你有大型脚本。

这就是`async`和`defer`属性的用武之地。它们提供了更高效地加载脚本的方法。

让我们从`async`属性开始：

```js
<script src="example.js" async></script>
```

通过向`script`标签添加`async`属性，浏览器在下载脚本时会继续解析HTML。一旦脚本完全下载，浏览器将暂停HTML解析，执行脚本，然后恢复解析HTML。这可以显著加快页面加载速度。

需要注意的是，异步脚本在下载完成后立即执行，这意味着它们可能不会按照我们期望的正确顺序运行。这就是`defer`属性来救援的地方。让我们看看`defer`属性的样子：

```js
<script src="example.js" defer></script>
```

`defer`属性与`async`属性类似。然而，延迟脚本在下载完成后不会立即执行。相反，它们会等到HTML解析完成。此外，延迟脚本按照它们在HTML代码中出现的顺序执行。

简而言之，对于执行顺序不重要的脚本使用`async`，当你需要确保脚本按正确顺序运行时使用`defer`。这两个属性都可以通过允许浏览器在后台下载脚本时继续解析HTML来显著改善页面加载时间。
