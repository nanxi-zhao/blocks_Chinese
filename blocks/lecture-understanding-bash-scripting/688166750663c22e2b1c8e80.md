# --description--

在之前的课程中，你学习了Bash和一些常用的命令。但是Bash支持完整的脚本语言，你可以用它来执行更复杂的自动化任务哦！

Bash脚本编写涉及将一系列Bash命令写入文件中，然后可以用Bash执行该文件来运行文件内容。

让我们来看看bash脚本可能是什么样子的：

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

这个脚本从远程服务器列表中拉取NGINX配置文件，这对于将这些配置备份到本地仓库非常有用呢。NGINX是一个流行的Web服务器，在未来的模块中你会学到更多关于服务器操作的知识哦！

第一行被称为"shebang"（`#!`），它告诉系统应该使用哪个解释器来运行脚本。常见的值有`/usr/bin/bash`、`/bin/bash`或`/bin/sh`。

第一行`servers=("prod" "dev")`实例化了一个表示服务器名称的字符串列表。"prod"通常用来指代生产站点，而"dev"指代开发站点，开发站点通常用于在部署到实时生产环境之前测试新功能和更改。

然后，通过`for`循环遍历列表 - `servers[@]`语法将列表扩展为单个元素。每次迭代时，每个元素都被赋值给server变量。

`do`语句表示循环逻辑块的开始，也就是每次迭代应该运行的代码。

`echo "Pulling $server"`插入当前server变量的值，在这种情况下是prod或dev，并将其记录到终端。

然后使用`rsync`从服务器拉取实际文件并保存在本地，使用更多的插值来管理该命令。

`done`表示循环逻辑块的结束。

理解和利用Bash脚本有很多重要的优势呢！

第一个优势是几乎每个Unix环境都预装了Bash，这意味着你可以直接运行脚本，而无需配置Node.js或Python等语言运行时。

第二个优势是你能够轻松访问系统上所有可用的二进制应用程序。如果你的系统安装了doctl，你的脚本就可以执行那个二进制文件。二进制文件是计算机运行的编译程序。当你有了bash脚本，你就可以执行二进制文件来启动程序。

最后，你用于脚本的语法也可以直接在终端中执行。这意味着测试脚本的一小部分变得简单多了 - 你可以直接将那部分复制粘贴到终端中查看结果！

理解Bash不仅能提升你的脚本编写技能，还会让你成为命令行魔法师呢！
