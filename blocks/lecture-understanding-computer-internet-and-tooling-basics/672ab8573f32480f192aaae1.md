# --description--

专业开发人员依靠各种工具来提高生产力和代码质量。让我们来了解这些工具，包括那些看起来很明显的工具。

工具中的第一个是计算机。计算机是主要的开发环境。它可以是笨重的台式机，也可以是便携式笔记本电脑，操作系统可以是Windows、macOS或Linux。

专业开发人员通常选择处理能力强、内存充足的计算机来处理资源密集型任务。

计算机之后是用于在该计算机上编写和编辑代码的程序。那就是代码编辑器或集成开发环境（IDE）。

像Visual Studio、IntelliJ IDEA、JetBrains和PyCharm这样的IDE提供强大的功能，如代码补全、调试和集成终端支持。

Visual Studio Code（VS Code）本质上是一个代码编辑器，但它也通过其丰富的扩展库提供这些功能。

其他代码编辑器还有Sublime Text、Atom、Notepad++和Brackets。

当你使用代码编辑器和IDE编写代码时，你需要跟踪对它们所做的更改。让你能够跟踪这些更改的工具是版本控制系统。

Git是开发社区中使用最广泛的版本控制系统。

像GitHub、GitLab和Bitbucket这样的平台为你的Git仓库提供基于云的托管服务。这些平台使与其他开发人员协作成为可能，允许你在多个分支上工作，并促进代码更改的合并。

包管理器是另一个关键工具。它们帮助开发人员简化添加、更新和删除库及项目依赖项的过程。

流行的包管理器示例包括：

* JavaScript的NPM、Yarn和PNPM
* Python的PIP
* PHP的Composer
* Java的Maven

使用不同工具编写代码后，开发人员会测试这些代码以确保其按预期工作。

为此，开发人员使用Cypress、Playwright、Selenium等测试框架。还有特定于语言的测试框架。例如：

* JavaScript的Jest
* Python的pytest
* Java的JUnit
* PHP的PHPUnit

你不仅需要测试代码以确保其按预期工作。还需要测试代码在最终用户眼中看起来如何。开发人员为此使用网络浏览器。

像Chrome、Firefox、Edge和Safari这样的现代浏览器提供开发者工具来检查HTML、CSS和JavaScript代码。还有用于调试和性能分析的工具。

这些工具帮助开发人员在不同浏览器上测试和优化他们的Web应用程序，以服务最终用户。
