# --description--

让我们学习事件中`preventDefault()`方法的目的。

在DOM中触发的每个事件都有一些默认行为。复选框上的`click`事件默认会切换该复选框的状态。在聚焦的按钮上按空格键会激活按钮。这些事件对象上的`preventDefault()`方法会阻止这种行为发生。

让我们看一个例子。让我们定义一个供用户输入的`input`元素：

```html
<label>输入一些字符:
  <input type="text">
</label>
```

如果我们查看结果，可以按预期在输入字段中输入。但我们可能不希望这样。也许，相反，我们想在单独的元素中显示用户输入的字符。首先，让我们为这个定义我们的元素：

```html
<label>输入一些字符:
  <input type="text">
</label>
<p id="output"></p>
```

然后，我们需要挂钩到`keydown`事件来监听键盘上输入的字符。注意，我们在这里不想要`change`或`input`事件，因为我们需要键盘信息。

```javascript
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {

})
```

`keydown`事件在你按下键盘键时触发。当这种情况发生时，让我们在`p`元素中显示字符。

```javascript
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  output.innerText = `你按下了 ${e.key} 键`;
});
```

`e.key`给你按下的键的值，比如`a`键的`a`或`Enter`键的`Enter`。

使用上面的代码，当你在`input`中输入时，你输入的字符将显示在`p`元素中。

这很棒，但我们不想在`input`中也显示字符。这就是我们的`preventDefault()`方法派上用场的地方。`keydown`的默认行为是在输入中渲染字符。让我们通过调用`e.preventDefault()`来避免这种情况：

```javascript
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `你按下了 ${e.key} 键`;
});
```

就这样，你已经阻止了默认行为，允许自己实现自定义事件处理。

使用`e.preventDefault`方法的另一个常见例子与表单提交有关。默认情况下，提交表单会将数据发送到服务器并重新加载页面。使用`e.preventDefault()`可以防止这种情况发生。

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // 其余代码在这里
});
```

当你需要更多控制用户如何与页面交互时，阻止默认行为很棒，但重要的是要记住可访问性等事情——你的自定义行为应该提供与默认行为相同的特性。
