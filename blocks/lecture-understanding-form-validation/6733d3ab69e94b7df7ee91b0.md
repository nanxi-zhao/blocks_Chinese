# --description--

让我们学习HTML表单中`submit`事件是如何工作的。首先，我们需要了解如何提交表单。有三种方式可以提交表单。

第一种是当用户点击表单中`type`属性设置为`submit`的按钮时。

第二种是当用户在表单的任何可编辑输入字段中按Enter键时。

第三种是通过JavaScript调用表单元素的`requestSubmit()`或`submit()`方法。

但是当表单被提交时会发生什么呢？有几件事会发生，表单的行为取决于其属性。

我们需要看的第一个属性是`action`属性。`action`属性应该包含URL或当前域的相对路径。这个值决定了表单尝试发送数据的位置。如果你没有设置`action`属性，表单将把数据发送到当前页面的URL。以下是具有设置为特定URL的`action`属性的表单示例：

```html
<form action="https://MasterPuti.org">
  <input type="number" id="input" placeholder="输入一个数字" name="number" />
  <button type="submit">提交</button>
</form>
```

当这个表单被提交时，它会将数据发送到MasterPuti的主页，这可能不会做任何事情。

这是另一个提交到相对路径的表单：

```html
<form action="/data">
  <input type="number" id="input" placeholder="输入一个数字" name="number" />
  <button type="submit">提交</button>
</form>
```

这个表单提交到当前域的`/data`路径。在我们的情况下，它会提交到`http://127.0.0.1:5500/data`。

控制提交行为的第二个属性是`method`属性。这个属性接受标准HTTP方法，如`GET`或`POST`，并在向`action`URL发出请求时使用该方法。

HTTP代表超文本传输协议，用于在网络上传输数据。

HTTP方法用于定义可以在资源上执行的操作，如`GET`、`POST`、`PUT`、`DELETE`等。你将在未来的课程中学习更多关于这些方法的知识。

当没有设置方法时，表单将默认为`GET`请求。`GET`请求用于从指定资源检索数据而不对其进行任何更改，参数通常以查询字符串的形式附加到URL中。

表单中的数据将被URL编码为`name=value`对，并作为查询参数附加到`action`URL。例如，通过提交我们前面例子中的表单，输入字段的值为`3342`，表单会尝试`GET`URL`http://127.0.0.1:5500/data?number=3342`。

这对于搜索表单等场景很棒，用户在查询数据。但如果你想让用户提交新数据，比如在注册表单中，惯用的方法是使用`POST`方法。`POST`方法用于向服务器发送数据以创建或更新资源。

让我们将`method`属性设置为`POST`值：

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="输入一个数字" name="number" />
  <button type="submit">提交</button>
</form>
```

当你发送`POST`请求时，可以包含一个包含请求数据的主体。所以与`GET`不同，数据不会附加到URL上，我们的表单将`POST`请求发送到`http://127.0.0.1:5500/data`。相反，数据可以在请求的主体中找到，仍然是URL编码的表单数据。

URL编码的表单数据是指表单数据被转换为键值对字符串，如`name=John+Doe&email=john%40example.com`，其中特殊字符被替换为编码版本以安全地通过网络发送数据。你将在未来的课程中学习更多关于这方面的知识。

但也许你不想将数据作为URL编码的表单载荷发送？

`form`元素接受一个`enctype`属性，它表示用于数据的编码类型。

这个属性只接受三个值：`application/x-www-form-urlencoded`（这是默认值，将数据作为URL编码的表单主体发送）、`text/plain`（将数据以纯文本形式发送，以`name=value`对的形式，用换行符分隔），或`multipart/form-data`，这是专门用于处理包含文件上传的表单。

在这节课中，你学习了表单提交的基础知识。在未来的课程中，我们将更深入地学习如何使用表单以及它们如何与服务器交互。
