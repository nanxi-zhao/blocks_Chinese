# --description--

柯里化是一种将接受多个参数的函数转换为一系列接受单个参数的函数的技术。

让我们从一个基本示例开始。假设我们有一个将两个数字相加的函数：

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 输出: 7
```

这是一个接受两个参数并返回它们之和的函数。现在，让我们看看如何对这个函数进行柯里化：

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 输出: 7
```

在这个柯里化转换的代码中，我们不是一次性接受两个参数，而是有一个函数接受第一个参数并返回另一个函数。这个返回的函数然后接受第二个参数并执行加法。我们像这样调用它：`curriedAdd(3)(4)`，其中每对括号代表一次函数调用。

但是我们为什么要这样做呢？

柯里化让我们能够轻松创建一些特殊函数。例如，我们可以创建一个总是将五加到任何数字上的函数：

```js
const addFive = curriedAdd(5);
console.log(addFive(10)); // 输出: 15
console.log(addFive(20)); // 输出: 25
```

在这里，`addFive`是一个总是准备将五加到我们给它的任何数字上的函数。这是部分应用的一个简单示例，其中我们将一定数量的参数固定到一个函数上，产生另一个接受更少参数的函数。

虽然我们的示例专注于有两个参数的函数，但柯里化可以应用于具有任意数量参数的函数。

当你继续JavaScript之旅时，你可能会发现许多情况下柯里化可以让你的代码更干净、更具表现力。

## --text--

以下哪项是JavaScript中柯里化函数的示例？

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

这是一个一次性接受所有参数的常规函数。

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

这个函数一次性接受所有参数并将它们像函数一样调用，这不符合柯里化函数的定义。

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

内部函数从未被返回，所以`add(2)`将返回`undefined`，`add(2)(3)`将抛出错误。
