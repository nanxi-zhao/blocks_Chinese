# --interactive--

在现代JavaScript中，类就像蓝图一样，你可以在代码中定义它们来创建多种相同类型的对象。

例如，如果你正在建模一个库存系统，你需要为每种产品创建多个对象。

每个单独的产品都会有自己独特的属性和动作，但同一种产品的对象可以从相同的"蓝图"创建。这就是类的用途。

类可以定义为类表达式或类声明。

以下是类声明的基本语法：

```js
class MyClassName {
  // 类方法
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

你首先写[class](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)关键字，然后是类的名称。在命名类时，你应该使用[PascalCase](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@typescript-eslint/type-utils/dist/getTypeName.js#L45-L45)命名约定而不是camelCase。

花括号内的内容被称为类体。这里是构造函数和方法所在的地方。

构造函数是一个特殊的方法，当从类创建新对象时会自动调用。它通常用于初始化对象的属性。

在这个例子中，你可以看到一个带有构造函数的[Dog](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)类。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

这里的[this](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/usedPropTypes.js#L45-L45)关键字用于引用当前的[Dog](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)类实例，允许你将[name](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-working-with-strings-in-javascript/687b7400741c0e81bc060515.md#L22-L23)属性分配给正在创建的对象。你将在未来的课程中了解更多关于[this](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/usedPropTypes.js#L45-L45)关键字的知识。

在构造函数下面，你可以有所谓的方法。方法是在类中定义的函数，用于执行动作或操作。它们用于定义类实例可以执行的行为。

这里是向[Dog](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)类添加一个名为[bark](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)的方法的例子。

``js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

当[bark](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)方法被调用时，消息将被记录到控制台，其中包含传递给它的实际名称。如果你创建一个名为`"Buddy"`的[Dog](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)实例，调用该实例的[bark()](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)方法将在控制台打印`"Buddy says woof!"`。

一旦你定义了类，你可以使用[new](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)关键字、类的名称及其参数来创建类的实例。

在这个例子中，我们创建了一个名为`Gino`的[Dog](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)实例，并将其赋值给变量`dog`。

``js
const dog = new Dog("Gino");
```

当你在代码中写`new Dog("Gino")`时，会创建一个新对象，并自动调用构造函数，传入给定的参数。

然后你可以使用点符号访问这个属性，就像你在下面的代码行中看到的那样：

``js
console.log(dog.name);
```

以下是完整的例子：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

你也可以调用方法。在这个例子中，我们在`dog`实例上调用[bark()](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)方法。

``js
dog.bark();
```

以下是包含方法调用的完整例子：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

正如前面提到的，你也可以将类定义为类表达式。在这种情况下，类是匿名的，并被赋值给一个变量。

以下是前面例子作为类表达式的写法：

``js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

大多数时候，你会使用类声明，但理解类表达式也很有价值，因为它们可以在动态编程场景中提供更多的灵活性。

通过使用类并理解它们的基本语法，你可以创建可重用的代码，将其组织成逻辑单元，并促进代码模块化。
