# --interactive--

CSS中的浮动是一种最初设计用于允许文本环绕元素（如图像）的技术。然而，随着时间的推移，开发人员发现了使用浮动的新方法，以创造性的方式将它们应用于布局设计。虽然现在更常使用Flexbox和Grid等现代布局方法，但理解浮动仍然很重要，特别是在处理旧代码或需要实现特定布局效果时。

当一个元素被浮动时，它会从正常的文档流中取出，并被推到容器的左侧或右侧。接下来的内容将环绕浮动元素，填充剩余空间。一个经典的用例是文本环绕图像，其中图像被浮动到一侧，文本环绕它。这种技术仍然被广泛使用，特别是在文章和博客中，图像需要与文本并排放置。以下是代码示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> 这是文本环绕浮动图像的示例。</p>
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
  height: 200px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

在上面的例子中，容器没有环绕浮动图像。图像脱离了正常的文档流，容器坍塌为零高度，因为它"看不到"浮动的子元素。

在Flexbox和Grid普及之前，浮动也常用于创建多列布局。通过将元素彼此浮动，开发人员可以创建水平对齐的列。然而，使用浮动时，重要的是要处理子元素浮动时父元素坍塌的问题。clearfix技术解决方案应用于[container](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-how-to-work-with-floats-and-positioning-in-css/672aa86da9937560d3dfe3d6.md#L18-L21)类元素来解决这个问题。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> 这是文本环绕浮动图像的示例。</p>
</div>
```

```css
/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- [::after](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node-fetch/node_modules/@types/node/globals.d.ts#L177-L177)是一个伪元素，它在容器内容后添加一个不可见的块。
- [content: ""](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node-fetch/node_modules/@types/node/globals.d.ts#L131-L131)确保伪元素存在但不显示任何内容。
- [display: block](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)使伪元素成为块级元素。
- [clear: both](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)确保伪元素清除其上方任何浮动元素的两侧。

clearfix技术确保父元素正确地环绕其浮动子元素。Clearfix通过在浮动内容后添加清除属性来强制父容器"看到"浮动子元素。

虽然由于Flexbox和Grid等现代技术的出现，浮动不再是复杂布局的首选方法，但它们在某些场景中仍然发挥着重要作用。无论你是要让文本环绕图像还是处理遗留项目，理解浮动的工作原理以及如何正确清除它们都是维护结构良好和响应式布局的关键。
