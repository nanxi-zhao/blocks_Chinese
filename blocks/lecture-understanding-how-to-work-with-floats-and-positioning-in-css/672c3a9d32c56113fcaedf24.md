# --interactive--

固定定位和粘性定位是两种重要的CSS定位策略，每种都提供了与绝对定位不同的独特行为。当一个元素使用[position: fixed](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)定位时，它会从正常的文档流中移除，并相对于视口放置，这意味着即使用户滚动页面，它也会保持在同一位置。这通常用于需要始终保持可见的元素，如页眉或导航栏。例如，如果你想要一个标题固定在页面顶部，可以使用以下代码：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>固定页眉</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
body {
  margin: 0;
  padding-top: 60px;
  font-family: Arial, sans-serif;
  line-height: 1.6;
}

h1 {
  position: fixed;
  top: 0;
  width: 500px;
  background: white;
  padding: 10px;
  border-bottom: 2px solid #ccc;
}

p {
  max-width: 600px;
  margin: 20px auto;
}
```

:::

在这个例子中，[h1](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)元素将固定在视口的顶部，即使用户滚动页面，它也会保持在原位。这对于创建持久的UI元素非常有用，如粘性页眉或始终可见的导航。

[position: sticky](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)表现为相对定位和固定定位的混合体。最初，元素表现得像相对定位一样，保持在文档流中。然而，一旦用户将元素滚动到某个点之后，它就会"粘"在视口上（通常是顶部），表现得像固定定位一样。这对于创建粘性导航栏等元素非常有用，这些元素只有在用户滚动到某个位置时才会变成固定定位。以下是应用粘性定位的方法：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>粘性页眉</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
h1 {
  position: sticky;
  top: 30px;
  left: 30px;
}
```

:::

在这个例子中，[h1](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)元素将正常表现，直到用户将其滚动到距离顶部`30px`的位置，此时它会"粘"在视口的顶部。

以下是绝对定位、固定定位和粘性定位之间的一些关键区别：

- 绝对定位将元素从文档流中移除，并相对于最近的已定位祖先元素定位，如果不存在则相对于初始包含块定位。元素将保持在该位置，无论是否滚动。
- 固定定位也会将元素从文档流中移除，但相对于视口固定，这意味着即使页面滚动，它也会保持在相同的位置可见。
- 粘性定位最初将元素保持在正常的文档流中，但在滚动超过定义的阈值后允许它粘在原位。

这些策略允许你创建包含浮动元素、粘性页眉和更多动态UI组件的布局，通过使重要信息始终可访问来增强用户体验。
