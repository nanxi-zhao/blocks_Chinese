# --description--

在JavaScript中，模块是封装相关函数、类或变量的自包含代码单元。

可以把模块想象成应用程序的构建块，就像书中的章节一样。每个模块都专注于特定的功能，使你的代码更有组织性、可维护性和可重用性。模块有助于防止命名冲突，并允许你将应用程序构建成独立的、相互连接的部分。

JavaScript中的模块概念随着时间的推移而发展，但最广泛使用和支持的方法是ES6（ECMAScript 2015）模块系统。该系统提供了一种标准化的方式来定义和使用跨不同JavaScript环境的模块。

要创建模块，你需要将JavaScript代码写在单独的文件中。任何你想让应用程序其他部分可用的变量、函数或类都需要明确导出。你可以使用[export](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)关键字来实现。

例如，假设你有一个名为[math.js](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L40-L40)的文件，其中包含一些数学函数：

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

在这个例子中，我们导出了两个函数（[add](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)和[subtract](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)）和一个常量（[PI](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L29-L29)）。你可以从单个模块中导出任意数量的项目。

要在应用程序的其他部分使用这些导出的项目，你需要导入它们。这是使用[import](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)关键字完成的。

假设你想在名为[app.js](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L72-L72)的文件中使用这些数学函数：

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // 输出: 8
console.log(subtract(10, 4));  // 输出: 6
console.log(PI);               // 输出: 3.14159
```

在这里，我们从[math.js](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L40-L40)模块中导入我们需要的特定函数和常量。[./math.js](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L40-L40)部分告诉JavaScript在哪里找到相对于当前文件的模块文件。

有时，你可能想要导入模块导出的所有内容。你可以使用星号（[*](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)）语法来实现：

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // 输出: 8
console.log(Math.subtract(10, 4));  // 输出: 6
console.log(Math.PI);               // 输出: 3.14159
```

在这种情况下，[math.js](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/blocks_english/lecture-understanding-modules-imports-and-exports/67329fd6ad99c75d4a4b74e4.md#L40-L40)的所有导出都被导入为名为[Math](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/propTypes.js#L45-L45)的对象的属性。

另一种常见的模式是在模块中有一个默认导出。这通常用于模块主要导出单个函数的情况。每个模块只能有一个默认导出。

以下是它的样子：

```js
// 在math.js中
export default function multiply(a, b) {
    return a * b;
}

// 在app.js中
import multiply from './math.js';

console.log(multiply(4, 5));  // 输出: 20
```

注意，当导入默认导出时，你不需要使用花括号，并且你可以给导入任意命名。

重要的是要注意，要在浏览器中使用ES6模块，你需要在[script](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)标签中指定[type](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)为[module](file:///e:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/eslint-plugin-react/lib/util/pragma.js#L42-L42)：

```html
<script type="module" src="app.js"></script>
```

模块为组织和构建JavaScript代码提供了一种强大的方式。它们允许你将应用程序分解为更小的、可管理的部分，促进代码重用，并帮助保持关注点的清晰分离。当你构建更大的应用程序时，你会发现模块成为JavaScript开发工具包中的重要工具。
