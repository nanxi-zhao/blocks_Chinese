# --description--

为什么有些网站感觉敏捷且响应迅速，而其他网站感觉迟缓？答案在于影响页面加载和渲染的关键性能概念。理解关键的Web性能概念对于构建快速、流畅且用户友好的网站至关重要。

让我们分解源码顺序、关键渲染路径、延迟等概念。

首先让我们看看源码顺序。

源码顺序是指HTML元素在文档中的结构化方式。这决定了什么内容先加载，可以显著影响性能和可访问性。

这方面的一些最佳实践包括：

- 将关键内容（如标题、导航或正文文本）放置在HTML结构的较高位置。

- 延迟加载非必要脚本（如分析脚本或第三方小部件），这样它们不会阻塞渲染。

- 使用渐进增强，确保即使在样式和脚本加载之前，核心体验也能正常工作。渐进增强是一种基于首先让页面使用HTML工作的理念来构建网站和应用程序的方法。

这里是一个使用我们刚刚讨论过的最佳实践的良好源码顺序示例。

```html
<!-- 良好的源码顺序：关键内容优先 -->
<h1>欢迎来到FastSite!</h1>
<p>关键信息首先加载。</p>
<script src="slow-script.js" defer></script>
```

通过优化源码顺序，我们确保用户尽快看到重要内容。

现在让我们看看关键渲染路径。

关键渲染路径是浏览器将代码转换为屏幕上像素的步骤序列。

以下是稍后将详细介绍的关键步骤：

1. 解析HTML：构建DOM（文档对象模型）

2. 解析CSS：构建CSSOM（CSS对象模型）

3. JavaScript执行：可以修改DOM和CSSOM

4. 渲染树构建：结合DOM和CSSOM

5. 布局和绘制：确定元素大小并绘制像素

6. 优化：

    - 最小化渲染阻塞资源（例如，大型CSS文件、未使用的JS）。

    - 对脚本使用async和defer属性：

    - 首先只加载必要的样式；延迟非关键CSS。

总的来说，较短的关键渲染路径等于更快的感知性能。我们稍后会更详细地讨论这一点。

最后，让我们看看延迟。

延迟是请求在浏览器和服务器之间传输所需的时间。换句话说，高延迟等于页面加载缓慢。

减少延迟的一些方法包括：

- 使用CDN（内容分发网络）从更近的位置提供文件。

- 启用压缩（如Gzip）以减少文件大小。

- 优化图像并使用懒加载——我们稍后也会讨论这一点。

通过减少延迟，我们使交互感觉即时。

所以总之，通过优化源码顺序、减少关键路径和降低延迟，你可以让你的网站感觉快速。
