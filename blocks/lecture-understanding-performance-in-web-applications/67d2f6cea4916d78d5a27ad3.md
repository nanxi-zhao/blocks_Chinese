# --description--

让我们通过一个实际的方法来了解如何使用Chrome DevTools测量Interaction to Next Paint（INP）。

首先，让我们通过按"F12"或右键单击并选择"检查"来打开DevTools。

现在，假设你的网页上有一个搜索栏。

想象一个用户输入查询，但建议下拉菜单出现需要600毫秒。Chrome DevTools会标记这个高INP值。

你可以通过延迟重型JavaScript、减少长任务和改进事件处理来优化。然后INP会降到150毫秒，使交互感觉即时。

很酷，对吧？

现在，让我们看看一种实用的测量技术，它有助于识别交互延迟，这样你就可以创建更快、更流畅的用户体验！

首先，我们要打开Chrome DevTools。我们再次通过打开Google Chrome，导航到我们想要分析的网页，然后按"F12"或右键单击页面并选择"检查"来实现。现在，让我们转到"性能"选项卡——这就是你需要的一切。

现在我们准备好识别Interaction to Next Paint（INP）了。我们要寻找记录的最长交互延迟。良好的INP通常低于200毫秒，而差的INP高于500毫秒。就是这样！
