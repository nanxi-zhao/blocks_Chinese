# --description--

改善"可用时间"，换句话说，从用户请求页面到他们能够有意义地与之交互的时间间隔，对于增强用户体验至关重要。

让我们探讨有效的策略来实现更快的可用时间。

首先关注用户首先看到的内容。这意味着立即加载必要元素并延迟非关键组件。

例如，你可以尝试实现懒加载。

我们已经讨论过懒加载，但这里是一个懒加载图像的示例作为复习：

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

你所需要做的就是在图像标签上添加`loading="lazy"`属性。这会延迟图像的加载，直到需要时，通常当它们进入用户的视口时。通过只在图像进入视口时加载图像和视频，我们既节省了带宽，又加快了初始渲染速度。

通常，网页会一次性加载所有图像，甚至包括屏幕上不可见的图像。使用懒加载，折叠线以下或屏幕外的图像只有在用户滚动到它们时才会加载。

这里是懒加载iframe的示例：

```html
<iframe src="video.html" loading="lazy"></iframe>
```

你可以应用之前使用的相同属性，但这次是针对iframe，同样的概念也适用。

你还可以使用`defer`属性来延迟非关键JavaScript，直到初始页面加载完成。这里是我们的示例：

```html
<script src="non-critical.js" defer></script>
```

你还可以考虑最小化渲染阻塞资源。

渲染阻塞资源会延迟页面变得可交互。

为了避免这种情况，你可以尝试异步加载CSS。对于非关键CSS，使用`media`属性来有条件地加载样式表。这里是我们的示例：

```html
<link rel="stylesheet" href="print.css" media="print">
```

通过将media值指定为`print`，`print.css`只会在打印页面或在屏幕上预览时加载。在下一个示例中，通过将`min-width: 800px`作为值传递给`media`属性，我们表示我们只希望`desktop.css`在宽度大于800px的屏幕上加载：

```html
<!-- 这个样式表只在宽度大于800px的屏幕上加载 -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

这通过跳过不必要的样式来节省带宽。

所以，总之，通过实施这些策略，你可以显著减少网站的"可用时间"，确保用户能够及时与你的内容交互。
