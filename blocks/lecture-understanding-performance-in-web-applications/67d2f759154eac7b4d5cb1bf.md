# --description--

正如我们所知，感知性能不仅仅是网站实际有多快——而是用户感觉有多快。所以，例如，如果你曾经使用过一个感觉很慢的网站，即使它并不慢？这就是感知性能——网站给用户的感觉有多快。

小的调整可以对用户体验产生巨大影响。让我们探讨一些改善感知性能的实用方法！

旋转器可能让用户感觉等待时间比实际更长。相反，尝试骨架屏幕——模仿真实内容的灰色占位符，并逐渐加载：

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ 内容已加载!";
  }, 2000);
</script>
```

用户感觉页面正在逐步加载，而不是在黑暗中等待。

除此之外，用户不应该等待一切加载完成才能开始交互。先加载重要内容，然后在后台加载其余内容。我们之前在懒加载中讨论过这一点：

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

这使得页面感觉响应迅速，即使不是所有内容都已加载。

现在——如果你的网站能预测用户接下来会点击什么怎么办？通过预加载，你可以在用户请求之前获取下一页：

```html
<link rel="preload" href="next-page.html" as="document">
```

这减少了用户导航时的感知加载时间，几乎为零。

最后，用户不喜欢不确定性。当他们点击按钮时，立即确认他们的操作：

```html
<button onclick="this.innerText='⏳ 处理中...'; setTimeout(() => this.innerText='✅ 完成!', 2000);">
  点击我
</button>
```

这意味着，即使有延迟，用户也会感觉有事情在发生。

所以，总之，改善感知性能不仅仅是原始速度，而是速度给用户的感觉。使用骨架屏幕，优先处理重要内容，智能预加载，并提供即时反馈，让你的网站感觉更快。
