# --description--

让我们来学习终端，以及它与命令行的区别。

命令行是一个基本的文本输入界面，允许用户输入"命令"，通常以一系列字符的形式，并提交或执行它们，通常通过按"Enter"键。你最常见到的命令行界面是在终端内。

终端是一个特殊的应用程序，提供命令行界面来执行超出基本读/写操作的系统级命令。你也可能听说过"终端模拟器"，这些本质上是包装基本终端界面的应用程序，以提供额外的功能或特性。但对于大多数一般用途，你可能可以安全地将这些称为"终端"。

最后，我们还需要谈谈"shell"。shell是包装命令行并解释你的输入作为命令的软件，返回输出。

那么你如何在系统上访问终端呢？

好吧，如果你运行的是现代版本的Windows，你可能会有两个不同的选择。Windows开箱即用地提供了两个不同的shell应用程序：Powershell和命令提示符。

这两个shell都配有各自的终端应用程序（与各自的shell同名），所以你可以使用应用程序菜单启动Powershell终端或命令提示符终端。

Windows还提供了一个相对较新且现代化的终端模拟器，恰当地命名为Microsoft Terminal。

你可以从Windows商店安装这个应用程序，然后可以像启动Powershell或命令提示符终端一样启动它。Microsoft Terminal的优势在于它允许你从同一应用程序访问Powershell和命令提示符shell，以及如果你配置了Windows Linux子系统的话，还可以访问相应的Linux shell。

MacOS提供了一个恰当地命名为"Terminal"的终端。你可以通过聚光搜索或应用程序启动器访问它。

MacOS还支持第三方终端模拟器，如iTerm或Ghostty。你可以从它们的主页安装这些应用程序。

在Linux上，默认终端（以及如何启动它）高度依赖于你使用的发行版和桌面环境。对第三方终端模拟器也有很多支持。例如，你可以在Arch Linux中设置kitty - 并且可以使用wofi等应用程序管理器启动它。

Linux可以说拥有最多样的终端应用程序可供选择。大多数这些都可以通过你的发行版的包管理器安装。

"终端"、"shell"和"命令行"之间的区别有时可能看起来相当学究。在大多数情况下，这些术语通常可以互换使用。但重要的是要知道它们有特定的含义，以及每个含义是什么。
