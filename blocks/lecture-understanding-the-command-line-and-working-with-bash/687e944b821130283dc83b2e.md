# --description--

让我们来学习Bash和它的一些基本命令。

Bash代表Bourne Again SHell，可以说是你在类Unix环境中遇到的最常见的shell。因此，了解如何导航这个shell和使用常用命令是非常有益的。

第一个命令是[pwd]，它将当前工作目录打印到终端。"工作目录"指的是终端当前指向的目录，这对于许多其他命令来说是一个重要的信息。

你的终端完全可能指向错误的目录。[cd]命令允许你更改目录。你可以指定一个以正斜杠（[/]）为前缀的绝对路径，一个没有前缀的相对路径，并使用双点（[..]）语法移动到父目录。

下一个命令是[ls]，它列出当前工作目录的内容。这有助于查看文件或文件夹是否存在于你的目录中。[ls]命令还接受标志来显示隐藏文件（[-a]标志）、文件权限（[-l]标志）和其他功能。你将在下一课中了解更多关于命令标志的信息。你也可以使用[less]或[cat]来查看文件内容。

如果目录不包含你需要的文件或文件夹怎么办？有叫做[mkdir]和touch的命令允许你创建它们。[mkdir]创建一个新目录或文件夹，[touch]创建一个新文件。这两个命令都接受你想要创建的文件或文件夹的名称。例如，[touch readme.md]创建一个名为[readme.md]的新Markdown文件。

也许你的新文件不应该命名为[readme.md]，而应该命名为[readthis.md]。幸运的是，[mv]命令允许你移动（或重命名）文件。它需要旧文件名和新文件名。

[rm]命令允许你删除文件，使用[-r]标志将删除目录。有时文件可能受到保护，你需要包含[-f]标志。

[cp]命令可用于将文件或目录复制到新位置。与[mv]命令不同，[cp]命令不会删除原始文件 - 并且为了复制目录，你需要传递[-r]标志。

[echo]命令可以被认为是Bash中[console.log()]或[print()]函数的等价物。[echo]接受一个用引号括起来的字符串参数，并将其打印到终端。在终端中运行它只是为了在终端中获得文本可能看起来很愚蠢，但你实际上可以将[echo]链接到一个控制符号。[>]符号允许你指定要创建或用新字符串覆盖的文件名，如[echo "Naomi was here." > readme.md]，而[>>]符号将追加到文件中。

还有更多Bash命令可供你使用，如[head]、[tail]、[chown]和[chmod]。在这节课中不可能涵盖所有内容。但你可以使用man命令查看几乎所有命令的手册或帮助页面。

你可能会记住最常使用的命令。对于其余的命令，你总是可以查看手册页。
