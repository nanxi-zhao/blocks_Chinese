# --description--

让我们来学习命令选项和标志。

在之前的课程中，你学习了向命令传递参数，如[touch readme.md]。选项或标志是你可以传递给命令的特殊参数，它们会影响命令的行为方式。这两个术语可以互换使用，尽管"标志"更具体地用于作为开/关切换的选项。

选项通常以一个或两个连字符（[-]）符号为前缀，这有助于在选项和参数之间提供视觉区分。首先，让我们看看双连字符或"长格式"语法。

许多应用程序，如[ls]，接受[--version]标志，它将应用程序的当前版本打印到终端而不是运行应用程序的命令。

许多应用程序还提供[--help]标志，它打印如何使用应用程序的说明。

单连字符或"短格式"语法通常使用单字母选项。例如，带有[ls]的[-a]标志列出所有文件，包括以点（[.]）开头的隐藏文件，如[.env]。

这些短选项的优势在于你可以将多个标志链接在一起。而不是[ls --all --human-readable --size]，你可以一次性使用单个字母[ls -ahs]。

一些选项期望传递一个值给它们。

使用长格式选项时，通常需要使用等号。在这种语法中，值直接与选项用等号（[=]）符号连接。以下是修改[ls]行为以包含或排除颜色的示例：

```bash
ls --color=never
```

使用短格式选项时，通常用空格分隔值。例如，以下是设置[ls]结果的[width]的长格式和短格式选项：

```bash
ls -w 50
```

```bash
ls --width=50
```

注意短格式使用[-w 50]，而长格式使用[--width=50]。了解这种区别很重要，以避免将选项值作为位置参数传递。

但如果你不确定，记住你通常可以使用[--help]标志查看选项的预期语法！

## --text--

命令选项的"长格式"和"短格式"语法有什么区别？

## --answers--
