# --description--

在上一课中，你了解到服务器处理 HTTP 请求并发送回响应。该响应包括你在网络上看到的各种资源。

HTTP 响应返回的资源包括基于文本的资源、媒体资源、字体等等。这些资源中的每一种都带有 **Content-Type** 头部，它告诉浏览器如何正确处理和显示资源。

基于文本的资源一直是网络的基本组成部分。实际上，网络最初是纯文本的。如今，基于文本的资源包括以下内容：

* **HTML（超文本标记语言）**：任何网页的基本结构。没有 HTML，网页就无法存在。

* **CSS（层叠样式表）**：控制网页外观的资源。使用 CSS，你可以将颜色、字体和布局等样式应用到 HTML 上。

* **JavaScript**：JavaScript 为 HTML 和 CSS 添加交互性和动态内容。

* **JSON 和 XML**：这两种格式允许你在服务器和浏览器之间传输结构化数据。

媒体资源包括图像、音频和视频。它们通过视觉和声音让网络变得生动。

许多网站依靠图像来传达视觉信息。图像有不同的格式，如 `JPG`、`JPEG`、`PNG`、`WEBP` 和 `GIF`。

像 YouTube 和 Vimeo 这样的网站提供视频内容，格式包括 `MP4`、`MOV`、`AVI`、`WebM` 等。

音频是许多平台的关键组成部分，因为它允许用户流式传输音乐、播客和其他声音内容。常见的音频格式包括 `MP3`、`AAC`、Ogg 和 `WAV`。Spotify 和 YouTube Music 等平台依靠这些格式提供高质量的音频体验。

HTTP 响应中返回的其他资源包括文档如 `PDF`、`DOCX` 和 `PAGES`，压缩档案如 `ZIP` 和 `RAR`，字体文件如 `WOFF`、`WOFF2` 和 `TTF`，以及可执行文件如 `EXE`。

以下是这些常见资源及其内容类型的表格：

| 资源类型  | Content-Type 头部    |
| ----------- | ---------------------- |
| HTML        | text/html              |
| CSS         | text/css               |
| JavaScript  | application/javascript |
| JPEG        | image/jpeg             |
| PNG         | image/png              |
| GIF         | image/gif              |
| SVG         | image/svg+xml          |
| WebP        | image/webp             |
| WOFF        | font/woff              |
| WOFF2       | font/woff2             |
| TTF         | font/ttf               |
| OTF         | font/otf               |
| MP4         | video/mp4              |
| WebM        | video/webm             |
| Ogg (Video) | video/ogg              |
| MP3         | audio/mpeg             |
| AAC         | audio/aac              |
| WAV         | audio/wav              |
| Ogg (Audio) | audio/ogg              |
| JSON        | application/json       |
| XML         | application/xml        |
| PDF         | application/pdf        |
| ZIP         | application/zip        |
