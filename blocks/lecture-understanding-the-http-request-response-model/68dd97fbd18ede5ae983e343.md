# --description--

网站和应用程序需要收集数据并处理它们，或者生成用户需要的内容，如搜索结果。因此，就像文本一样，表单也是网络的基本组成部分。

通过表单，用户可以注册并登录网站、发送消息和搜索项目。例如，你可以通过在 Web 上最流行的表单——Google 搜索输入框中输入查询，来访问 Google 搜索结果页面。

让我们来看看表单的组成，如输入字段、[action] 属性、表单方法，最重要的是，提交表单时会发生什么。

表单通过 [form] 元素嵌入到网页中，但要让输入框显示出来，你还需要 [input] 元素，这取决于你的需要。其中一些输入元素包括文本、电子邮件、密码、选择、文件、隐藏输入等。

要使用这些输入字段，你必须用 [form] 元素将它们包装起来。每个输入都需要一个 [name] 属性，这样服务器就知道哪些数据对应哪个字段。

下面是一个例子：

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

输入字段还需要进行验证，以确保用户输入了正确的信息。

这种验证可以在客户端和服务器端进行。HTML 允许你使用 [required] 和 [pattern] 属性进行基本的客户端验证。即便如此，你也应该在服务器端进行验证以增加安全性，因为 HTML 验证主要是为了用户体验和即时反馈。

[required] 属性指定输入字段必须填写，而 [pattern] 属性允许你输入正则表达式，强制用户按照你需要的方式输入信息。

下面是用户名字段的一个示例：

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

输入字段验证后，应该进行提交。那么表单提交时会发生什么呢？

首先，要进行提交，表单需要有一个 [action] 属性，指定浏览器应该将输入字段的值发送到哪里，以及一个 [method] 属性，指定表单数据如何提交。稍后会详细介绍 [method] 属性。

下面是 action 属性的一个示例：

```html
<form action="/register">
    <!-- input fields -->
</form>
```

[method] 属性可以是 `GET`、`POST`、`PUT`、`PATCH` 或 `DELETE`：

* `GET` 从服务器检索某项内容，例如，搜索结果。

* `POST` 向服务器发送某项内容，例如，通过输入字段输入的登录或注册信息。

* `PUT` 修改服务器上的某项内容。

* `PATCH` 部分修改服务器上的某项内容。

* `DELETE` 从服务器删除某项内容。

请注意，HTML 不直接支持 `PUT`、`PATCH` 和 `DELETE` 方法，但有变通方法。如果你使用纯 JavaScript，可以使用 Fetch API 指定方法，在 Node.js 和 Express 中，可以使用 `method-override` 包。

下面是进行 `GET` 请求的表单示例：

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

下面是进行 `POST` 请求的表单示例：

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

`GET` 或 `POST` 请求之后，服务器会检查服务器端验证，并处理或拒绝输入值。

如果验证成功，服务器会发送请求的信息或在验证失败时抛出错误。

如果验证成功，服务器的响应可能是重定向（例如身份验证或搜索），或者确认操作成功的成功消息。

下面是表单提交时发生的事情的总结：

* 用户填写表单字段。

* 浏览器检查客户端验证（如果有），并在验证成功后继续。

* 浏览器根据 action 和 method 属性将表单数据发送到服务器。

* 服务器接收数据并进行处理（将其存储在数据库中、注册用户、登录用户、发送搜索查询等）。

* 如果有服务器端验证，服务器在处理之前重新检查数据。

* 服务器根据验证状态（成功或失败）响应成功消息、错误或重定向用户。

* 用户根据服务器响应看到反馈。
