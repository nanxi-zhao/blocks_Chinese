# --interactive--

**注意**：一些交互式示例可能使用了你尚未学习的CSS。不用担心理解所有代码。示例的目标是向你展示这些设计概念的预览，以便你更好地理解工作原理。

互联网上有成千上万的电子商务网站，购物车是电子商务体验的重要组成部分。好的设计可以让购物车体验更加愉快并增加销售额。糟糕的设计会导致购物车被遗弃和销售损失。

在本课程中，我们将讨论设计购物车的一些最佳实践。第一个设计考虑因素是确保购物车始终对用户可见。大多数购物车设计会将购物车显示在页面的右上角。用户应该看到购物车图标旁边显示的购物车中商品数量，并能够点击购物车查看他们正在购买的商品的更多详细信息。

另一个考虑因素是为用户提供清晰的方式更新购物车中商品的数量。这可以通过在购物车中每个商品旁边提供数量输入字段来实现。用户可以通过更改输入字段中的数字轻松更新商品数量。

你还应该在购物车中每个商品旁边提供一个"删除"按钮。这允许用户轻松地从购物车中删除商品。你不想让用户难以从购物车中删除商品，因为这可能导致挫败感和购物车被遗弃。

另一个考虑因素是购物车图标本身。图标应该是一些所有用户都能轻松识别的东西。常见的图标是一个带手柄和轮子的购物车。其他图标可能是购物袋或篮子。但你不想选择一个过于抽象或难以理解的图标。

当用户想要查看购物车中的总计时，他们应该能够轻松找到购物车中所有商品的总成本。这应该在页面上显著显示，这样用户就不必搜索它。

最后，你应该为用户提供一个清晰的行动号召按钮来继续结账。这个按钮应该在页面上显著显示，这样用户就不必搜索它。

你不想在页面上有太多按钮，因为这可能导致混淆。行动号召按钮应该是页面上最显著的按钮，这样用户就知道接下来要做什么。你应该使用品牌的主色调作为按钮，这样它就能从页面的其余部分中脱颖而出。
