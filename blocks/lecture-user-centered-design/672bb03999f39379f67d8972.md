# --interactive--

**注意**：一些交互式示例可能使用了你尚未学习的CSS。不用担心理解所有代码。示例的目标是向你展示这些设计概念的预览，以便你更好地理解工作原理。

渐进式披露是一种设计模式，仅向用户显示基于他们当前活动的相关内容，而隐藏其余内容。这样做是为了减少认知负荷，使用户体验更加直观。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="progressive-disclosure">
  <button id="show-details-btn" aria-expanded="false" aria-controls="extra-content">
    显示更多详情
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">
    <p>
      这里是只有当你想看时才会显示的额外详情。
      这保持了界面的整洁，减少了认知负荷。
    </p>
  </div>
</div>
```


```js
const btn = document.getElementById('show-details-btn');
const content = document.getElementById('extra-content');

btn.addEventListener('click', () => {
  const isHidden = content.classList.toggle('hidden');

  if (!isHidden) {
    content.focus();
  }
});

```

例如，当你使用Google的搜索页面时，大多数搜索都很简单。所以拥有一个选项很少的简单用户界面是有意义的。但对于少数需要高级搜索的时候，Google提供了一个高级搜索选项，会显示更多选项。

你可以从Google首页的设置菜单中找到Google的高级搜索选项。或者你可以访问[google.com/advanced_search]。

渐进式披露的另一个示例是电子商务网站上产品的"更多详情"按钮。当你访问像亚马逊这样的网站时，你会看到不同类别中的产品图片列表。这允许用户轻松购物，不会被太多信息压倒。

如果用户对某个特定产品感兴趣，他们可以点击产品图片来显示更多关于该产品的详情。立即显示所有信息可能会让用户感到不知所措。因此，通过使用渐进式披露，用户可以在准备好时选择查看更多信息。

:::interactive_editor

```js
document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = '隐藏详情';
    } else {
      button.textContent = '更多详情';
    }
  });
});
```

那么，使用渐进式披露技术设计的一些最佳实践是什么？第一个考虑因素是始终保持所有重要信息可见。用户不应该因为额外的高级功能而难以找到关键信息。

另一个考虑因素是为用户提供访问额外功能或信息的单一接入点。这可以是一个始终在页面上可见的按钮或链接。添加多个接入点可能会让用户感到困惑，导致糟糕的用户体验。
