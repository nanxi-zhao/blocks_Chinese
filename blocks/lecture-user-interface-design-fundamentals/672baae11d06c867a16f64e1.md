# --description--

渐进增强是一种设计方法，确保所有用户无论使用何种浏览器或设备，都能访问应用程序的基本内容和功能。

它专注于为每个人提供核心体验，同时为使用更高级浏览器或更好网络连接的用户提供额外功能和改进。

渐进增强方法遵循以下核心原则：

- 所有核心内容和基本功能都应在所有浏览器上可访问
- 所有高级布局都应通过外部 CSS 样式表提供
- 所有高级功能都应通过外部 JavaScript 文件提供
- 应尊重用户的浏览器偏好设置

使用渐进增强方法可以使您的应用程序更具可访问性，因为所有核心内容和功能在不支持的环境中都不应被阻止。

在速度方面，渐进增强方法也有助于提高应用程序的性能。

使用较慢网络连接的用户仍然能够访问内容，因为浏览器会首先下载必要的资源。

在SEO方面，渐进增强也有助于提高应用程序的可见性。

搜索引擎将能够爬取您的应用程序内容，因为核心内容在初始HTML响应中是可用的。

虽然有些人批评这种方法，认为对于严重依赖JavaScript功能的应用程序来说，这种方法并不总是现实的，但在构建应用程序时，这仍然是一个值得遵循的良好实践。
