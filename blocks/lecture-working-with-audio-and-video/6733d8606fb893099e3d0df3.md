# --description--

你已经学习了 `HTMLMediaElement` 和 MediaStreams API，但还有几个 API 在处理视频和音频时非常有用。

第一个是屏幕捕获 API。顾名思义，这个 API 允许你录制用户的屏幕。通过调用 `mediaDevices` 对象的 `getDisplayMedia()` 方法并消费返回的媒体流来公开此 API。

然后是 MediaStream Recording API。这个 API 与 MediaStreams API 协同工作，允许你录制 MediaStream（甚至直接录制 `HTMLMediaElement`）。然后它会触发带有 Blob 负载的 `dataavailable` 事件，你可以将其写入本地文件存储。

所有这些技术的基础是媒体源扩展 API。媒体源扩展 API 允许你直接将用户的网络摄像头源传递给视频元素的 `srcObject` 属性，例如。

最后是为网络上所有可听内容提供动力的 Web Audio API。这个 API 包括重要的对象，如 `AudioBuffer`（表示专门包含音频数据的缓冲区）或 `AudioContext`。

这结束了我们关于音频和视频的课程！
