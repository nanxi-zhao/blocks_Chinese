# --description--

首先，你需要知道 VS Code 将项目视为"工作区"。工作区就是你在 VS Code 中打开的目录。

首先，你需要使用命令行界面为你的新项目创建一个新目录。目录是文件夹的另一个名称。

命令行界面（简称 CLI）允许你通过基于文本的命令与操作系统交互。你将在未来的课程中了解更多关于这些工具的知识。

如果你使用的是 Windows，你可以使用命令提示符或 PowerShell。如果你使用的是 Mac，你可以打开终端应用程序。

命令行打开后，通过输入 `cd ~` 命令并按键盘上的 `enter` 键导航到主目录。然后输入 `mkdir my-project` 命令并按 `enter`。`mkdir`（即"make directory"）是用于创建新目录的命令。然后你应该用 VS Code 打开你的新目录。

还有一些其他图形方式来创建新目录。

根据你的操作系统，你可能有一个上下文菜单（或右键选项）可以直接从文件资源管理器中打开 VS Code。你也可以选择双击 VS Code 应用程序。如果你对命令行感到舒适，可以运行命令 `code /path/to/folder`。

打开后，你应该看到一个新的空白工作区。

要创建新文件，点击左上角的 `File` 菜单并选择 `New File...`。然后将文件命名为 `index.html` 并点击 `Save`。

现在你应该在资源管理器选项卡中看到你的新文件。尝试在你的 `index.html` 文件中添加一些 HTML。

现在你可能很想直接在浏览器中打开 HTML 文件。虽然这可以工作，但它也可能导致各种意外的错误，比如 CSS 无法正确加载。

相反，你应该将你的 HTML 文件放在适当的网络服务器后面。如果你是编程新手，这听起来可能是一项令人生畏的任务，但实际上有一种方法可以直接从 VS Code 中做到这一点。

你需要先获取 _Live Server_ 扩展，它在市场中免费提供。

点击左侧角落的扩展选项卡并在搜索栏中输入 `Live Server`。

安装后，你应该在状态栏中看到一个 `Go live` 按钮。确保你的 HTML 文件已打开，然后点击它。

你的新页面应该会在浏览器的新标签页中自动打开。如果没有，你可以直接访问 `http://localhost:5500/`

恭喜！你现在已经成功在 VS Code 中设置了你自己的项目，并可以通过实时服务器查看你的更改。
