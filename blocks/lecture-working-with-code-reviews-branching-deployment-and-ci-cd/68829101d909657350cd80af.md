# --description--

GitHub Pages 是部署静态站点或不需要后端服务器处理逻辑的应用程序的绝佳选择。也就是说，完全在客户端或用户浏览器中运行的应用程序可以完全部署在这个平台上。

此外，GitHub Pages 对于开源项目是完全免费的。有很多充分的理由让你的项目开源，而 Pages 又增加了一个理由。

但你实际上是如何部署到 GitHub Pages 的呢？这取决于你的应用程序是如何工作的。

让我们从一个基本的 HTML 页面开始。如果你愿意，可以将以下内容添加到你现有的仓库中：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>MasterPuti</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>欢迎来到 MasterPuti 的课程！</h1>
    <p>让我们一起学习 Git 和 GitHub 吧！</p>
  </body>
</html>
```

现在我们需要访问我们的仓库并点击"Settings"选项卡进入仓库设置。然后我们需要从侧边栏选择"Pages"。

最初只有几个选项，但当我们开始配置设置时，这将会改变。首先，我们需要告诉 GitHub 从哪里部署。

我们要将此保留为"Deploy from a branch"，因为我们没有使用框架，也没有配置任何 GitHub 操作。你将在即将到来的课程中了解更多关于这些的内容。

接下来，将我们部署的分支设置为 `main` 以部署上面的内容。

然后你会得到一个选择，从项目的根文件夹或 `docs` 文件夹进行部署。这个功能被像 MasterPuti 这样的项目使用——我们过去常常将文档与我们的应用程序捆绑在一起，所以我们可以通过 `docs` 文件夹将文档部署到 GitHub Pages。现在，将其保留为 `/`（根）文件夹，并点击此下拉列表旁边的"Save"按钮保存你的更改。

保存后，你会看到一个新输入框来配置自定义域。如果你已经有了，并且想使用它，你可以按照链接到文档进行操作。然而，在本课中，我们将留空。

你还可以看到你的新 GitHub Pages 站点已部署到的域。没有自定义域时，这默认为 `<username>.github.io`，其中 `<username>` 是你的账户名称（或者如果你从组织拥有的仓库部署，则是组织的账户）。并且仓库将在与仓库名称匹配的路径上提供服务，例如 `<username>.github.io/<repo-name>`。如果你访问 URL，你会看到你的站点！

现在，当你将新更改推送到你的 `main` 分支时，GitHub Pages 将自动用最新更改更新我们部署的站点。这个过程可能需要几分钟，取决于服务器负载——你可能需要硬刷新来清除缓存。

这样，我们就成功地将一个基本的 Web 应用程序部署到了 GitHub Pages。你还可以在你的 Pages 部署中包含 CSS 和 JavaScript 文件，这样你就可以构建美观和交互式的站点。一旦你学习了像 React 或 Angular 这样的框架，你甚至可以在 GitHub Pages 上部署它们！
