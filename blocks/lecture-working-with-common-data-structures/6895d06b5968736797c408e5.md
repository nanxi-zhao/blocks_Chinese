# --description--

栈和队列是计算机科学中常用的数据结构。

它们是遵循特定规则添加和删除元素的线性数据结构。

## 栈

让我们从**栈**开始。

**栈**是一种后进先出（LIFO）的数据结构。

这意味着最后添加到栈中的元素是第一个被删除的元素。

栈有两个端点，我们称之为顶部和底部。

元素从栈的顶部添加和删除。

您可以将栈想象成一堆盘子，您只能在堆的顶部放置盘子并从堆的顶部取走盘子。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-1.png" alt="stack data structure visualization" />

在这种情况下，添加和删除元素的操作有特殊的名称。

向栈中添加元素称为"push"操作。当我们向栈的顶部添加元素时，我们说我们在栈上"push"一个元素。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-2.png" alt="stack push operation visualization" />

从栈中删除元素称为"pop"操作。当我们从栈的顶部删除元素时，我们说我们从栈中"pop"一个元素。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-3.png" alt="stack pop operation visualization" />

您可以看到我们实际上不在栈的底部执行任何操作，但我们将其保留在那里作为参考。

push和pop操作的时间复杂度通常是[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e5.md#L42-L42)，即常数时间复杂度。

当您向栈中push一个元素时，该元素简单地添加到顶部。

当您从栈中pop一个元素时，顶部的元素被删除。

因此，执行这些操作所需的时间保持不变，无论栈的大小如何。

push和pop操作的空间复杂度通常是常数[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e5.md#L42-L42)。这意味着执行这些操作所需的内存量保持不变，无论栈的大小如何。

## 队列

现在您对栈有了更多了解，让我们学习**队列**。

队列是一种先进先出（FIFO）的线性数据结构。这意味着添加到队列中的第一个元素是第一个被删除的元素。

队列有两个端点：前端和后端。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-4.webp" alt="queue data structure visualization" />

元素被添加到队列的后端，从队列的前端删除。

您可以将队列想象成在超市等待付款的人群。队伍中的第一个人是第一个去收银台的人，而新来的人在队伍的末尾加入。

在队列的上下文中，添加和删除元素的操作有特殊的名称。

向队列后端添加元素称为"enqueue"操作。

在enqueue操作中，新元素被添加到队列的末尾，成为队伍的末尾。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-5.png" alt="queue enqueue operation visualization" />

从队列前端删除元素称为"dequeue"操作。

在dequeue操作中，队列前端的元素被删除，队伍中的下一个元素成为新的前端。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-6.png" alt="queue dequeue operation visualization" />

enqueue和dequeue操作的时间复杂度是[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e5.md#L42-L42)，即常数时间。执行这些操作所需的时间保持不变，无论队列的大小如何。

enqueue和dequeue操作的空间复杂度通常是常数[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e5.md#L42-L42)。这意味着执行这些操作所需的内存量保持不变，无论队列的大小如何。

栈和队列是计算机科学中用于组织和管理元素的数据结构。理解它们对于在各种编程应用中构建高效算法至关重要。
