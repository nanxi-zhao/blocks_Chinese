# --interactive--

字体族是一组共享共同设计的字体。属于同一字体族的所有字体都基于相同的核心字体，但它们在样式、粗细和宽度上也有变化。您可以将它们视为共享相似特征但也有差异的兄弟姐妹。例如，Arial是一个字体族，包括Arial Bold和Arial Italic等变体。

在CSS中，您可以使用[font-family](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-fonts/672bd814105a0ffcf36f9233.md#L24-L24)属性设置字体族。这是将段落字体设置为[Arial](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-fonts/672bd814105a0ffcf36f9233.md#L24-L24)的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="arial-font">Example paragraph using Arial font.</p>
<p>Paragraph not using Arial font.</p>
```

```css
#arial-font {
  font-family: Arial;
}
```

:::

但如果找不到字体族怎么办？您可以通过用逗号分隔的方式按优先级顺序指定多个字体族，从高到低。这些替代字体将作为回退选项。在此示例中，Arial是主要字体，Lato是替代字体：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="specified-font">Example paragraph using specified fonts.</p>
<p>Paragraph not using specified fonts.</p>
```

```css
#specified-font {
  font-family: Arial, Lato;
}
```

:::

如果在设备上找到第一个字体（Arial），浏览器将渲染它。如果找不到，将渲染第二个字体（Lato）。

您还应该了解这个过程在幕后是如何工作的。如果第一个字体可用，选择过程不会停止。字体族是逐个字符选择的，因此如果字体缺少特定字符，浏览器会在优先级较低的字体中查找它。有趣，对吧？

在Web开发的背景下，您还会发现通用字体族。它们非常重要。通用字体族是当高优先级字体不可用时Web浏览器将渲染的默认字体。为确保内容仍然可读，浏览器会根据指定的通用字体族，用找到的最合适的字体替换原始字体。一些最常用的通用字体族包括：

- serif
- sans-serif
- monospace
- cursive
- fantasy

但还有更多选项可供选择。在下面的代码中，我们有三个替代字体族：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="specified-font">Example paragraph using specified fonts.</p>
<p>Paragraph not using specified fonts.</p>
```

```css
#specified-font {
  font-family: Arial, Lato, sans-serif;
}
```

:::

Arial具有最高优先级。如果找不到Arial，则浏览器将尝试渲染Lato。如果都找不到，浏览器将使用通用sans-serif字体族，从用户系统上安装的字体中选择具有这些特征的字体。

您应该始终在字体族列表的末尾包含一个通用字体族。请记住，通用字体可能与您在设计中最初设想的字体看起来不同，因此检查这些回退字体在不同浏览器中的外观总是有帮助的。

为了使用户体验尽可能一致，还建议使用Web安全字体。这些字体族通常安装在大多数设备上，因此很可能会被找到并为大多数用户正确渲染。您将在接下来的课程中了解更多关于它们的信息。

字体族在Web设计领域是必不可少的。通过使用font-family属性，您可以立即改变文本的外观和感觉。
