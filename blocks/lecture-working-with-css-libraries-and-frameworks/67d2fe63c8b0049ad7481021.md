# --description--

CSS预处理器是一种工具，它通过强大的特性如变量、混入、嵌套和选择器继承来扩展标准CSS。

这些特性中的一些，如变量和嵌套，现在在原生CSS中已经得到支持或正在获得更多支持，这种趋势很可能会继续下去。然而，早期版本的原生CSS并不支持这些特性。

这就是CSS预处理器最初变得如此广泛使用的原因。它们将你编写的CSS规则和样式连同它们的扩展语法一起编译成浏览器可以理解的原生CSS文件。

通过使用CSS预处理器，你可以以更可重用和逻辑的方式构建你的CSS代码。你还可以访问强大的特性，如混入，这些特性在CSS中不直接支持。

一些最流行的CSS预处理器是Sass、Less和Stylus。

让我们来谈谈Sass。

Sass代表"语法上令人惊叹的样式表"。它与所有版本的CSS兼容，并由一个大型开发者社区维护。

Sass支持以下特性：

- 变量，这样你可以在整个电子表格中存储和重用值。

- 嵌套的CSS规则，这样你可以在文件中创建视觉层次结构。

- 模块，这样你可以将样式拆分到多个样式表中。

- 混入，这样你可以在整个站点中重用CSS声明。

- 继承，这样多个CSS选择器可以共享属性。

- 运算符，用于基本的数学运算。

Sass还支持两种语法。

SCSS语法代表Sassy CSS，它是CSS的超集。这意味着SCSS扩展了CSS的基本语法。

SCSS是你在使用Sass时最常用的语法。它需要在CSS属性周围使用花括号（`{`和`}`）并在CSS声明末尾使用分号（`;`），就像原生CSS一样。

这些文件具有`.scss`扩展名。这是一个示例：

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

你在这里看到的，在顶部，是在SCSS中定义的变量。这个变量在下面的CSS规则中使用。

让我们将SCSS与较少使用的语法——缩进语法进行比较。这也被称为"Sass语法"，因为它是Sass的原始语法。

这种语法依赖缩进来定义规则。这是一个示例：

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

注意在CSS规则周围没有花括号，或者在CSS声明末尾没有分号。在顶部，你也可以看到一个变量。注意在这个语法中它的名称以美元符号（`$`）开头，这与原生CSS变量不同，但工作方式非常相似。

Sass还支持一个强大的特性，称为混入。

混入允许你将多个CSS属性及其值组合在名称下，并在整个样式表中重用该CSS代码块。

这使你的CSS代码减少重复并且更容易维护，因为如果你在该块中更改了某些内容，更改会应用到你使用混入的每个地方。

由于混入可以有自定义和描述性的名称，它们有助于理解每个CSS代码块的作用，这可以使你的样式表更容易理解。

混入还通过应用统一的样式来促进样式表中的一致性。

这是SCSS语法中混入的示例：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

要定义一个混入，你首先编写`@mixin`指令，然后是混入的名称。在这种情况下，混入被称为`center-flex`。它有三个使用Flexbox居中元素的CSS属性。

然后，一旦你定义了混入，你可以使用`@include`指令将这些属性包含在CSS规则中。你只需要编写`@include`后跟混入的名称。在这种情况下，名称是`center-flex`：

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

这是包含混入和CSS规则的完整代码：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

你可以根据需要在尽可能多的CSS规则中包含混入。如果你需要进行任何更改，你只需要更改混入本身，更改会自动应用到每个地方。这个示例是SCSS语法。注意它有花括号和分号。

这是缩进语法（也称为Sass语法）中的等效代码，没有花括号或分号：

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

这是编译后的CSS代码，浏览器在文件编译成原生CSS后实际解释的内容：

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

注意它在顶部包含了三个混入属性：`display`、`justify-content`和`align-items`。

这只是CSS预处理器功能的简单示例。它们有许多强大的特性，值得在你的日常工作中学习和应用。

然而，它们确实有潜在的劣势，你应该了解。

首先，CSS预处理器必须将CSS规则编译成标准CSS。虽然这种编译过程的潜在开销在时间和资源方面通常很小，但你应该在实际项目中考虑这一点。

接下来，它们可能造成调试挑战，因为浏览器直接使用编译后的CSS。与标准CSS相比，找出从扩展语法生成问题样式的内容可能需要额外的步骤。

然而，CSS预处理器的优势通常超过它们的劣势，特别是对于复杂的项目。它们对于编写更干净、可重用、较少重复和可扩展的CSS非常有帮助。
