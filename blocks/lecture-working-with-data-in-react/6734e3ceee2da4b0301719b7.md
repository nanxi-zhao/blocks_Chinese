# --description--

在之前的课程中，我们学习了如何在React中构建像这样的小组件：

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>你好 {developerName}！</h1>;
}
```

我们可以选择将这个组件嵌套在另一个父组件或根组件中，就像这样：

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>你好 {developerName}！</h1>;
}
```

虽然这段代码可以正常运行并在屏幕上显示"你好 Jessica！"的结果，但这不是一个非常灵活的组件。

如果我们想要显示不同的名字，比如Naomi、Tom或Oliver呢？这就是React props发挥作用的地方。Props是properties（属性）的缩写，是父组件向子组件传递数据的方式。Props可以是任何类型：字符串、数字、布尔值、对象或数组。

让我们更新之前的例子，现在接受一个`name` prop：

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>你好 {props.name}！</h1>;
}
```

对于名为`Greeting`的子组件，我们现在使用`props.name`而不是硬编码的名字"Jessica"。我们还在控制台中记录了`props`，它显示为一个对象。

然后，在父组件`App`内部，我们将值传递给`name` prop，这样它就可以传递给子组件。屏幕上的结果将与之前相同，但现在我们创建了一个更灵活的组件。

现在我们有能力多次重用子组件，并每次传递不同的名字：

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

你也可以选择在props中使用对象解构来使其更具可读性。以下是你可以重写`Greeting`组件的方式：

```jsx
function Greeting({ name }) {
  return <h1>你好 {name}！</h1>;
}
```

这段代码实现了相同的结果，但更清楚地表明了组件期望接收哪些props。

有时，你可能有很多属性需要作为props传递。与其逐个传递，不如在将它们转换为对象后使用展开运算符（`...`）。

这是一个名为`DeveloperCard`的新子组件的例子：

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>开发者：{name}</h1>
      <p>年龄：{age}</p>
      <p>国家：{country}</p>
    </div>
  );
}
```

这个`DeveloperCard`组件接受三个props：`name`、`age`和`country`。

在父组件`App`中，我们可以使用展开语法将对象中的所有属性作为单独的props传递给子组件：

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

当你处理对象数组并向子组件传递多组属性时，这特别有用。例如，你可能有一个开发者列表，其中数组中的每个对象都有相同的结构，但代表不同的人。

在未来的课程中，你将学习更多关于如何渲染数组中的列表的知识。

在React中使用props使你的组件更加灵活和可重用，让你能够构建更复杂的UI。但是，重要的是要注意props是不可变的，这意味着一旦传递给组件就不能更改。如果你需要处理用户输入和修改数据，你应该使用state。你将在未来的课程中学习更多关于管理state的知识。
