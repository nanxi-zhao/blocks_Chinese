# --interactive--

在上一课中，您学习了如何在 JavaScript 中使用 `Date` 对象。但在 JavaScript 中有几种不同的日期格式化方法。

在本课中，我们将看看如何使用 `toISOString()`、`toString()` 和 `toLocaleDateString()` 方法来格式化 JavaScript 中的日期。

在我们查看不同的方法之前，让我们先回顾一下 `Date` 对象的样子：

:::interactive_editor

```js
const date = new Date();
console.log(date);
```

:::

当您将 `date` 对象记录到控制台时，您会看到基于用户系统设置的当前日期和时间。以下是您可能看到的输出示例：

```js
Sun Sep 29 2024 19:45:37 GMT-0700 (Pacific Daylight Time)
```

如果您在 `date` 对象上使用 `toString()` 方法，您会看到与上面相同的输出。

要以扩展的 ISO 格式（ISO 8601）格式化日期，您可以像这样使用 `toISOString()` 方法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toISOString());
```

:::

ISO 8601 是表示日期和时间的国际标准。格式为 `YYYY-MM-DDTHH:mm:ss.sssZ`。

以下是您可能看到的输出示例：

```js
2024-09-30T02:47:20.292Z
```

另一种格式化日期的方法是使用 `toLocaleDateString()` 方法。此方法允许您根据用户的区域设置来格式化日期。以下是使用 `toLocaleDateString()` 方法的基本语法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString());
```

:::

以下是您可能看到的输出示例：

```js
9/29/2024
```

`toLocaleDateString()` 方法接受两个可选参数：`locales` 和 `options`。

`locales` 参数是一个表示要使用的区域设置的字符串。例如，您可以传入 `en-US` 表示英语（美国）或 `fr-FR` 表示法语（法国）。如果您不传入 `locales` 参数，则使用默认区域设置。以下是使用 `locales` 参数的示例：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString("fr-FR"));
```

:::

第二个可选参数是 `options` 参数。此参数是一个对象，允许您指定日期字符串的格式。以下是使用 `options` 参数的示例：

:::interactive_editor

```js
const date = new Date();
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options));
```

:::

在上面的示例中，我们指定了选项为英语（英国），并包括完整的星期几、年份、月份和日期。

以下是您可能看到的输出示例：

```js
Sunday, September 29, 2024
```

在本课中，我们只介绍了几种在 JavaScript 中格式化日期的方法。还有许多其他方法和库可以帮助您在 JavaScript 中格式化日期。但 `toISOString()` 和 `toLocaleDateString()` 方法是格式化 JavaScript 日期的良好起点。
