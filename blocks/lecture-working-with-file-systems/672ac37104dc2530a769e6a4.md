# --description--

当您开始构建自己的 Web 应用程序时，您需要留意为文件命名的方式。

您需要以一种易于理解和维护的方式来命名文件。

让我们来看一些 HTML 文件命名的好例子和坏例子。请注意，这些例子中的 `.html` 扩展名告诉我们这是一个 HTML 文件。

这是一个不好的文件名示例：

```md
index1234.html
```

仅从名称来看，不清楚文件的内容。这个文件可能是产品页面、博客页面或任何其他类型的页面，但我们从文件名中无法得知。

这是一个更好的 HTML 文件名示例：

```md
about-us.html
```

通过使用像 `about-us.html` 这样更具描述性的名称，可以清楚地知道文件的内容。

您还会注意到这个文件名中没有空格。相反，您使用连字符 `-` 来分隔单词。

有时您可能会看到一个名为 `index.html` 的文件名。这是一个特殊文件名，用于表示网站的主页。当您访问网站时，`index.html` 文件是第一个加载的文件。

命名文件时另一个重要的考虑因素是特殊字符的使用。

混合使用特殊字符、数字和字母会使我们难以理解文件的内容。

这是一个不好的文件名示例：

```md
file-1!@.html
```

这个名称看起来不必要地复杂，并且没有给我们任何关于文件内容的信息。

所以最好只在文件名中使用字母和连字符。

当您开始学习更多语言时，您会发现这些语言中也有命名文件的常见约定。

在这种情况下，最好查阅官方文档或其他资源来了解该语言中命名文件的最佳实践。

此外，您还会遇到在团队中工作的情况，需要遵循团队为命名文件建立的约定。

始终查阅团队的样式指南，以确保您的文件名与团队其他成员保持一致。
