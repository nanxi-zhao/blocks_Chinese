# --interactive--

`parseFloat()` 和 `parseInt()` 是 JavaScript 中用于将字符串转换为数字的两个基本方法。当处理用户输入或处理以字符串格式但需要作为数值处理的数据时，这些方法特别有用。

让我们从 `parseFloat()` 开始。这个方法解析一个字符串参数并返回一个浮点数。它被设计为从字符串的开头提取数字，即使字符串后面包含非数字字符。记住浮点数是带小数点的数字。以下是 `parseFloat()` 的工作方式：

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

正如你所看到的，`parseFloat()` 从字符串的开头开始解析，直到遇到不能成为浮点数一部分的字符为止。如果在字符串开头找不到有效数字，它返回 `NaN`（Not a Number）。

另一方面，`parseInt()` 解析一个字符串参数并返回一个整数。与 `parseFloat()` 一样，它从字符串的开头开始，但在遇到第一个非数字字符时停止。以下是 `parseInt()` 的工作方式：

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

:::

`parseInt()` 在遇到第一个非数字字符时停止解析。对于浮点数，它只返回整数部分。如果在字符串开头找不到有效整数，它返回 `NaN`。

这两种方法都有一些值得注意的行为。它们忽略前导空格：

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

它们处理字符串开头的加号和减号：

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

值得注意的是，虽然这些方法很强大，但它们有一些局限性。例如，它们不能直接处理所有数字格式，如科学计数法。对于更复杂的解析需求，你可能需要使用额外的技术或库。

总之，`parseFloat()` 和 `parseInt()` 是在 JavaScript 中将字符串转换为数字的宝贵工具。了解它们的工作原理和特定行为可以让你在应用程序中更有效地处理数字数据，特别是在处理用户输入或外部数据源时。
