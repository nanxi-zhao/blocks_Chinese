# --interactive--

在构建网页表单时，输入元素的外观和行为很重要。具有响应用户操作的输入表单在改善用户体验方面大有帮助，不应造成混淆或挫败感。

CSS提供了几种输入伪类，可以使您的表单更加直观和用户友好。让我们详细看看这些伪类。

[:focus]伪类允许您在用户选择或点击输入元素时将其作为目标，将注意力吸引到活动的输入字段上。这有助于用户轻松识别他们当前正在输入的位置：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

顾名思义，[:hover]伪类在用户将光标放在元素上时触发。它在用户与输入交互之前提供视觉反馈，有效地提醒他们输入已准备好操作。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

在上面的示例中，当用户将鼠标悬停在输入元素上时，输入元素的背景会变为橙色，让他们知道该字段已准备好进行交互。

[:checked]伪类允许您在用户选择单选按钮或复选框时以不同方式为其设置样式。这样，用户可以轻松看到他们选择了哪个选项。

这是一个带有复选框的示例，用于同意网站条款。

**注意**：此示例中的一些CSS使用了尚未涉及的属性。这只是为了让您了解如何创建自定义复选框。您将在未来的课程和研讨会中学习所有这些内容的工作原理。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  同意 <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}
```

:::

[:required]针对具有[required]属性的输入元素。它向用户发出信号，表明他们必须填写该字段才能提交表单。

这是一个带有某些必需输入字段的表单示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">姓名：</label>
  <input type="text" id="name" name="name" required />

  <label for="email">电子邮件：</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">电话号码：</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">提交</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

在验证表单时，您可以使用[:valid]伪类为符合验证标准的输入字段设置样式，使用[:invalid]为不符合标准的输入字段设置样式。通常，您会对有效输入使用绿色，对无效输入使用红色。

这是使用[:valid]伪类的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">电子邮件：</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

这是使用[:invalid]伪类的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">电子邮件：</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

[:disabled]伪类允许您选择和设置禁用的输入元素的样式。这些元素具有[disabled]属性，防止用户与它们交互。当输入被禁用时，无法点击、聚焦或编辑。

这是一个标签和禁用输入元素的示例。

在CSS中，我们针对该禁用输入并给它一个浅灰色背景，并将光标指针更改为[not-allowed]。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">姓名：</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

[css: not-allowed;] CSS属性值会更改光标的外观，以指示不允许执行操作。

应用于元素时，它会显示一个带有斜线符号的光标（通常是一个带对角线的圆圈），向用户发出信号，表明该元素不可交互或无法点击或与之交互。

以下是一些其他输入伪类示例及其作用：

- [:autofill]：为浏览器自动填充保存数据的输入字段应用样式。
- [:optional]：为非必需且可以留空的输入元素设置样式。
- [:in-range]和[:out-of-range]：根据值是否在指定范围内约束内设置元素样式。
