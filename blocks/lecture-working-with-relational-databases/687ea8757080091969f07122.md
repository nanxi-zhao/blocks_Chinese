# --description--

SQL代表结构化查询语言。它是在1970年代开发的，当时最初被称为结构化英语查询语言(SEQUEL)。这个术语后来被缩短为SQL。

SQL是一种用于在关系数据库中存储和管理数据的编程语言。

正如您在前面的课程中学到的，关系数据库将数据组织成表格。每个表格都有一组行和列，其中数据属性及其关系被表示出来。每行代表一个记录，而每列代表该记录的特定属性或字段。

使用SQL，您可以添加、更改、删除、查找和从关系数据库中检索数据。它提供了一套全面的命令用于查询、过滤、排序和聚合数据。

SQL可以通过仅查询必要的数据来帮助优化数据库性能，并且可以轻松地与各种编程语言集成，因此您可以使用它直接从应用程序与数据库交互。

它还被采用为行业标准。它强制执行数据完整性，并包含许多安全功能，如支持用户认证和数据加密。

它还具有可扩展性、可移植性，并与各种数据库管理系统兼容。

SQL基于命令，通常被称为SQL语句或SQL查询。

作为开发人员，您将使用特定的SQL语言元素或关键字编写这些语句。它们允许您在数据库上执行必要的操作。

在您输入SQL命令之前，您需要打开psql shell，以便可以直接与PostgreSQL数据库交互。在命令提示符或终端中，输入：

```bash
psql -U <username> -d <database_name>
```

将`username`替换为您的用户名，将`database_name`替换为您要连接的数据库。如果您还没有创建自己的数据库，请使用`postgres`，它随安装一起提供。

连接后，您将看到提示符更改为：

```bash
postgres=#
```

它显示您连接到的数据库名称并等待SQL命令。

让我们回顾一些您应该熟悉的基本SQL命令。您会注意到，按照惯例，SQL命令以分号（`;`）结尾。

首先，您可以使用此命令创建一个名为`my_database`的数据库：

```sql
CREATE DATABASE my_database;
```

您可以输入`CREATE`，然后是`DATABASE`，以及您要创建的数据库的名称。请注意，命令的SQL关键字是用大写字母书写的。

然后，一旦您有了数据库，就可以连接到它。

如何连接到数据库实际上取决于您使用的环境和工具。

在命令提示符或终端中，您可以使用前面显示的方法。在交互式psql shell中，您可以使用`\c`快捷命令后跟数据库名称来切换数据库，如下所示：

```bash
\c my_database
```

请注意，`\c`命令和其他快捷命令（前面有反斜杠的命令）是psql shell的一部分，而不是SQL语言本身的一部分，因此它们不需要分号来完成命令。

现在您已经连接到新数据库，您将看到提示符已更改：

```bash
my_database=#
```

创建并连接到数据库后，您可以使用`CREATE TABLE`关键字创建表格：

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

在命令中，您写入`CREATE TABLE`，然后是您要创建的表格的名称。在这种情况下，它将创建一个名为`products`的表格。

然后，在括号内，您写入表格应有的列名，并指定将存储在每列中的值的数据类型。

在示例中，表格将有两列，一列用于产品ID，另一列用于产品名称。

表格和列名的标准命名约定是蛇形命名法，用小写字母书写单词并用下划线（`_`）分隔。例如：`delivery_orders`

这些是您应该了解的一些基本SQL命令。

使用SQL，您可以轻松地查询、操作和分析数据，以做出明智的决策并解决现实问题。
