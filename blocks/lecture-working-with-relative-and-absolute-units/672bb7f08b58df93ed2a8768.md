# --interactive--

在上一课中，我们学习了像素等绝对长度单位。虽然绝对单位在某些情况下很有帮助，但有时您会希望使用相对单位。

在本课中，我们将学习两种相对单位：`em`和`rem`。

`em`单位相对于元素的字体大小。如果您在`font-size`属性中使用em，文本大小将相对于父元素的字体大小。

为了更好地理解其工作原理，让我们看一个例子：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p class="para">I am a paragraph element</p>

<div class="blue-box"></div>
```

```css
.para {
  font-size: 20px;
  margin-bottom: 1.5em;
  border: 2px solid red;
}

.blue-box {
  background-color: blue;
  color: white;
  padding: 10px;
  width: 100px;
  height: 100px;
}
```

:::

对于HTML，我们有一个段落和一个`div`元素。段落元素有一个`para`类，`div`元素有一个`blue-box`类。

对于`para`类，我们将`font-size`设置为`20px`，将`margin-bottom`设置为`1.5em`。这意味着边距将是段落元素字体大小的1.5倍。`1.5em`导致段落底部有30像素的边距。我们还设置了`2px solid red`的`border`，以便您能更好地看到边距。

对于`blue-box`类，我们将背景颜色设置为`blue`，文本颜色设置为`white`，并将四边的`padding`设置为`10px`。

从这个例子中，段落元素底部和蓝色盒子之间会有明显的空间。

那么如果我们从`para`类中移除`font-size`属性会发生什么？

那么，底部边距将相对于父元素的字体大小。在这种情况下，父元素是body元素，它的默认字体大小为`16px`。

`em`的良好用例是在处理模块化组件（如按钮或卡片）时。通过使用`em`单位，您可以确保组件的所有方面（如内边距、外边距和边框）都与字体大小成比例缩放，保持一致的比例。

到目前为止，我们一直在使用像素设置元素的字体大小。但这确实给用户带来了一些挑战。

在您的浏览器设置中，您可以控制默认字体大小。

对于有视觉障碍的用户，他们可能会增加字体大小以便更容易阅读。但如果您在网页设计中为字体大小设置像素，文本将不会与其余内容成比例缩放。

解决这个问题的一种方法是对排版使用`rem`单位。`rem`单位相对于根元素的字体大小，即`html`元素。

默认情况下，`html`元素的字体大小为`16px`。如果用户在浏览器设置中增加了字体大小，`html`元素的字体大小将增加，所有rem单位将成比例缩放。

下面是使用`rem`单位而不是像素设置字体大小的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p>This is regular text.</p>
<p class="para">This text is slightly larger.</p>
```

```css
.para {
  font-size: 1.2rem;
  margin-bottom: 1.5em;
  border: 2px solid red;
}
```

:::

通过将字体大小设置为`1.2rem`，段落元素的字体大小将是根元素字体大小的1.2倍。如果用户没有更改默认字体大小，段落元素的字体大小将是`19.2px`，因为它是1.2倍的`16px`。

那么什么时候应该使用`rem`单位呢？`rem`单位比像素更受青睐，因为它们会与用户的浏览器设置成比例缩放。这使得您的内容对有视觉障碍的用户更加可访问。

`rem`单位还可以帮助在不同元素之间保持一致的间距和布局。
