# --interactive--

在CSS中，`vh`和`vw`是相对于视口的单位，允许您根据浏览器窗口的尺寸来调整元素大小。这些单位在创建适应不同屏幕尺寸的响应式设计时特别有用。

`vh`代表"视口高度"，`1vh`等于视口高度的1%。

同样，`vw`代表"视口宽度"，`1vw`等于视口宽度的1%。

这意味着如果您将元素的高度设置为`100vh`，它将占据视口的完整高度，无论设备的实际像素尺寸如何。

这些单位在您想要创建全屏布局或保持屏幕特定比例的元素时特别方便。

例如，您可能希望使用它们来创建始终填充整个屏幕的英雄区域：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

这个CSS确保英雄区域始终与视口大小完全相同，无论设备的屏幕尺寸如何。

`vh`和`vw`单位也可以用于排版，以创建响应式文本大小。

尝试调整预览窗口的大小，查看文本如何随着视口大小缩放：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

这将把`h1`元素的字体大小设置为视口宽度的5%，允许文本随着浏览器窗口大小平滑缩放。

`vh`和`vw`单位的优点之一是它们会实时响应视口大小的变化。这意味着如果用户调整浏览器窗口大小，使用这些单位调整大小的元素将相应调整，而无需重新加载页面。但是，重要的是要谨慎使用这些单位。例如，仅使用`vw`单位设置字体大小可能导致文本在窄屏幕上过小或在宽屏幕上过大。

另一个需要考虑的因素是，在移动设备上，当浏览器地址栏出现或消失时，视口高度可能会发生变化，如果您大量使用`vh`单位，这可能导致意外的布局变化。

总之，`vh`和`vw`单位是创建响应布局和适应视口大小的元素的强大工具。它们特别适用于全屏区域、保持宽高比以及创建平滑缩放的设计。然而，应该深思熟虑地使用它们，并且经常与其他CSS技术结合使用，以确保在所有设备上获得最佳用户体验。
