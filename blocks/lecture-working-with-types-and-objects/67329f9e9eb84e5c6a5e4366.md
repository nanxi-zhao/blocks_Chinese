# --interactive--

在之前的模块中，您已经习惯了使用这样的字符串字面量：

```js
const greeting = "Hello, World!";
```

但JavaScript也有字符串对象。字符串对象和字符串原始类型都用于处理文本，但它们在底层的工作方式不同。字符串对象是使用字符串构造函数创建的，该函数将原始值包装在对象中。以下是创建字符串对象的方法：

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

当我们使用`typeof`操作符时，可以看到结果是object类型而不是string类型。您可能一直在思考的一个问题是，如何在字符串原始类型上使用像`.length`属性这样的属性。

当您在字符串原始类型上使用`length`属性时，JavaScript会临时将字符串原始类型包装在字符串对象中，以执行操作。这就是为什么您可以使用`length`属性以及不同的方法如`repeat()`、`concat()`和`slice()`的原因。这些类型的方法和属性被称为实例方法以及属性和静态方法。您将在未来的模块中详细了解其工作原理。

字符串对象和字符串原始类型的一个关键区别是它们与内存和性能的关系。字符串原始类型通常比字符串对象更节省内存且更快。尽管您在代码中主要使用字符串原始类型，但了解字符串对象的工作原理仍然很有用。
