# --interactive--

一元运算符对单个操作数执行操作，以执行类型转换、值操作或检查某些条件等任务。让我们看看一些常见的一元运算符以及它们是如何工作的。

一元加法运算符将其操作数转换为数字。如果操作数已经是数字，则保持不变。

:::interactive_editor

```js
const str = '42';
const strToNum = +str;

console.log(strToNum); // 42
console.log(typeof str); // string
console.log(typeof strToNum); // number
```

:::

一元加法在你想确保处理的是数值时很有用。你可能猜到了，还有一元取反运算符。它对操作数的值取反。它的工作方式与一元加法类似，只是它会翻转符号。

:::interactive_editor

```js
const str = '42';
const strToNegativeNum = -str;

console.log(strToNegativeNum); // -42
console.log(typeof str); // string
console.log(typeof strToNegativeNum); // number
```

:::

逻辑非运算符由感叹号（`!`）表示，是另一个一元运算符。它翻转其操作数的布尔值。所以，如果操作数是 `true`，它变成 `false`，如果它是 `false`，它变成 `true`。

:::interactive_editor

```js
let isOnline = true;
console.log(!isOnline); // false

let isOffline = false;
console.log(!isOffline); // true
```

:::

按位非运算符是一个较少使用的一元运算符。由波浪号 `~` 表示，它反转数字的二进制表示。计算机以二进制格式（1和0）存储数字。`~` 运算符翻转每一位，意味着它将所有1变为0，所有0变为1。你将在未来的课程中了解更多关于二进制和位的内容。

:::interactive_editor

```js
const num = 5; // 5的二进制是 00000101

console.log(~num); // -6
```

:::

在这个例子中，`5` 变成了 `-6`，因为通过将 `~` 运算符应用于 `5`，你得到 `- (5 + 1)`，由于二进制补码表示法，这等于 `-6`。二进制补码是计算机在二进制中表示负数的方式。除非你正在处理直接操作位的低级编程任务，否则你可能不会经常使用按位非运算符。

`void` 关键字是一元运算符，它计算表达式并返回 `undefined`。

:::interactive_editor

```js
const result = void (2 + 2);

console.log(result); // undefined
```

:::

`void` 也常用于超链接中以防止导航：

```js
<a href="javascript:void(0);">点击我</a>
```

最后，还有你在之前课程中学到的 `typeof` 运算符。这会将其操作数的类型作为字符串返回。

:::interactive_editor

```js
const value = 'Hello world';

console.log(typeof value); // string
```

:::
