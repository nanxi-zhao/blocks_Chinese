# --description--

JavaScript引擎有能力读取、理解和执行你的代码。它就像一个转换器，接收你的代码，将其转换为计算机能够理解和相应工作的指令。

最著名的JavaScript引擎之一是Google开发的V8，用于Chrome和Node.js。JavaScript引擎的工作分为几个步骤。首先，它解析你的代码，逐行读取以确保JavaScript代码中没有错误。然后，它将此代码转换为字节码，这是你的代码的更简单、中间版本，计算机更容易理解和执行。最后，它运行这个字节码来执行你的程序指令。下面是JavaScript代码的示例：

```js
const greeting = "Hello, World!";
console.log(greeting);
```

当你运行这段代码时，JavaScript引擎首先解析它以检查任何语法错误。解析意味着引擎读取代码并将其分解为它能理解的结构，在此过程中检查错误。

然后，它将代码编译成中间格式(通常是字节码或机器码，取决于引擎)。编译是将人类可读的代码转换为计算机可以更快执行的更高效格式的过程。

最后，引擎执行代码，在控制台打印`Hello, World!`。

现在，让我们谈谈JavaScript运行时。JavaScript运行时是你的JavaScript代码被执行的环境。它包括JavaScript引擎(如Chrome中的V8或Firefox中的SpiderMonkey)，它处理和执行代码，以及环境提供的额外功能(如Web浏览器或Node.js，你将在未来的课程中了解更多)。

虽然核心JavaScript语言处理变量、循环和函数等事物，但运行时提供了额外的工具，允许JavaScript与语言本身之外的事物交互，如DOM(用于网页)或Fetch API(用于发出网络请求)。

简而言之，运行时是允许JavaScript做更多基本编程任务之外的事情——比如与网页交互或处理基于时间的操作——通过提供超出语言本身的这些额外功能。

虽然你不需要知道编写JavaScript的引擎和运行时的每个细节，但有基本的了解可以帮助你编写更高效的代码并更有效地调试问题。
