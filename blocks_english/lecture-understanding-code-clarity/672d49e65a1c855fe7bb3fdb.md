# --description--

编程中的注释用于为代码提供额外的上下文或为自己和他人留下笔记。

注释是JavaScript引擎在执行代码时忽略的文本行或文本块。它们纯粹是为了阅读代码的人（无论是你自己还是其他人）的利益而存在的。

JavaScript提供了两种向代码添加注释的方式：单行注释和多行注释。

单行注释使用两个正斜杠（`//`）创建。以下是一个示例：

```js
// 我是JavaScript中的单行注释
```

这种类型的注释非常适合简短的解释或澄清。

以下是一个来自主课程项目文件的真实示例：

```js
// 这是为了允许英语版本构建而无需下载i18n文件。
// 如果这些文件不存在，当尝试解析i18n-curriculum路径时会失败。
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

不用担心试图理解代码实际在做什么，因为它比你目前学到的内容更高级。相反，专注于开发人员留下的注释。这个注释为为什么存在这段代码提供了重要的上下文。

像这样的注释对团队合作的人员很重要，原因有两点：

1. 项目中的其他开发人员将理解这段代码的目的。

2. 它有助于防止在未咨询团队的情况下进行不必要的更改或删除，这可能导致错误或问题。

另一种注释是多行注释。以下是基本语法：

```js
/*
 我是多行注释。
 这对于较长的解释很有帮助。
*/
```

当你需要在代码中编写较长的描述、解释或笔记时，多行注释非常有用。

让我们再次看看主课程项目文件，看看多行注释在现实世界中是如何使用的。

```js
/* 由于完成认证可能有多种方式（例如，使用
旧课程或新课程），我们需要一个认证
字段来跟踪这属于哪个认证。 */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

就像之前一样，忽略所有的JavaScript代码，因为它使用了尚未教授的概念。相反，专注于开发人员留下的注释。

团队中的开发人员，甚至是项目的新贡献者，可以理解为什么这段代码在这里，并在处理项目这个区域之前获得完整的上下文。

虽然注释在编程中很有用，但重要的是要避免过度注释。你不需要为每一行代码都添加注释，特别是如果代码是直截了当且自解释的。

以下是一个用注释解释显而易见内容的示例：

```js
// 这段代码使用const关键字创建一个名为price的新变量。
// 我们将数字10赋值给price变量。
const price = 10;
```

在这种情况下，不需要添加任何注释，因为代码是自解释的。目标是增强可读性，而不是用不必要的解释来 clutter 代码。

如果你想在学习编码时为个人项目添加注释，这很好。但一旦你开始与其他开发人员一起处理现实世界的项目，重要的是不要为自解释的代码使用注释。

同样重要的是不要使用注释来帮助解释混乱、过于复杂或编写糟糕的代码。在这些情况下，最好重构或更改代码，以便其他开发人员能更好地理解发生了什么。

注释是记录代码和使其更易于理解的强大工具。你应该使用注释来提供上下文或为自己和他人留下笔记。
