# --description--

函数式编程是关于编写干净、可预测、易于测试和调试的代码。今天我们将会探讨两个关键概念：纯函数和避免副作用。

让我们从纯函数开始。

纯函数是指给定相同输入时，总是返回相同输出，并且不会修改自身之外任何东西的函数。

以下是纯函数的一个示例：

```js
function add(a, b) {
  return a + b;
}
```

这个函数是纯函数，因为它总是为相同的输入返回相同的结果，并且不会改变自身之外的任何东西。无论你调用多少次`add(2, 3)`，它总是返回`5`，并且不会影响程序的任何其他部分。

现在，让我们将其与非纯函数进行对比：

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

这个函数是非纯函数，因为它修改了自身之外的变量（`total`）。`addToTotal(5)`的结果将取决于函数调用前`total`的值。这种不可预测性会使我们的代码更难理解和调试。

这引出了我们下一个重要概念，那就是副作用。

副作用是指函数被调用时程序状态发生的任何变化。这可能包括修改全局变量、写入文件或进行API调用。

纯函数没有副作用，但非纯函数有这些副作用。

避免副作用是函数式编程的关键原则。当我们最小化副作用时，代码变得更加可预测且更易于测试。

在编写代码时，我们的主要目标应该是让函数做一件事，并且要做好，而不改变程序的其他部分。

以下是带有副作用的函数示例：

```js
function greet(name) {
  console.log(`你好, ${name}!`);
}
```

这个函数有副作用，因为它通过记录到控制台与外部世界交互。

在函数式编程中，我们可能会这样重写：

```js
function greet(name) {
  return `你好, ${name}!`;
}
console.log(greet("Alice"));
```

现在`greet`是一个纯函数，只返回一个字符串，我们单独处理副作用（记录到控制台）。

函数式编程鼓励我们通过将代码分解为小的、可预测的部分，使用纯函数来构建程序。因此，我们可以构建更容易理解、测试和维护的程序。
