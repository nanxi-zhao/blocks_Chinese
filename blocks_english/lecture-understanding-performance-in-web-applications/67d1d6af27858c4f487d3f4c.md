# --description--

当我们谈论性能时，这个词本身可能感觉有点主观，对吧？什么是好的性能？什么是差的性能？

有标准的方法来衡量性能，还是完全取决于用户的个人偏好和感知？让我们来看看。

我们可以将网站的性能分为两个不同的类别：感知性能和真实性能。

感知性能是用户对网站性能的感知。它是用户在响应性和可靠性方面的评估。这是一种主观的测量，所以很难量化，但它非常重要，因为用户体验决定了网站的成功或失败。

相比之下，真实性能是网站的客观和可测量的性能。它使用页面加载时间、服务器响应时间和渲染时间等指标进行测量。这些测量受与网络和代码本身相关的多个因素的影响。

即使一个网站具有出色的感知性能，也可以进一步优化以获得更好的用户体验。

有几种技术可以提高感知性能。

通过在后台加载非必要资源，尽可能减少初始加载时间是至关重要的。这种技术被称为"懒加载"。第一印象可以决定用户对你的网站的看法。

懒加载也会影响你网站的真实性能。通过使用这种技术，你可以减少必须预先加载的内容量，所以页面会加载得更快，这是一个真实的性能指标。

提供对用户交互的快速响应和反馈也是非常重要的。

例如，在用户点击元素时立即显示长时间运行过程的加载指示器，可以帮助用户感觉与过程连接并参与其中，使等待时间感觉更短——或者至少他们会感觉更投入。

让用户积极参与你的网站对于改善他们的时间感知非常重要。尽快显示文本有助于从一开始就让用户保持参与，即使其他资源（如图像）尚未加载。

如果用户正在等待长时间运行的任务完成，请让他们了解进度并定期更新。

另一个提示是避免内容重排和跳跃内容。例如，当广告或图像正在加载时，如果它们在布局中还没有空间，网站可能会跳动或重新调整以腾出空间。

这些突然的变化可能导致糟糕的用户体验，因为用户可能感觉网站仍在加载。为了避免这种情况，提前计划并从一开始就为这些元素分配空间。

如果你的网站有自定义字体，你也应该尽量减少字体加载延迟，因为这可能导致闪烁——在加载自定义字体时显示后备字体。对此的建议是使用与自定义字体相似的后备字体，所以如果发生这种情况，变化会更加微妙。

另外，确保界面元素是活跃的。用户应该能够以最小的延迟与它们交互。

我们将在接下来的几节课中更详细地介绍这些技术。

虽然真实性能很重要，但感知性能可以对用户体验产生巨大影响。通过优化两者，你可以创建既快又感觉快的网站，创造流畅且引人入胜的用户体验。
