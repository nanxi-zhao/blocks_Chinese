# --description--

为了衡量和改善Web性能，我们使用关键性能指标，如首次内容绘制（FCP）、速度指数、总阻塞时间（TBT）、跳出率和独立用户数。让我们逐一查看这些指标。

首次内容绘制（FCP）。这衡量第一个内容片段（如文本或图像）出现在屏幕上的速度。良好的FCP被认为是低于1.8秒的时间，而差的FCP通常超过3秒。你可以使用Chrome DevTools检查你的FCP，如下所示：

步骤1：打开Chrome DevTools

- 打开Google Chrome。

- 转到你想要测试的网页。

- 按"F12"或右键单击并选择"检查"。

步骤2：在DevTools中找到并点击"Lighthouse"选项卡。

步骤3：点击"分析页面加载"按钮并等待分析完成。

现在我们可以分析结果。我们通过滚动到"指标"部分并寻找"首次内容绘制"标记来实现。我们可以看到以毫秒或秒为单位的确切时间。

FCP和我们之前讨论过的下一绘制交互（INP）都是性能指标，但它们衡量用户体验的不同方面。重要的是不要混淆它们。

提醒一下，FCP是关于用户在加载后看到内容的速度，而INP是关于用户在点击或输入后看到反馈的速度。

接下来我们有速度指数。

速度指数衡量页面可见部分绘制的速度。分数越低表示性能越好！你可以在Lighthouse报告中检查你的速度指数。

我们还有总阻塞时间（TBT）。这显示主线程被重型JavaScript任务阻塞的时间。如果TBT很高，用户会体验到卡顿的交互。为了改善TBT，可以分解长任务并延迟非必要脚本。

接下来我们有跳出率。这是没有交互就离开的访问者百分比。如果你的网站跳出率很高，可能是因为你的页面太慢。

最后我们有独立用户数。这个指标跟踪访问你网站的独立访客数量。要查看跳出率和独立用户数，你可以使用Google Analytics。它将允许你监控这些指标并提高参与度。
