# --description--

在软件开发中，你应该了解两种测试方法。这些方法是TDD（测试驱动开发）和BDD（行为驱动开发）。

让我们首先看看TDD。测试驱动开发是一种方法论，强调在构建应用程序功能之前先编写测试。

以下是TDD的典型过程：

1. 1. 编写测试

2. 2. 确保测试失败

3. 3. 为该测试编写代码

4. 4. 确保测试这次通过

5. 5. 对代码进行必要的重构

重复步骤1-5。

在构建功能之前编写测试，在开发过程中为开发人员提供实时反馈。它还有助于在开发阶段早期发现bug。

虽然一些软件团队会选择遵循TDD方法，但TDD在行业中是一个两极分化的话题。你会发现大量文章和视频都在支持或批评TDD。无论你对这种方法有什么看法，了解它都很重要，因为你可能会在现实场景中遇到它。

我们要看的第二种测试方法是行为驱动开发(BDD)。行为驱动开发是将一系列测试与业务目标对齐的方法。这些测试应该用技术和非技术人员都能理解的语言编写。

在测试软件应用程序中的特定行为时，BDD通常使用Gherkin语言，这种语言在业务中用于描述行为和场景。Gherkin语言在描述场景时使用Given、When和Then语句。

这是一个例子：

**场景：** 创建新博客文章

**假设** 用户已登录其账户并在"创建新文章"页面上，

**当** 用户输入文章标题和内容并点击"发布"按钮时，

**那么** 博客文章应该被保存并在首页上显示正确的标题、内容和时间戳。

你首先想要定义一个场景。示例场景包括用户登录应用程序或在网站上下订单。在这个例子中，场景是用户在网站上创建新博客文章。

在定义场景后，你应该从Given语句开始。这为用户在执行操作之前所处的位置提供了上下文。When语句用于表示用户采取的操作。

最后，Then语句用于表示用户先前操作的期望结果。一旦你定义了所有场景，就可以为每个场景编写测试。

BDD测试框架的例子包括Cucumber、JBehave和SpecFlow。
