# --interactive--

`change`事件是一个特殊事件，当用户修改某些输入元素的值时会触发该事件。更具体地说：

- 当复选框被勾选或取消勾选时。

- 当单选按钮被勾选时。

- 当用户从日期选择器或下拉菜单等组件中进行选择时。

- 当输入框失去焦点（用户切换到下一个字段，或点击表单外）且用户已更改值后。

- 当用户以其他方式确认值时，例如在输入一些文本后按回车键。

请注意，当用户在输入框中输入内容时，`change`事件不会触发。`change`事件只会在用户聚焦到另一个元素后才会触发。

下面是使用`change`事件与下拉菜单的示例：

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

每次用户从下拉菜单中选择不同的选项时，`change`事件都会触发，并将选中的值记录到控制台。

`change`事件仍然会生成一个`Event`对象，但与大多数其他事件不同，它不会生成自定义实现——你只能访问基`Event`对象上的属性和方法。

这与生成专用`InputEvent`对象的`input`事件不同。`change`事件在几个方面也有所不同。例如，当用户在字段中输入内容时，`input`事件会触发。

这些差异很重要，因为这些事件触发的时机可能会让你感到困惑。
