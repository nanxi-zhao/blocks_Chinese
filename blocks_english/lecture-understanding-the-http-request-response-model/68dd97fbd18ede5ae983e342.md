# --description--

在上一课中，您学习了服务器处理 HTTP 请求并向后发送响应。该响应包括您在 Web 上看到的各种资产。

HTTP 响应返回的资产包括基于文本的资产、媒体资产、字体等。这些资产中的每一个都带有 **Content-Type** 头，它告诉浏览器如何正确处理和显示资产。

基于文本的资产一直是 Web 的基本组成部分。实际上，Web 最初是纯文本的。如今，基于文本的资产包括以下内容：

* **HTML（超文本标记语言）**：任何网页的基本结构。没有 HTML，网页就无法存在。

* **CSS（层叠样式表）**：控制网页外观的资产。使用 CSS，您可以将颜色、字体和布局等样式应用到 HTML。

* **JavaScript**：JavaScript 为 HTML 和 CSS 添加交互性和动态内容。

* **JSON 和 XML**：这两种格式允许您在服务器和浏览器之间传输结构化数据。

媒体资产包括图像、音频和视频。它们通过视觉和声音让 Web 变得生动。

许多网站依靠图像来传达视觉信息。图像有不同的格式，如 [JPG]、[JPEG]、[PNG]、[WEBP] 和 [GIF]。

像 YouTube 和 Vimeo 这样的网站提供视频内容，格式包括 [MP4]、[MOV]、[AVI]、[WebM] 等。

音频是许多平台的关键部分，因为它允许用户流式传输音乐、播客和其他声音内容。常见的音频格式包括 [MP3]、[AAC]、Ogg 和 [WAV]。Spotify 和 YouTube Music 等平台依靠这些格式提供高质量的音频体验。

HTTP 响应中返回的其他资产包括 [PDF]、[DOCX] 和 [PAGES] 等文档，[ZIP] 和 [RAR] 等压缩档案，[WOFF]、[WOFF2] 和 [TTF] 等字体文件，以及 [EXE] 等可执行文件。

以下是这些常见资产及其内容类型的表格：

| 资产类型    | Content-Type 头        |
| ----------- | ---------------------- |
| HTML        | text/html              |
| CSS         | text/css               |
| JavaScript  | application/javascript |
| JPEG        | image/jpeg             |
| PNG         | image/png              |
| GIF         | image/gif              |
| SVG         | image/svg+xml          |
| WebP        | image/webp             |
| WOFF        | font/woff              |
| WOFF2       | font/woff2             |
| TTF         | font/ttf               |
| OTF         | font/otf               |
| MP4         | video/mp4              |
| WebM        | video/webm             |
| Ogg (Video) | video/ogg              |
| MP3         | audio/mpeg             |
| AAC         | audio/aac              |
| WAV         | audio/wav              |
| Ogg (Audio) | audio/ogg              |
| JSON        | application/json       |
| XML         | application/xml        |
| PDF         | application/pdf        |
| ZIP         | application/zip        |
