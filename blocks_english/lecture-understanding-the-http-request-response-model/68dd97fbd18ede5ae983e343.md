# --description--

网站和应用程序需要收集数据并处理数据，或生成用户需要的内容，如搜索结果。因此，就像文本一样，表单也是 Web 的基本组成部分。

通过表单，用户可以注册并登录网站、发送消息和搜索项目。例如，您可以通过在 Web 上最流行的表单（Google 搜索输入）中键入查询来访问 Google 搜索结果页面。

让我们来看看表单的组成，如输入字段、[action] 属性、表单方法，最重要的是，提交表单时会发生什么。

表单使用 [form] 元素嵌入到网页中，但要使输入显示出来，您还需要 [input] 元素，根据您的需要而定。其中一些输入元素包括文本、电子邮件、密码、选择、文件、隐藏输入等。

要使用这些输入字段，您必须使用 [form] 元素将它们包装起来。每个输入都需要一个 [name] 属性，以便服务器知道哪些数据对应于每个字段。

以下是一个示例：

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

输入字段还需要进行验证，以确保用户输入正确的信息。

这种验证可以在客户端和服务器端进行。HTML 允许您使用 [required] 和 [pattern] 属性进行基本的客户端验证。即便如此，出于安全考虑，您还应该在服务器端进行验证，因为 HTML 验证主要用于用户体验和即时反馈。

[required] 属性指定必须填写输入字段，而 [pattern] 属性允许您输入正则表达式，强制用户按照您需要的方式输入信息。

以下是用户名字段的示例：

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

输入字段验证后，应进行提交。那么提交表单时会发生什么呢？

首先，要进行提交，表单需要有一个 [action] 属性，该属性指定浏览器应将输入字段的值发送到何处，以及一个 [method] 属性，该属性指定表单数据的提交方式。稍后会详细介绍 [method] 属性。

以下是 action 属性的示例：

```html
<form action="/register">
    <!-- input fields -->
</form>
```

[method] 属性可以是 [GET]、[POST]、[PUT]、[PATCH] 或 [DELETE]：

* [GET] 从服务器检索内容，例如搜索结果。

* [POST] 向服务器发送内容，例如通过输入字段输入的登录或注册信息。

* [PUT] 修改服务器上的内容。

* [PATCH] 部分修改服务器上的内容。

* [DELETE] 从服务器删除内容。

请注意，HTML 不直接支持 [PUT]、[PATCH] 和 [DELETE] 方法，但有解决方法。如果您使用原生 JavaScript，可以使用 Fetch API 指定方法，在 Node.js 和 Express 中，可以使用 [method-override] 包。

以下是发出 [GET] 请求的表单示例：

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

以下是发出 [POST] 请求的表单示例：

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

在 [GET] 或 [POST] 请求之后，服务器检查服务器端验证，并处理或拒绝输入值。

如果验证成功，服务器会发送请求的信息，如果验证失败则抛出错误。

如果验证成功，服务器的响应可能是重定向（在身份验证或搜索的情况下），或确认操作的成功消息。

以下是提交表单时发生的情况的回顾：

* 用户填写表单字段。

* 浏览器检查客户端验证（如果有），并在验证成功后继续。

* 浏览器根据 action 和 method 属性将表单数据发送到服务器。

* 服务器接收数据并处理数据（将其存储在数据库中、注册用户、登录用户、发送搜索查询等）。

* 如果有服务器端验证，服务器在处理前会重新检查数据。

* 服务器根据验证状态（成功或失败）响应成功消息、错误或重定向用户。

* 用户根据服务器响应看到反馈。
