# --description--

用户研究是对使用你产品的人群进行的系统性研究。目标是测量用户需求、行为和痛点。

用户研究有多种形式。也许最常见的之一是净推荐值(NPS)。NPS测量你的用户向朋友推荐你的产品的可能性。NPS通过在用户旅程的关键里程碑处提供的调查来测量，例如7天、30天和90天后。NPS在0到10的范围内测量，9和10表示你的网站的积极推广者。

另一个研究方向是退出访谈。这是在用户取消订阅或删除账户时向他们显示的调查。这份调查的数据可以让你深入了解导致用户流失的因素，这样你就可以解决这些问题。

另一方面，用户测试是指在用户与你的应用程序交互时捕获数据的实践。例如，正在进行beta测试的视频游戏就是一种用户测试。作为Web开发者，你可能会遇到A/B测试。A/B测试涉及将新功能提供给随机选择的用户子集。然后你可以利用分析数据来确定该功能是否有益。

最后，用户需求是指你的应用程序需要遵循的故事或标准。这可以指导开发过程。用户需求可能由用户研究或行业标准定义，甚至可能由利益相关者输入定义。

这些需求可能是功能性的，意味着它们规定了你的应用程序应该如何工作，或者非功能性的，意味着它们定义了你的应用程序应该如何表现。用户需求也不是静态的。用户测试和用户研究的信息都会影响需求，随着你的用户群体的变化，它们也会发生变化。

了解这些差异对于收集最准确的数据至关重要，这样你就能为终端用户提供最佳体验。
