# --interactive--

在使用CSS时，你经常会遇到元素的三种不同显示行为：[inline]、[block]和[inline-block]。

这些属性中的每一种都会影响元素的定位方式以及它们与页面上其他元素的交互方式。

在本课程中，我们将重点介绍[inline-block]属性的工作原理以及它与内联元素和块级元素的区别。

块级元素表现得像大容器或"块"，占据其父容器的全宽。它们总是在新行开始，并且可以调整其高度和宽度。

内联元素只占据所需的空间。它们在周围内容内流动，不会换到新行。

[inline-block]属性是这两种行为的混合体。像内联元素一样，[inline-block]元素保持在文本流中，不在新行开始。

但是，与内联元素不同，你可以调整[inline-block]元素的宽度和高度，就像块级元素一样。

简而言之，[inline]和[inline-block]之间的关键区别在于[inline]元素无法控制其大小，而[inline-block]元素允许完全控制尺寸，同时仍与其它内容保持内联。

让我们看一个示例。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block Element 1</span>
  <span class="inline-block-element element2">Inline-Block Element 2</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在上面的示例中，我们有一个[class]为[container]的[div]。在该[div]元素内，我们有两个[span]元素。

每个span元素都设置为[display:inline-block]，并且还设置了宽度和高度。

内联块元素将并排出现，因为它们表现得像内联元素，但它们也有指定的宽度和高度，这使它们具有块状属性。

但是，如果你移除[display: inline-block]属性，即使你明确定义了[height]和[width]，它们也不会被应用。

以下是修订后的CSS：

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block Element 1</span>
  <span class="inline-block-element element2">Inline-Block Element 2</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在此代码中，我们移除了[display: inline-block;]属性，但保持其他内容不变。在这里，[span]元素恢复为内联元素的默认行为。

结果，指定的宽度和高度被忽略，元素只占据其内容所需的空间。

理解[inline-block]的工作原理很有用，因为你可以用它来创建需要在连续文本流中同时进行对齐和尺寸控制的布局。
