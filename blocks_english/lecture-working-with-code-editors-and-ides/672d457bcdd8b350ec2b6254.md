# --description--

_Better Comments_ 是一个为特定代码注释提供特殊高亮的扩展。例如，它会标注 "TODO" 注释，以及表示问题或警告的语法。

_Code Spell Checker_ 提供在代码中拼写错误时的指示。因为它专为代码文件设计，所以会考虑驼峰命名法等。

_Error Lens_ 将帮助你捕获代码中的任何高亮错误。与其依赖 VS Code 默认显示的下划线，此扩展会高亮整行并显示错误消息。

_Indent Rainbow_ 为你的不同缩进级别添加颜色。结合 VS Code 原生的括号对颜色化，这可以极大地帮助识别代码的作用域。

或者也许你在寻找更实用的东西？考虑使用 AI 助手，如 _GitHub Copilot_ 或 _Tabnine_，在你编写代码时提供内联建议。

图标包，如 _VS Code Great Icons_，可以帮助你的文件树更清晰，更容易一目了然地解析。而像 _Colorize_ 这样的扩展可以帮助你理解 CSS 属性中的值。

你还需要针对项目的特定语言扩展。如果你使用 JavaScript，你可能需要 _ESLint_ 和 _Prettier_ 来检查和格式化代码。如果你使用 TypeScript，你可能需要 _Pretty Typescript Errors_ 来获得更易读的消息。

最后，你还可以让你的编辑器更有趣。_VS Code Pets_ 提供可配置的虚拟宠物，在你解决 bug 时陪伴你。

_Power Mode_ 将在你通过编写更多代码达到足够高的"连击"时创建炫酷的效果。

_Discord Presence_ 将让你向所有朋友展示你正在处理什么。

还有更多扩展可供选择。请随意探索扩展市场，看看什么最适合你。
