# --description--

**链表**是一种线性数据结构，其中每个节点都连接到序列中的下一个节点。

这些连接创建了一个看起来像节点链的数据结构，其中每个节点存储数据和对链表中下一个节点的引用。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-1.png" alt="linked list basic structure visualization" />

我们使用这些引用来从第一个节点到下一个节点，依此类推。

链表通常用于实现其他数据结构，如栈、队列和双端队列。它们也可以用于实现重要的图算法，如深度优先搜索和广度优先搜索。

## 单向链表

**单向链表**是一种链表类型，其中每个节点都连接到序列中的下一个节点。

每个节点通过存储对下一个节点的引用来连接到下一个节点。

每个节点的这个单一引用允许您沿一个方向遍历链表，从开始到结束。

搜索只能向前移动，不能向后移动。

在这个示例中，您将从头节点（节点A）开始。

**头**节点是链表中的第一个节点。

在单向链表中，头节点通常是唯一可以直接访问的节点。当您试图查找特定节点时，搜索过程将从这里开始。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-2.webp" alt="singly linked list traversal visualization" />

过程将从节点A开始，然后继续到节点B，然后是节点C，最后是**尾**节点D。如果您在代码中实现特定逻辑，它也可能在此之前停止。

**尾**节点是最后一个节点。它用于确定过程何时到达链表的末尾。

### **插入节点**

链表的伟大之处之一是它们没有固定大小。它们可以通过简单地更新节点之间的连接来根据需要扩展或收缩。

您可以在链表的开始、中间和末尾**插入**节点。

链表不一定需要按特定顺序存储节点。顺序将由节点之间的连接决定。

然而，如果您确实需要为特定用例保持节点的特定顺序，您可以通过在代码中实现该逻辑来实现，您实现的标准将决定节点是插入到开始、中间还是末尾。

要在链表的开始插入节点，您只需要在新节点和曾经是头节点的节点之间创建连接，并使新节点成为头节点。

这是一个示例，我们在开始处插入节点E并使这个新节点成为链表的头节点。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-3.png" alt="singly linked list insert at beginning visualization" />

在链表开始插入节点具有常数时间复杂度[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L58-L58)，因为它只需要更新对头节点的引用以及新头节点与序列中下一个节点之间的连接。

在这个示例中，我们在链表的开始插入节点E。这将正常工作。但如果我们要保持链表按字母顺序排序，节点E将必须插入到链表的末尾。

要在链表的末尾插入节点，首先您需要到达末尾，然后添加与新节点的连接使其成为新的尾节点。

此操作具有线性时间复杂度[O(n)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L64-L64)，其中n是存储在链表中的节点数，因为首先您需要到达链表的末尾进行插入，这需要从一个节点到下一个节点依次进行，直到到达末尾。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-4.png" alt="singly linked list insert at end visualization" />

如果节点必须插入到链表的中间某处，则节点之间的连接也必须更新。序列中的前一个节点应该连接到新节点，新节点应该连接到下一个节点，如下图所示。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-5.webp" alt="singly linked list insert in middle visualization" />

插入操作具有常数空间复杂度[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L58-L58)，因为插入新节点只需要创建它并更新节点之间的连接。此操作不依赖于链表本身的大小。

### **删除节点**

就像您可以插入节点一样，您也可以从链表的开始、中间和末尾删除它们。

要从开始删除节点，您需要更新对头节点的引用，该引用应该是序列中的下一个节点。

此操作具有常数时间复杂度[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L58-L58)，因为它只需要更新链表对头节点的引用。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-6.png" alt="singly linked list remove from beginning visualization" />

要从链表的中间删除节点，您需要更新前一个节点的引用，将其连接到序列中的下一个节点，形成它们之间的某种"桥梁"，如此图所示。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-7.png" alt="singly linked list remove from middle visualization" />

这将从连接序列中删除您想要删除的节点（在本例中是节点B），因此下次遍历时将无法到达它。

要从链表的末尾删除节点，您需要删除前一个节点的连接并使该节点成为新的尾节点。现在链表将在新的尾节点处结束。

此操作具有线性时间复杂度[O(n)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L64-L64)，因为首先您必须到达链表的末尾。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-8.png" alt="singly linked list remove from end visualization" />

删除操作具有常数空间复杂度[O(1)](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-common-data-structures/6895d06b5968736797c408e6.md#L58-L58)，因为删除节点不需要额外的内存。

## 双向链表

现在您对单向链表有了更多了解，让我们来谈谈双向链表。

在**双向链表**中，每个节点存储两个引用：对序列中下一个节点的引用和对序列中前一个节点的引用。

这意味着双向链表可以沿两个方向遍历。

<img src="https://cdn.MasterPuti.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-9.webp" alt="doubly linked list structure visualization" />

在这种类型的链表中，通常也在链表本身中保持对尾节点的引用，以便在必要时从末尾开始遍历。

听起来很棒，对吧？它们比单向链表更灵活。

然而，双向链表确实比单向链表需要更多的内存，因为每个节点存储**两个**引用而不是一个。

这是您在为项目选择正确的数据结构时应该记住的事情。

存在**权衡**。

插入和删除操作的工作方式完全相同。唯一的区别是现在您需要更新每个节点的两个引用，并跟踪对尾节点的引用，以便非常高效地在双向链表的末尾插入元素，并在必要时从后面开始遍历过程。

单向和双向链表是计算机科学中用于按顺序存储和操作元素的基本数据结构。了解它们的区别对于为您的特定应用程序选择正确的数据结构至关重要。
