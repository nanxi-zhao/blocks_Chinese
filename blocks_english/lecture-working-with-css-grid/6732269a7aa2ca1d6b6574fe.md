# --interactive--

隐式网格是指浏览器在网格布局中放置项目时自动创建的行和列——那些没有使用[grid-template-rows](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)或[grid-template-columns](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)显式定义的行和列。

控制浏览器隐式创建的列和行的属性是[grid-auto-columns](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L12-L12)和[grid-auto-rows](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L12-L12)。

隐式网格也指当您将项目放置在显式定义的行和列之外时，浏览器自动生成的额外行和列。

例如，假设您这样定义网格布局中只有两列显式列：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="grid-item">Item 1</div>
  <div class="grid-item">Item 2</div>
  <div class="grid-item">Item 3</div>
  <div class="grid-item">Item 4</div>
  <div class="grid-item">Item 5</div>
  <div class="grid-item">Item 6</div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: 100px 100px; /* Only 2 explicit columns */
}

.grid-item {
  background-color: burlywood;
  border: 1px solid orangered;
  padding: 0.5rem;
  margin: 0.5rem;
}
```

:::

两个项目填充了第一行，使用两个显式列——第一列中的[Item 1](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L23-L23)和第二列中的[Item 2](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L24-L24)。

接下来的项目开始新的一行，[Item 3](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L25-L25)在第二行的第一列，[Item 4](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L26-L26)在第二行的第二列，以此类推。

正如您已经看到的，显式网格是您有意设置的网格区域。也就是说，您使用[grid-template-rows](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)和[grid-template-columns](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)属性为网格布局显式定义的行和列。

下表总结了隐式网格和显式网格之间的区别：

| **特性**   | **显式网格**                                                 | **隐式网格**                                                  |
| -------- | -------------------------------------------------------- | --------------------------------------------------------- |
| **尺寸控制** | 完全可自定义使用[grid-template-rows](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)和[grid-template-columns](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L35-L35)。 | 由[grid-auto-rows](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L12-L12)和[grid-auto-columns](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L12-L12)控制，或者默认为[auto](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-css-grid/6732269a7aa2ca1d6b6574fe.md#L58-L58)。 |
| **默认行为** | 除非显式定义，否则不会更改。                                           | 自动适应放置在显式网格之外的项目。                                         |
| **复杂度**  | 需要更多的布局结构规划。                                             | 对于非结构化或可变内容更容易实现。                                         |
| **灵活度**  | 具有定义的行和列的刚性结构。                                           | 灵活并适应动态放置的内容。                                             |
| **性能**   | 由于布局是预定义的，因此可能具有更高的性能。                                   | 隐式网格生成可能需要额外的浏览器计算。                                       |
| **用例**   | 当网格结构是可预测和预先定义的时候很有用。                                    | 对于内容未知或经常更改的动态布局非常有用。                                     |
