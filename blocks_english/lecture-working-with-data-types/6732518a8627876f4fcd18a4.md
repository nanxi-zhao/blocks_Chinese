# --interactive--

JavaScript 中的 `typeof` 运算符是一个简单而强大的工具，可以让您查看变量或值的数据类型。它总是返回一个表示类型的字符串。

让我们看几个例子：

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

在第一个例子中，我们创建了一个名为 `num` 的变量并为其分配了数字 `42`。当您在名为 `num` 的变量上使用 `typeof` 运算符时，它将返回字符串 `number`。

这里是另一个在名为 `isUserLoggedIn` 的变量上使用 `typeof` 运算符的例子：

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

当您在 `isUserLoggedIn` 变量上使用 `typeof` 运算符时，它将返回字符串 `boolean`，因为布尔值 `true` 被分配给了该变量。

使用 `typeof` 运算符在您调试或试图理解代码中处理的数据类型时特别有用。

然而，在 JavaScript 中关于 `null` 有一个众所周知的怪异行为。

让我们看一个例子：

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

在这个例子中，我们有一个名为 `exampleVariable` 的变量，并为其分配了值 `null`。但是当我们使用 `typeof` 运算符时，它返回了字符串 `object`。

这被广泛认为是 JavaScript 中的一个 bug，可以追溯到它的早期。这种行为的原因根植于 JavaScript 最初的设计方式。

当语言首次实现时，像 `null` 这样的值被表示为一种特殊类型的对象，导致了这种意外的结果。

不幸的是，这已经成为语言的一部分，虽然令人困惑，但这是您需要了解的内容。
