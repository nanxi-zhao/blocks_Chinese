# --interactive--

`break` 语句用于提前退出循环，而 `continue` 语句用于跳过循环的当前迭代并移动到下一个迭代。

以下是 `for` 循环中使用 `break` 语句的示例：

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break;
  }
  console.log(i);
}
```

:::

在上面的例子中，循环从 `0` 开始计数，当 `i` 小于 `10` 时，循环将继续运行。

在循环内部，我们检查 `i` 是否等于 `5`。如果是，我们使用 `break` 语句提前退出循环。如果不是，我们将 `i` 的值记录到控制台。所以代码的输出将打印数字 `0`、`1`、`2`、`3` 和 `4`。

当你想基于某个条件提前退出循环时，`break` 语句很有用。例如，如果你在数组中搜索特定值，你可以使用 `break` 语句在找到该值后退出循环。

有时你可能想跳过循环的特定迭代而不完全退出循环。这就是 `continue` 语句的作用。以下是 `for` 循环中使用 `continue` 语句的示例：

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    continue;
  }
  console.log(i);
}
```

:::

就像之前一样，我们将 `i` 初始化为 `0`，并有一个条件，当 `i` 小于 `10` 时运行循环。

在循环内部，当 `i` 等于 `5` 时，我们使用 `continue` 语句跳过当前迭代并移动到下一个迭代。

这段代码的输出将打印数字 `0`、`1`、`2`、`3`、`4`、`6`、`7`、`8` 和 `9`。数字 `5` 被跳过是因为 `continue` 语句。

你还可以对 `break` 和 `continue` 语句使用标签来指定要中断或继续哪个循环。

当你有嵌套循环并且想从内部循环控制外部循环的流程时，这很有用。

以下是使用标签与 `break` 语句的示例：

:::interactive_editor

```js
outerLoop: for (let i = 0; i < 3; i++) {
  innerLoop: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outerLoop;
    }
    console.log(`i: ${i}, j: ${j}`);
  }
}
```

:::

在这个例子中，我们有一个标记为 `outerLoop` 的外部 `for` 循环和一个标记为 `innerLoop` 的内部 `for` 循环。

当 `i` 等于 `1` 且 `j` 等于 `1` 时，我们使用带 `outerLoop` 标签的 `break` 语句提前退出外部循环。这将退出内部和外部循环。

这段代码的输出将在控制台中记录以下内容：

```md
"i: 0, j: 0"
"i: 0, j: 1"
"i: 0, j: 2"
"i: 1, j: 0"
```

大多数时候你不会发现需要在 `break` 和 `continue` 语句中使用标签，但知道如果你需要的话有这个选项是很好的。
