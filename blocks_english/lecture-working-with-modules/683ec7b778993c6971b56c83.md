# --description--

在软件开发中，库就像开发者的工具箱。

你不必从头开始实现代码的每个部分，库为你提供了预编写和可重用的代码，如函数、类和数据结构，你可以在项目中使用。

Python 拥有广泛的内置模块标准库。它们都是标准化、经过良好验证的解决方案，用于解决你作为程序员日常面临的许多问题和任务，例如：

* 与操作系统交互。
* 处理文件。
* 网络。
* 处理日期和时间。
* 执行数学运算。
* 使用正则表达式。
* 测试和调试代码。
* 还有更多！

一些流行的内置模块示例包括 `math`、`random`、`re`（"regular expressions" 的缩写）和 `datetime`。

`math` 模块有用于执行更复杂数学运算的有用函数。

`random` 模块有助于生成随机数。

`re` 模块用于处理正则表达式。

`datetime` 模块有助于在 Python 中处理日期和时间。

但是如何访问这些内置模块中定义的变量、常量、函数和类呢？

你使用导入语句。这些语句让你将模块导入到 Python 脚本中。导入语句通常写在文件的顶部。此外，你可以根据需要自定义它们。首先，你使用 `import` 语句，后跟模块的名称：

```python
import module_name
```

假设你想导入 `math` 模块。在这种情况下，你会在文件顶部写这个：

```python
import math
```

然后，如果你需要在 Python 脚本中调用该模块的方法，你会使用点表示法，模块名称后跟方法名称：

```python
module_name.method_name() 
```

例如，要获取 36 的平方根，你会写 `math` 后跟一个点，然后是 `sqrt`（平方根的缩写），在括号内传递任何必要的参数。在这种情况下，我们只需要传递我们想要平方根的数字：

```python
math.sqrt(36)
```

这是导入语句的最基本版本，但还有其他替代方案。

如果你想用不同的名称（也称为"别名"）导入模块，你可以使用这种语法，在导入语句末尾使用 `as` 后跟别名：

```python
import module_name as module_alias
```

这通常用于缩短长模块名称，或避免命名冲突。

例如，要在代码中将 `math` 模块称为 `m`，你可以给它分配一个别名，像这样：

```python
import math as m 
```

然后，你可以使用别名访问模块的元素：

```python
m.sqrt(36)
```

但有时你不需要从模块中导入所有内容。也许你只需要一两个特定的函数或类。Python 正好有你需要的东西。

现在导入语句以 `from` 开始，后跟模块名称，然后是 `import` 关键字，后跟你想要导入的元素名称：

```python
from module_name import name1, name2
```

然后，你可以在 Python 脚本中不使用模块前缀直接使用这些名称。

如果你想给这些名称分配别名，可以在每个名称后使用 `as` 关键字，后跟你想使用的别名：

```python
from module_name import name1 as alias1, name2 as alias2
```

假设你只想从 `math` 模块导入弧度、正弦和余弦函数。你会写：

```python
from math import radians, sin, cos
```

现在你可以直接在代码中调用这些函数，而不需要 `math` 模块作为前缀。

这里我们有一个更详细的示例：

要找到最初以度数表示的特定角度的正弦和余弦值，我们可以调用 `radians` 函数将其转换为弧度，然后调用正弦和余弦函数，传递以弧度表示的角度：

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

注意我们是如何直接调用函数的，没有使用模块名称作为前缀。这是因为我们用这种替代语法导入了函数。

这很有帮助，但如果 Python 脚本中已经定义了同名的函数或变量，它可能导致命名冲突。所以在选择要使用哪种导入语句时要记住这一点。

最后，我们找到以星号结尾的导入语句。星号告诉 Python 你想要导入该模块中的所有内容，但你希望导入它，这样你就不需要使用模块名称作为前缀：

```python
from module_name import *
```

例如，如果你在导入 `math` 模块时这样做，你将能够调用该模块中定义的任何函数，而无需指定模块名称作为前缀。这里有一些示例：

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

然而，这通常不被推荐，因为它可能导致命名空间冲突，并使确定某些名称的来源变得更加困难。

导入语句对于函数、类、常量、变量和模块中定义的任何其他元素都完全相同。

这是来自 `math` 模块的常量示例，数字 `pi`：

```python
import math
print(math.pi)
```

这是来自 `datetime` 模块的类示例。我们创建一个表示 1959 年 7 月 15 日的日期对象。然后，我们将该 `date` 对象分配给一个变量，并使用点表示法分别访问日、月和年：

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

你可以在该模块的官方 Python 文档中找到有关模块内容的更多信息。

很好。现在你知道了更多关于模块的知识，你也应该了解 Python 脚本中这个非常重要的习语，因为它们密切相关：

```python
if __name__ == '__main__': 
    # Code
```

`__name__` 是 Python 中的一个特殊内置变量。

当 Python 文件直接执行时，Python 将此变量的值设置为字符串 `"__main__"`。

但如果 Python 文件作为模块导入到另一个 Python 脚本中，`__name__` 变量的值被设置为该模块的名称（通常是不带 `.py` 扩展名的文件名）。

这就是为什么你经常在 Python 脚本中找到这个条件的原因。它包含你只想在 Python 脚本作为主程序运行时执行的代码：

```python
if __name__ == '__main__': 
    # Code
```

但如果脚本作为模块导入，该块中的代码不会运行。

这很有帮助，因为它允许 Python 脚本有两个目的。它们可以直接运行以执行其主要逻辑，或者可以导入到另一个模块中而不执行其主要逻辑。
