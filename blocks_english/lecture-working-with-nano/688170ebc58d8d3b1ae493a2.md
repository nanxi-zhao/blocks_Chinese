# --description--

在终端中工作时，标准文本编辑器并不总是可用的，但有程序可以直接在终端中编辑文件。

Linux 环境通常包含一个文本编辑器。Vi 或 Vim 是最常见的。Emacs 和 Nano 可能需要手动安装，但它们几乎在每个发行版中都 readily available。

Vim 的挑战在于它有相当大的学习曲线。如果你不熟悉它，你甚至可能不知道如何关闭应用程序！Emacs 可能稍微用户友好一些，但仍然有自己的键盘快捷键大全。

Nano 相当更加简化和用户友好，特别是对于新学习者。使用 Nano 可以更快地开始编辑文件，应用程序甚至在屏幕上提供基本键盘快捷键的帮助。

然而，这样做的代价是缺乏可扩展性。与 Vim 或 Emacs 不同（它们可以通过插件扩展成为完整的开发环境），Nano 相当简单。

让我们来看看 Nano 能做什么。要打开一个文件，你可以在终端中输入 `nano <filename>`。

文件打开后，你会在屏幕顶部看到你正在运行的 Nano 版本和你打开的文件名。

在那之下是文件的内容。你可以使用箭头键来导航光标，并用键盘插入或删除文本。文件会根据需要滚动。

然后，在底部，你会看到可用键盘快捷键的列表，看起来像这样：

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

以插入符号（`^`）开头的快捷键表示你需要按住 Ctrl 键并按下指示的字母。以 `M-` 开头的快捷键要求你按住"元键"，在大多数设置中应该是 Alt 键，并按下字母。

要在 Nano 中编辑文件，你只需要知道几个快捷键命令。对文件进行更改后，你可以按 `Ctrl + O` 来"写出"或保存文件。输入命令后，你会看到底部的菜单变成这样：

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

按回车保存文件并返回编辑文件，或按 `Ctrl + C` 取消并返回编辑文件。

在编辑器中，你可以按 `Ctrl + X` 退出 Nano 并返回终端。如果在尝试退出时有未保存的更改，你会在底部菜单中看到这个：

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

按 `Y` 保存未保存的更改，或按 `N` 丢弃它们。

有了这些知识，下次你需要在远程服务器上快速编辑文件时，你应该会准备得更好，在那里你可能无法访问完整的图形环境。
