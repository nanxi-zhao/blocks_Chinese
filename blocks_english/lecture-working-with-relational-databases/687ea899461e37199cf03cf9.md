# --description--

要在SQL中定义表格，您需要指定每列的数据类型。

记住这是在SQL中创建表格的基本语法。每列都有自己的数据类型。

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

作为开发人员，您的目标是尽可能准确地选择这些数据类型。

SQL中有六种流行的数据类型类别：

- 数值型，如`INTEGER`、`FLOAT`、`SERIAL`和`DECIMAL`。
- 日期和时间，如`TIMESTAMP`、`DATE`和`TIME`。
- 字符和字符串。包括`CHAR`、`VARCHAR`和`TEXT`。
- Unicode，包括`NTEXT`和`NVARCHAR`。这些用于确保文本将被正确存储和检索，无论字符的来源如何。
- 二进制，用于存储非文本数据，如图像、音频和视频文件。
- 以及其他杂项数据类型，如`XML`和`TABLE`。

在本课程中，我们将介绍PostgreSQL中最广泛使用的一些数据类型。数据类型在不同的数据库管理系统中会有所不同，但它们通常非常相似。因此，您需要在文档中查看完整列表。

我们从数值开始。

首先，我们有`INTEGER`数据类型：

```sql
units_sold INTEGER
```

在这个例子中，我们将此数据类型分配给`units_sold`列。

`INTEGER`类型在数据库中占用4个字节，其值范围从-2,147,483,648到2,147,483,647。PostgreSQL官方文档将其描述为"整数的典型选择"。

我们还有`SMALLINT`和`BIGINT`，它们基本上与`INTEGER`相同，只是由于在数据库中分配的大小不同，它们的数字范围更小和更大。

PostgreSQL中一个有用的功能是使用`SERIAL`关键字创建列。这里使用`SERIAL`创建了一个`id`列：

```sql
id SERIAL
```

虽然它不是真正的数据类型，但对于创建唯一标识符列非常有帮助，因为该列将具有`INTEGER`类型，不允许`NULL`值，并且在添加行时自动递增。

例如，第一行将自动具有`1`的`id`，第二行将具有`2`的`id`，依此类推，为每个记录创建唯一的ID。

在MySQL中，`SERIAL`的等价物是`AUTO_INCREMENT`属性，用于自动生成连续整数：

```sql
id INT AUTO_INCREMENT
```

您可以看到不同的数据库管理系统可能有不同的方式来实现数据库中的完全相同的功能。

这些是最常用的数值数据类型，但有时您可能需要表示文本或字符序列。

为此，您有`VARCHAR`数据类型。此数据类型用于可变长度字符串。您可以在括号内设置最大字符长度：

```sql
name VARCHAR(50)
```

在这个例子中，我们定义了一个`name`列，其中的值可以长达50个字符。

此数据类型为字符串设置了最大长度，但如果您需要存储任意长度的字符串，可以使用`TEXT`代替：

```sql
name TEXT
```

除了数字和字符串，存储日期和时间也很常见。

例如，如果您创建一个表格来存储事件，您可能需要以特定格式存储每个事件的日期。为此，您有`DATE`数据类型：

```sql
event_date DATE
```

要存储时间，您还可以使用`TIME`数据类型。例如，您可以存储事件开始的时间：

```sql
start_time TIME
```

`TIMESTAMP`数据类型结合了两者。它包括日期和时间。如果您写入`TIMESTAMP WITH TIME ZONE`，它还可能包括时区：

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

最后，如果您需要存储布尔值`TRUE`或`FALSE`，可以使用`BOOLEAN`数据类型：

```sql
is_active BOOLEAN
```

现在您了解了SQL中一些最常见的数据类型，但还有更多。您可以查看数据库管理系统的文档以获取有关可以使用的数据类型的更多信息。
