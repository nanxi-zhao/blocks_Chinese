# --description--

在React中，渲染是组件出现在用户界面（UI）中的过程，通常是浏览器。

React获取您所有的JavaScript、JSX和CSS代码，确定它应该如何显示，然后在用户界面中显示它。

渲染过程包括三个阶段：触发、渲染和提交。让我们更详细地看看这些阶段。

触发阶段发生在React检测到某些东西已经改变，用户界面可能需要更新时。这种变化通常是由于状态或props的更新引起的。

例如，注意到是晚餐时间可以触发你去厨房开始做饭。

在下面的`Counter`示例中，点击递增或递减按钮会触发React显示新的`count`值：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

一旦触发发生，React进入渲染阶段。在这里，React重新评估您的组件并确定要显示什么。

为此，React使用"真实"DOM的轻量级副本。这被称为虚拟DOM。通过虚拟DOM，React可以快速检查组件中需要改变什么。

将这个阶段想象为您在厨房里，您已经收集了食材，正在烹饪晚餐。

对于`Counter`组件，渲染阶段是React使用新的`count`值再次运行函数的点。React根据更新的`count`值重新计算组件的`<h1>{count}</h1>`部分应该是什么样子，但在下一个阶段——提交之前，您在屏幕上看不到任何变化。

提交阶段是React将虚拟DOM中的准备好的更改应用到真实DOM的阶段。换句话说，这是您在屏幕上看到最终结果的阶段。

为了实现这一点，React将虚拟DOM与实际DOM进行比较，识别需要更新的部分，并将这些更改应用到真实DOM以更新用户界面。

您可以将这个阶段想象为您端上烹饪的食物，使其可见，就像React在提交更新到实际DOM时所做的那样。

对于`Counter`组件，提交阶段是新`count`值应用于`h1`元素的点，您可以在页面上看到变化。

这三个过程非常快速，因为React通过首先在虚拟DOM中计算更改来最小化直接DOM操作，然后只更新真实DOM中需要改变的部分。
