# --interactive--

缩写词是单词或短语的缩写形式。例如，"Dr" 后面带点是 "doctor" 的缩写。

缩写词有两种常见形式：首字母缩略词和首字母缩写词。

首字母缩略词是由一个短语的首字母组成的单词，每个字母代表该短语中一个单词的首字母。

GUI 是首字母缩略词的一个例子。它代表 Graphical User Interface（图形用户界面）。通过取每个单词 G、U 和 I 的首字母，你得到首字母缩略词 GUI。

首字母缩写词由短语的首字母组成，每个字母代表该短语中一个单词的首字母。

例如，HTML 是一个首字母缩写词；它代表 HyperText Markup Language（超文本标记语言），并且通过拼出每个字母 H、T、M、L 来发音。

首字母缩略词和首字母缩写词都是缩写词的类型。区别在于首字母缩略词作为单词发音，而首字母缩写词作为单个字母发音。

它们对于编写更简洁的文本非常有帮助，特别是当它们在给定的上下文中众所周知且易于理解时。

如果你需要在 HTML 中显示缩写词如首字母缩略词或首字母缩写词，缩写元素正是你要寻找的。在第一次使用时，应解释其全部含义。然后，你可以使用缩写元素来突出显示它们，并提供更多细节。

下面是一个例子，你可以看到一个段落中的句子 `HTML is the foundation of the web`：

:::interactive_editor

```html
<p><abbr>HTML</abbr> is the foundation of the web.</p>
```

:::

首字母缩写词 HTML 在一个缩写元素中。在浏览器中，你会发现没有任何变化。它看起来仍像普通文本。缩写元素在幕后提供有用的上下文，但用户仍会看到首字母缩写词作为正常文本。

如果你想帮助用户理解这个首字母缩写词的含义，你可以使用 `title` 属性显示其完整形式。

`title` 属性是可选的，但如果你决定包含它，它必须是缩写词、首字母缩略词或首字母缩写词的人类可读描述。

让我们以之前的示例为例，添加 `title` 属性。它将是 `HyperText Markup Language`，即首字母缩写词的扩展形式：

:::interactive_editor

```html
<p><abbr title="HyperText Markup Language">HTML</abbr> is the foundation of the web.</p>
```

:::

通常，添加该属性后，缩写元素的样式会发生变化。具体样式取决于浏览器。有些浏览器可能会显示虚线下划线，而有些浏览器则会将内容转换为小写，甚至指定某些默认样式，如虚线下划线。当用户将鼠标悬停在缩写词上时，完整形式会显示为工具提示。

虽然你不一定需要为网页上的每个缩写词使用缩写元素，但对于那些可能不清楚且可能需要额外上下文的缩写词，建议使用。

你应该根据自己的判断，在信息和表述之间找到适当的平衡，既要简洁明了，又要避免杂乱无章。
