# --interactive--

在上一课中，你学习了如何使用 `arguments` 对象，它是一个包含传递给函数的参数值的类数组对象。

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// 结果:
// 1
// 2
// 3
```

:::

虽然这是访问和处理函数中可变参数集的有效方法，但现代 JavaScript 应用程序会使用剩余参数语法。

这里是使用剩余参数语法而不是 `arguments` 对象的更新示例：

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// 结果:
// 1
// 2
// 3
```

:::

这个更新的示例不再直接引用 `arguments` 对象。相反，函数定义的最后一个参数前面有三个点。这导致这个剩余参数被放置在一个 `Array` 对象中。你可以给这个剩余参数起任何你喜欢的名字。只要确保它是函数定义中的最后一个参数，就像这样：

```js
function exampleFunction(a, b, ...restOfArgs) {
  // 一些代码在这里
}

function anotherFunction(x, y, ...theArgs) {
  // 一些代码在这里
}
```

在处理剩余参数语法时还有一些限制。一个限制是函数定义只能有一个剩余参数。所以下面的例子被认为是无效的。

```js
// 不会工作

function badFunction(...args, ...moreArgs) {
  // 一些代码在这里
}
```

下一个限制是在剩余参数后面不允许有尾随逗号：`function exampleFunction(a, b, ...restOfArgs, )`。

另一个限制是剩余参数不能有默认值。否则会抛出 `SyntaxError`。

```js
function badFunction(...args = [1,2]){
  // 一些代码在这里
}
```

那么 `arguments` 对象和剩余参数之间有什么区别呢？

一个关键区别是 `arguments` 对象不是真正的数组，所以它不支持 `includes`、`pop` 和 `push` 等方法。但剩余参数是一个 `Array` 实例。所以你可以使用有效的内置数组方法，而无需先将其转换为真正的数组。

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
