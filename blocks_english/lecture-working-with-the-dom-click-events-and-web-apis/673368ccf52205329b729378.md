# --interactive--

让我们来学习[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)、[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)和[innerHTML](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368c0161e6b326a7e03f0.md#L14-L14)。

这些是你可以在JavaScript中访问的属性，用于获取或更改HTML元素的内容。即使它们看起来很相似，但它们确实有关键的区别。选择正确的属性取决于你的具体用例，让我们深入了解。

让我们从[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)开始。

[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)表示HTML元素及其后代的可见文本内容。这个属性不包括隐藏文本或HTML标签，只包括渲染的文本。

例如，这里你可以看到一个包含两个段落的[div](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L21-L24)元素：

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

如果我们在JavaScript代码中使用[getElementById()](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-arrays-variables-and-naming-practices/6733b43e3b72d0180404c53d.md#L41-L47)获取这个HTML元素的引用，我们可以访问这个元素的[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)属性：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

这是这个元素的内部文本：

```md
Hello, World!
I'm learning JavaScript
```

该属性返回一个包含元素内文本的字符串，包括其后代的文本。

你应该知道[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)只返回在请求字符串时可见的文本。如果子元素被隐藏，它的文本将不可见。

这是一个第二个段落被隐藏的示例：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

如果我们再次尝试记录[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)，现在输出将不包含第二个段落的文本：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

你可以像这样设置HTML元素的[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)，但这将替换现有文本并为每个换行符添加一个换行元素（[br](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L89-L89)）元素：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

由于[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)考虑了可见性，获取其值会触发一个称为"重排"的过程，重新计算网站上某些元素的位置。这个过程可能会消耗大量计算资源，所以你应该尽可能避免触发它。

很好。现在让我们谈谈[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)。

[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)返回元素的纯文本内容，包括其所有后代中的文本。

[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)和[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)之间最重要的区别是[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)总是返回HTML元素及其后代的完整文本内容，无论它是可见还是隐藏的。

这里我们有相同的HTML示例：

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

如果你尝试访问这个属性，你会看到元素及其后代的文本作为输出，保持缩进和间距：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

如果HTML元素不可见，就像这里我们隐藏了第二个段落的情况，它的文本仍将包含在此属性中：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)还将包括[script](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L163-L163)和[style](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L163-L163)等元素的内容。

如果你尝试替换节点上的[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)值，它将删除所有子节点并用包含新字符串的单个文本节点替换它们：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

最后，让我们谈谈[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)和[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)与[innerHTML](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368c0161e6b326a7e03f0.md#L14-L14)的区别。

记住，使用[innerHTML](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368c0161e6b326a7e03f0.md#L14-L14)你可以设置元素的内部HTML内容。这对于动态地将新HTML注入DOM很有帮助。

然而，记住如果无法控制字符串（例如包含用户输入数据的字符串），这会带来安全风险。如果该数据是恶意的，可能导致严重的安全问题。

为了避免这种情况，建议使用[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)属性来插入纯文本。

JavaScript中的[innerText](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L16-L16)、[textContent](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368ccf52205329b729378.md#L112-L112)和[innerHTML](file:///F:/Code/blocks_Chinese/blocks_english/lecture-working-with-the-dom-click-events-and-web-apis/673368c0161e6b326a7e03f0.md#L14-L14)属性提供了不同的方式来访问和操作HTML元素的内容。如果你的目标是有效地在JavaScript中处理HTML内容，你应该理解这些属性之间的区别。
