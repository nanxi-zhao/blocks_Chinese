## 可访问性树

可访问性树是辅助技术（如屏幕阅读器）用来解释和与网页内容交互的结构。

## `max()` 函数

`max()` 函数返回一组逗号分隔值中的最大值：

```css
img {
  width: max(250px, 25vw);
}
```

在上面的示例中，如果视口宽度小于 1000 像素，图像的宽度将为 250px。如果视口宽度大于 1000 像素，图像的宽度将为 25vw。这是因为 25vw 等于视口宽度的 25%。

## CSS 和可访问性的最佳实践

- **`display: none;`**：使用 `display: none;` 意味着屏幕阅读器和其他辅助技术将无法访问此内容，因为它不包含在可访问性树中。因此，只有在你想要完全从视觉呈现和可访问性中删除内容时才应使用此方法。
- **`visibility: hidden;`**：此属性和值在视觉上隐藏内容，但将其保留在文档流中，这意味着它仍占据页面上的空间。这些元素也不会被屏幕阅读器读取，因为它们已从可访问性树中删除。
- **`.sr-only` CSS 类**：这是一种常用技术，用于在视觉上隐藏内容，同时保持屏幕阅读器的可访问性。

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**：此属性和值启用平滑滚动行为。
- **`prefers-reduced-motion` 功能**：这是一个媒体功能，可用于检测用户的动画偏好。

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

在上面的示例中，如果用户未在设备上设置动画偏好，则启用平滑滚动。

## 使用 HTML 属性隐藏内容

- **`aria-hidden` 属性**：用于隐藏使用辅助技术（如屏幕阅读器）的人的元素。例如，这可用于隐藏不提供任何有意义内容的装饰性图像。
- **`hidden` 属性**：大多数现代浏览器都支持此属性，并在视觉上和可访问性树中隐藏内容。它可以轻松地通过 JavaScript 切换。

```html
<p aria-hidden>这个段落对视力正常的用户可见，但对辅助技术隐藏。</p>
<p hidden>这个段落对视力正常的用户和辅助技术都隐藏。</p>
```

## `placeholder` 属性的可访问性问题

使用占位符文本对可访问性不利。用户经常将占位符文本与实际输入值混淆——他们认为输入中已经有值了。
