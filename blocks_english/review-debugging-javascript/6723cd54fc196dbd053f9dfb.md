## 常见的错误消息类型

- **SyntaxError**：当你在代码中写错某些东西时会发生这些错误，比如缺少括号或方括号。可以把这想象成句子中的语法错误。

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**：有几种类型的引用错误，以不同的方式触发。第一种引用错误是未定义的变量。引用错误的另一个示例是在使用[let](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)或[const](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)声明变量之前尝试访问它。

```js
console.log(num);
const num = 50;
```

- **TypeError**：当你尝试对错误的类型执行操作时会发生这些错误。

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**：当你的代码尝试使用超出JavaScript可处理范围的值时会发生这些错误。

```js
const arr = [];
arr.length = -1; 
```

## [throw](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)语句

- **定义**：JavaScript中的[throw](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)语句用于抛出用户定义的异常。在编程中，异常是指发生意外事件并中断程序正常流程的情况。

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

## [try...catch...finally](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)

- **定义**：[try](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)块用于包装可能抛出错误的代码。它作为一个安全空间来尝试可能失败的操作。[catch](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)块捕获并处理[try](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)块中发生的错误。你可以在catch中使用错误对象来检查出了什么问题。[finally](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)块在[try](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)和[catch](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)块之后运行，无论是否发生错误。它通常用于清理任务，比如关闭文件或释放资源。

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

## 调试技术

- **[debugger](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)语句**：此语句让你在特定行暂停代码以调查程序中发生的情况。

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

- **断点**：断点让你在你选择的特定行暂停代码执行。暂停后，你可以检查变量、评估表达式并检查调用堆栈。
- **监视器**：监视表达式让你在代码运行时监视变量或表达式的值，即使它们超出了当前作用域。
- **性能分析**：性能分析通过让你捕获屏幕截图和记录CPU使用率、函数调用和执行时间来帮助你识别性能瓶颈。
- **[console.dir()](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)**：此方法用于显示指定JavaScript对象属性的交互式列表。它输出一个可以展开以查看所有嵌套属性的分层列表。

```js
console.dir(document);
```

- **[console.table()](file:///E:/1.work/byy/%E5%9C%A8%E7%BA%BF%E8%AF%BE%E7%A8%8B/blocks/node_modules/@types/node/stream/consumers.d.ts#L11-L11)**：此方法在控制台中将表格数据显示为表格。它接受一个必需参数，该参数必须是数组或对象，以及一个可选参数来指定要显示的属性（列）。
