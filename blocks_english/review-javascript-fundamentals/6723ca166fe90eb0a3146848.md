## 字符串构造函数和 `toString()` 方法

- **定义**：字符串对象用于表示字符序列。字符串对象是使用 `String` 构造函数创建的，该构造函数将原始值包装在对象中。

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

- **`toString()` 方法**：此方法将值转换为其字符串表示形式。这是你可以用于数字、布尔值、数组和对象的方法。

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

此方法接受一个可选的基数参数，该参数是 2 到 36 之间的数字。这个基数代表进制，例如二进制的基数 2 或八进制的基数 8。如果未指定基数，则默认为十进制的基数 10。

```js
const num = 10;
console.log(num.toString(2)); // "1010"(二进制)
```

## 数字构造函数

- **定义**：`Number` 构造函数用于创建数字对象。数字对象包含一些有用的属性和方法，如 `isNaN` 和 `toFixed` 方法。大多数情况下，你会使用 `Number` 构造函数将其他数据类型转换为数字数据类型。

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

## 变量和函数命名的最佳实践

- **驼峰命名法**：按照惯例，JavaScript 开发人员会使用驼峰命名法来命名变量和函数。驼峰命名法是指第一个单词全小写，后续单词首字母大写。例如：`isLoading`。

- **布尔值命名**：对于布尔变量，通常使用 "is"、"has" 或 "can" 等前缀是一种常见做法。

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **函数命名**：对于函数，名称应清楚地表明函数的作用。对于返回布尔值的函数（通常称为谓词），你可以使用相同的 "is"、"has" 或 "can" 前缀。当你有检索数据的函数时，通常以 "get" 开头。当你有设置数据的函数时，通常以 "set" 开头。对于事件处理函数，你可以使用 "handle" 前缀或 "Handler" 后缀。

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **循环内变量命名**：在循环中命名迭代器变量时，通常使用单个字母如 `i`、`j` 或 `k`。

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## 使用稀疏数组

- **定义**：可以有带有空槽的数组。空槽被定义为其中没有任何内容的槽。这与值为 `undefined` 的数组槽不同。这些类型的数组被称为稀疏数组。

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

## 代码检查器和格式化工具

- **代码检查器**：代码检查器是一种静态代码分析工具，用于标记编程错误、bug、样式错误和可疑结构。常见的代码检查器示例是 ESLint。
- **格式化工具**：格式化工具是自动格式化代码以符合特定风格指南的工具。常见的格式化工具示例是 Prettier。

## 内存管理

- **定义**：内存管理是控制内存的过程，在需要时分配内存，在不再需要时释放内存。JavaScript 使用自动内存管理。这意味着 JavaScript（更具体地说，是你网页浏览器中的 JavaScript 引擎）会为你处理内存分配和释放。你不必在代码中显式释放内存。这个自动过程通常被称为"垃圾回收"。

## 闭包

- **定义**：闭包是一个函数，它可以访问其外部（封闭）词法作用域中的变量，即使外部函数已经返回。

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

## `var` 关键字和提升

- **定义**：`var` 是 2015 年之前的原始变量声明方式。但是 `var` 在作用域、重新声明等方面存在一些问题。这就是为什么现代 JavaScript 编程使用 `let` 和 `const` 的原因。

- **使用 `var` 重新声明变量**：如果你尝试使用 `let` 重新声明一个变量，你会得到一个 `SyntaxError`。但是使用 `var`，你可以重新声明一个变量。

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // 这是允许的，不会抛出错误

console.log(myNum) // 10
```

- **`var` 和作用域**：在块（如 `if` 语句或 `for` 循环）内声明的 `var` 变量在该块外部仍然可以访问。

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

- **提升**：这是 JavaScript 在执行代码之前的编译阶段将声明移动到其各自作用域顶部的默认行为。当你使用 `var` 关键字声明变量时，JavaScript 会将声明提升到其作用域的顶部。

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

当你使用函数声明语法声明函数时，函数名和函数体都会被提升。这意味着你可以在代码中声明函数之前调用它。

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

使用 `let` 或 `const` 声明的变量会被提升，但它们不会被初始化，你在代码中的实际声明之前无法访问它们。这种行为通常被称为"暂时性死区"。

```js
console.log(num); // 抛出 ReferenceError
let num = 10;
```

## 使用导入、导出和模块

- **模块**：这是一个自包含的代码单元，封装了相关的函数、类或变量。要创建模块，你需要将 JavaScript 代码写在单独的文件中。
- **导出**：任何你想让应用程序其他部分可用的变量、函数或类都需要使用 `export` 关键字显式导出。有两种类型的导出：命名导出和默认导出。
- **导入**：要在应用程序的另一部分使用导出的项目，你需要使用 `import` 关键字导入它们。类型可以是命名导入、默认导入和命名空间导入。

```js
// 在名为 math.js 的文件中，我们导出以下函数：

// 命名导出
export function add(num1, num2) {
  return num1 + num2;
}

// 默认导出
export default function subtract(num1, num2) {
  return num1 - num2;
}

// 在另一个文件中，我们可以导入 math.js 中的函数。

// 命名导入 - 这行导入 add 函数。
// 函数名称必须与从 math.js 导出的完全匹配。
import { add } from './math.js';

// 默认导入 - 这行导入 subtract 函数。
// 函数名称可以是任何名称。
import subtractFunc from './math.js';

// 命名空间导入 - 这行导入文件中的所有内容。
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
