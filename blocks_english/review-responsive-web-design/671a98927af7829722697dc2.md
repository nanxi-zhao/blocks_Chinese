## 响应式网页设计

- **定义**: 响应式设计的核心原则是适应性——网站根据查看设备的屏幕尺寸和功能调整其布局和内容的能力。
- **流体网格**: 这些网格使用相对单位（如百分比）而不是固定单位（如像素），允许内容根据屏幕尺寸调整大小和重新排列。
- **灵活图像**: 这些图像设置为在其包含元素内调整大小，确保它们在较小屏幕上不会溢出容器。

## 媒体查询

- **定义**: 这允许开发人员根据设备的特性（主要是视口宽度）应用不同的样式。

```css
@media screen and (min-width: 768px) {
  /* 适用于至少768px宽的屏幕的样式 */
}
```

- **`all` 媒体类型**: 适用于所有设备。如果未指定媒体类型，这是默认值。
- **`print` 媒体类型**: 适用于分页材料和在打印预览模式下查看的文档。
- **`screen` 媒体类型**: 主要适用于屏幕。
- **`aspect-ratio`**: 描述视口的宽度和高度之间的比率。

```css
@media screen and (aspect-ratio: 16/9) {
  /* 适用于16:9宽高比屏幕的样式 */
}
```

- **`orientation`**: 用于指示设备是处于横向还是纵向。

```css
@media screen and (orientation: landscape) {
  /* 适用于横向的样式 */
}
```

- **`resolution`**: 用于描述输出设备的分辨率，单位为每英寸点数(dpi)或每厘米点数(dpcm)。

```css
@media screen and (min-resolution: 300dpi) {
  /* 适用于高分辨率屏幕的样式 */
}
```

- **`hover`**: 用于测试主要输入机制是否可以悬停在元素上。

```css
@media (hover: hover) {
  /* 适用于支持悬停的设备的样式 */
}
```

- **`prefers-color-scheme`**: 用于检测用户是否请求了浅色或深色主题。
- **媒体查询和逻辑运算符**: `and` 运算符用于组合多个媒体特性，而 `not` 和 `only` 可用于否定或隔离媒体查询。

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* 适用于至少768px宽的横向屏幕的样式 */
}
```

## 常见媒体断点

- **定义**: 媒体断点是网站设计中的特定点，在这些点上布局和内容会调整以适应不同的屏幕尺寸。有一些通用的断点可以用来针对手机、平板电脑和桌面屏幕。但试图追踪每一种可能的设备屏幕尺寸是不明智的。

```css
/* 适用于宽度大于768px的屏幕的样式 */
@media screen and (min-width: 768px) {
  body {
    font-size: 1.125rem;
  }
}
```

- **小设备（智能手机）**: 最大640px
- **中等设备（平板电脑）**: 641px到1024px
- **大设备（桌面）**: 1025px及以上

## 移动优先方法

- **定义**: `移动优先` 方法是响应式网页设计中的一种设计理念和开发策略，它优先为移动设备创建网站，然后再为更大的屏幕设计。

```css
/* 移动端的基础样式 */
.container {
  width: 100%;
  padding: 10px;
}

/* 大屏幕的样式 */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```
